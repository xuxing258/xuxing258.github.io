前后端分离

  • 什么是前后端分离?

    • 前后端不分离
    特点:
    -用户访问url
    -执行视图函数,进行业务处理
    -render函数,返回html页面+数据渲染.将渲染完成的html+css+js返回并呈现给用户浏览器
    
开发:
    -前端:html+css+js
    -后端:前段代码给后端,后端+前端代码集成到项目
    • 前后端分离
    特点:
    -一般基于vue.js,react.js等框架来编写前段页面(html+css+js)
    -页面上如果需要呈现数据,则需要通过ajax的形式向后端发送请求并获取数据
    -后端接收到请求后,执行视图函数并进行业务处理
    -后端的视图执行完毕后,给前端返回json格式的数据
    -前端接收到json格式的数据后呈现爱浏览器上
    
开发:
    -前段:html,css,js(数据都是通过调用后端接口获取)
    -后端:写接口
    -前后端约定好接口的规则

  • 为什么使用前后端分离?

    • 前后端职责清晰.前端开发者只用自己的框架写页面,后端开发者只用python编写后端代码.两者通过json格式请求数据的传输
    • 开发高效,前后端做自己擅长的领域
    • 有利于项目的扩展

  • restful规范

    因为前后端双方进行合作开发,就必须要提前约定好一些规范.

    restful是一套主流的接口规范.

    • 之前
    接口:
    /users/list/--->用户列表
	/users/add/--->添加用户
	/users/detail/(\d+)--->某个用户详细信息
    /users/edit/(\d+)--->更新用户
    /users/delete/(\d+)--->删除用户
    • restful
    接口:
    /users/  --->方法:get--->用户列表
    /users/ --->方法:post--->添加信息
    /users/(\d+)--->方法:get--->获取单条数据
    /users/(\d+)--->方法:delete-->删除数据
    /users/(\d+)--->方法:put--->更新数据
    /users/(\d+)--->方法:patch--->更新数据

restful规范

https协议

  • http协议:用户浏览器再向服务器发送数据时,都是以明文的形式传输
  • https协议:用户浏览器再向服务器发送数据时,是以密文的形式传输.就算中途数据被获取,也是密文的.

域名

对于后端api接口要体现api标识

https://api.xxxx.com
https://xxxx.com/api/

版本

对于后端api接口要体现版本

https://api.xxxx.com/v1/
https://api.xxxx.com/?version=v1
https://v1.xxxx.com

路径

restful规范认为网络上的一切都成为资源,围绕着资源可以进行增删改查等操作

这些资源,在url要使用名词表示(可以用复数),围绕着资源进行的操作就使用method进行区分

https://api.xxxx.com/v1/person/
https://api.xxxx.com/v1/books/

请求方法

get:在服务器取出资源(一项或多项)
post:在服务器新建一个数据
put:在服务器更新资源(提供完整的资源)
patch:在服务器更新资源(提供改变的属性)
delete:在服务器删除资源

搜索条件

在路由中通过参数的形式来传递搜索条件

https://api.xxxx.com/v1/books/?limit=10--->指定返回记录的数量
https://api.xxxx.com/v1/books/?offset=2--->指定返回记录的开始位置
https://api.xxxx.com/v1/books/?page=2&per_page=20--->指定第几页,以及每页的记录数

返回数据

针对不同操作,服务器向用户返回的结果应该不同

路由方法描述返回数据
/users/get列表返回资源对象的列表:[{'id':1, 'name':'xixi'}, {....}]
/users/post添加返回新生成的资源对象:{'id':1, 'name':'xixi'}
/users/(\d+)get获取单个数据返回单个的资源对象:{'id':1, 'name':'xixi'}
/users/(\d+)delete删除数据返回一个空文档:null
/users/(\d+)patch局部更新返回完整的资源对象{'id':1, 'name':'xixi'}
/users/(\d+)put更新数据返回完整的资源对象{'id':1, 'name':'xixi'}

一般在实际的开发过程中会对上述返回的数据进行补充.每次请求都返回一个字典:

  • code:表示返回码,用于表示请求执行情况.0表示成功,1003表示非法参数
  • data:表示数据
  • error:表示错误信息
# 正确的返回
{
    'code': 0,
    'data': 数据
}

# 错误
{
    'code':...,
    'error': ...
    
}

状态码

后端在进行响应时,除了返回数据以外还可以返回状态码,来表示请求状况

200 OK-[GET]:服务器成功返回用户请求的数据
201 CREATED-[POST/PUT/PATCH]:用户新建或者修改数据成功
202 accepted-[*]:表示一个请求已经进入后台排队
204 NO CONTENT-[delete]:用户删除数据成功

400 invalid request-[POST/PUT/PATCH]:用户发出的请求有错误,服务器没有成功进行新建或修改数据的操作
401 Unauthorized-[*]:表示用户未认证
403 Forbidden-[*]:表示用户得到授权,但是访问被禁止
404 not found-[*]:用户发出的请求是不存在的

500 internal server error-[*]:服务器发生错误

from django.http import JsonResponse

def users(request, version, pk=None):
    print(version)
    if not pk:  # 没有pk的
        if request.method == 'GET':
            # 请求用户列表
            info = {
                'code': 0,
                'data': [{'id': 1, 'name': 'xixi'}, {'id': 2, 'name': 'xiaomi'}]
            }
            return JsonResponse(info)
        elif request.method == 'POST':
            # 新增用户
            info = {
                'code': 0,
                'data': {'id': 3, 'name': 'haha'}
            }
            return JsonResponse(info)
        else:
            # 报错
            info = {
                'code': 1000,
                'error': '请求错误'
            }
            return JsonResponse(info)

    else:  # 有id
        if request.method == 'GET':
            # 获取单个用户信息
            info = {
                'code': 0,
                'data': {'id': 3, 'name': 'haha'}
            }
            return JsonResponse(info)
        elif request.method == 'PUT':
            # 返回更新的对象的全部数据
            info = {
                'code': 0,
                'data': {'id': 3, 'name': 'haha'}
            }
            return JsonResponse(info)

        elif request.method == 'PATCH':
            # 返回更新的对象的全部数据
            info = {
                'code': 0,
                'data': {'id': 3, 'name': 'haha'}
            }
            return JsonResponse(info)

        elif request.method == 'DELETE':
            # 删除对应pk的数据
            info = {
                'code': 0,
                'data': {}
            }
            return JsonResponse(info)

        else:
            # 报错
            info = {
                'code': 1000,
                'error': '请求错误'
            }
            return JsonResponse(info)
