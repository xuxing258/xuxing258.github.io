字段的增删改

只需要对models.py进行增删改,然后进行数据库迁移两条命令即可

  • 增加新字段
# 直接在models.py的模型类中添加新的属性,然后进行数据库迁移
# 1.给新的字段指定默认值
sex = models.CharField(max_length=32, default='女')

# 2.允许新的字段为空
hobby = models.CharField(max_length=32, null=True)

  • 删除字段

    注释掉或者删除某个属性,再进行数据库迁移两条命令

  • 修改字段

    可以直接修改属性:修改字段名,修改字段类型,修改约束

    修改完之后,进行数据库迁移两条命令

注意:如果数据库迁移不成功,且检查了models.py中没有任何问题.
    那么直接去mysql中删除对应数据库,再将migrations文件夹中的迁移记录全部删掉,重新进行数据库迁移两条命令

数据的增删改查

增加数据

# 方式1:实例化类的对象,调用save方法将对象保存(不通过save保存数据库中不会有数据)
stu_obj = models.Student(name=name, age=age)
stu_obj.save()

# 方式2:利用create函数,该函数返回值就是创建的对象本身
# create函数能够创建对象之后直接保存到数据库中
stu_obj = models.Student.objects.create(name=name, age=age)
print(stu_obj)  # 创建的对象
print(stu_obj.name)  # 对象名.属性名查看对应的属性
print(stu_obj.age)

查询数据

# filter(条件):带有条件的查询,返回值是类似于一个列表套对象的格式(queryset对象)
# first():拿到queryset对象中的第一条数据(数据对象)
user_obj = models.User.objects.filter(username=username, password=password).first()
print(user_obj)  # 满足条件的第一个用户对象
print(user_obj.username)  # 查看对象的属性

删除数据

# delete():直接操作数据库删除数据
# 先获取到要删除的数据对象,然后调用delete函数
models.Student.objects.filter(id=id).delete()
  • 不删除,修改属性
# 正常逻辑,不会直接将数据从数据库中删除.而是给数据一个是否删除的属性,但数据调用删除功能时,修改属性值为是删除.之后在查询的过程中还需要给定一个条件:不查询删除了的数据
# models.py
class Student(models.Model):
    name = models.CharField(max_length=32)
    age = models.IntegerField()
    hobby = models.IntegerField(null=True)
    is_delete = models.BooleanField(default=False)  # False表示未删除,True表示已删除
    

# urls.py
path('delete/<int:id>', views.delete, name='delete'),

# views.py
# 删除功能
def delete(request, id):
    models.Student.objects.filter(id=id).update(is_delete=True)
    return redirect('index')


# 查询数据时,不再通过all()查询所有数据,而是给定过滤条件查询没有被删除的数据
# views.py
# 查询所有数据
def index(request):
    stus = models.Student.objects.filter(is_delete=False)
    return render(request, 'index.html', locals())
