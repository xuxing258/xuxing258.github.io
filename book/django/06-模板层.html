1.实现注册功能
	用户输入用户名密码,确认密码,后端判断两次密码是否一致

# urls.py中定义路由和视图之间的对应关系
path('register/', views.register),

# views.py中定义视图函数实现注册功能
def register(request):
    if request.method == 'POST':
        username = request.POST.get('username')
        password = request.POST.get('password')
        re_password = request.POST.get('re_password')
        # 判断两次密码是否一致
        if password != re_password:
            return HttpResponse('注册失败!')
        return HttpResponse('注册成功,欢迎%s' % username)
    return render(request, 'register.html')


# register.html
<h1>注册</h1>
<form action="" method="post">
    <p>用户名: <input type="text" name="username"></p>
    <p>密码: <input type="text" name="password"></p>
    <p>确认密码: <input type="text" name="re_password"></p>
    <input type="submit">
</form>

2.实现登录功能
	用户输入用户名和密码,判断用户名是否存在,以及密码是否正确
    
# urls.py中定义路由和视图函数之间的对应关系
path('login/', views.login)

# views.py中定义视图函数
def login(request):
    if request.method == 'POST':
        username = request.POST.get('username')
        password = request.POST.get('password')
        # 判断用户名和密码是否正确
        if username == 'xixi' and password == '123456':
            return HttpResponse('欢迎%s' % username)
        return HttpResponse('用户名或密码错误')
    return render(request, 'login.html')

模板层

模板语法传值

不能直接运行单个html文件查看效果,模板语法的效果由django提供,必须经过路由访问执行视图函数返回html页面查看效果

render(request, 'html文件名字', 传递给html页面的数据)
传递给html页面的数据只能属于当前视图函数内部的名字
1.通过字典传参
{键:值}

2.locals():将当前函数内部的名字全部传递

基本语法

  • :变量相关
1.python所有的基本数据,都可以传递给模板页面,通过模板语法进行使用

2.除了基本的数据类型之外,还可以传递函数.传递函数名字能够自动调用函数得到返回值.但是模板语法不支持给函数传递额外的参数

3.传递类名,能够自动实例化一个对象.同样的当前类不能有初始化参数.但是可以直接同类名调用类中的方法

4.不管是列表的索引还是字典的键还是类中的函数调用都是通过句点符(.)
<p>通过索引获取列表中的数据:{{ list1.2 }}</p>
<p>获取字典中的数据:{{ dict1.d.2 }}</p>
<p>类名调用函数:{{ Mclass.get_self }}</p>
<p>类名调用函数:{{ Mclass.get_cls }}</p>
<p>类名调用函数:{{ Mclass.get_static }}</p>

过滤器(最多有两个参数)

相当于模板语法中的内置函数

  • 基本语法
{{ 数据|过滤器:参数 }}
数据是第一个参数,参数是第二个参数传递给过滤器使用
  • 内置过滤器
1. 转义:后端写好的html标签传递到前端页面想要当做标签使用
<p>后端拼接好的标签: {{ html1|safe }}</p>
    
2.日期格式化:y表示年, m表示月, d表示日, h表示时, i表示分, s表示秒
<p>时间: {{ now|date:'Y年m月d日 h:i:s' }}</p>
    
3.统计字符串长度
<p>统计字符串长度: {{ s|length }}</p>
    
4.截取字符
截取中文字符:截取时个数会包括三个点
<p>截取字符: {{ s1|truncatechars:4 }}</p>
截取英文单词:不会包括三个点(按照空格进行切割)
<p>截取单词: {{ s|truncatewords:3 }}</p>

自定义过滤器,标签,inclusion_tag

  • 准备工作
1.在应用文件夹中创建一个名字必须叫templatetags的文件夹
2.在该文件夹中创建任意名字的py文件
3.在该py文件中导入模块使用
from django import template
register = template.Library()
  • 自定义过滤器
# 1.在templatetags文件夹中的mytags.py文件中进行定义过滤器
# 自定义过滤器
@register.filter(name='hello')
def my_sum(a, b):
    return a+b

# 2.在要使用过滤器的html页面导入文件之后调用过滤器
{% load mytags %}  # 只需要在html文件中导入一次,后续可以一直使用
{{ n|hello:666 }}
  • 自定义标签
# 1.在templatetags文件夹中的mytags.py文件中进行定义标签
# 自定义标签
@register.simple_tag(name='plus')
def my_plus(a, b, c, d):
    return a+b+c+d


# 2.在要使用标签的html页面导入文件之后调用标签
{% load mytags %}
{% plus 123 123 123 123 %}
  • 自定义inclusion_tag
先定义一个方法,在要使用的html页面中调用这个方法,并且可以传参.该方法可以生成一些数据传递给另一个html页面渲染好结果之后.返回到调用该方法的地方

# 1.在templatetags文件夹中的mytags.py文件中进行定义inclusion_tag函数
# 自定义inclusion_tag
@register.inclusion_tag('left_memu.html')
def left_memu(n):
    data = ['第{}项'.format(i+1) for i in range(n)]
    return locals()  # 将当前函数内部的名字以字典的形式传递给left_memu.html页面


# 2.去到要渲染部分数据的html页面中将函数生成的数据进行渲染
<ul>
    {% for datum in data %}
    	<li>{{ datum }}</li>
    {% endfor %}
</ul>


# 3.在要使用inclusion_tag的html页面中调用函数
{% loads mytags %}
{% left_memu 6 %}

模板继承

有些网站的页面都大差不差,只在某一些局部在做变化

# 1.将页面制作成模板
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
<div style="background-color: red; width: 100%;height: 100px">
    这是上面的div
</div>
<div style="background-color: bisque; width: 100%;height: 500px">
    {% block content %}  <!--通过block划分可以让子页面重新写的区域-->
    	这是中间的div
    {% endblock %}
</div>
<div style="background-color: blue; width: 100%;height: 100px">
    这是下面的div
</div>
</body>
</html>


# 2.子页面继承模板重写block区域
{% extends '父页面名.html' %}  <!--完成继承父页面效果-->

{% block 区域名 %}  <!--重写父页面block区域-->
	子页面效果
{% endblock %}
