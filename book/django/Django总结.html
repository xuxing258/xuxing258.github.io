Django框架

安装指定版本

pip install Django==5.0.3

在pychar工具进行创建Django项目:

设置自定义环境启动项目

通过python 文件名.后缀 runserver 启动项目

python manage.py runserver 8080 指定端口执行

创建django目录应用 (视图模块应用)

python manage.py startapp 文件夹名称

项目目录

  1. manage.py: 以后和项目交互基本上都是基于这个文件。一般都是在终端输入python manage.py [子命令]。可以输入python manage.py help看下能做什么事情。除非你知道你自己在做什么，一般情况下不应该编辑这个文件
  2. settings.py :本项目的设置项，以后所有和项目相关的配置都是放在这个里面。
  3. urls.py:这个文件是用来配置URL路由的。比如访问http:/127.0.0.1/news/是访问新闻列表页，这些东西就需要在这个文件中完成。
  4. wsgi.py : 项目与wsGI协议兼容的wb服务器入口，部署的时候需要用到的，一般情况下也是不需要修改的。
  5. asgi.py：作为你的项目的运行在 ASGI 兼容的 Web 服务器上的入口
  6. __init__.py：一个空文件，告诉 Python 这个目录应该被认为是一个 Python 包
  7. models.py文件，用于定义数据库表对应的模型类

path函数

from django.urls import path 获取path函数

path(route, view, kwargs=None, name=None)

  • route: 参数是路由地址:(限制路由地址类型)

    • str - 匹配除了 '/' 之外的非空字符串。如果表达式内不包含转换器，则会默认匹配字符串。
    • int - 匹配 0 或任何正整数。返回一个 int 。
    • slug - 匹配任意由 ASCII 字母或数字以及连字符和下划线组成的短标签。比如，building-your-1st-django-site 。
    • uuid - 匹配一个格式化的 UUID 。为了防止多个 URL 映射到同一个页面，必须包含破折号并且字符都为小写。比如，075194d3-6885-417e-a8a8-6c931e272f00
    • path - 匹配非空字段，包括路径分隔符 '/'

  • view: 参数是视图函数或基于类的视图的结果

  • kwargs: 参数允许你向视图函数或方法传递附加参数

  • name: 命名你的 URL 可以让你在 Django 的其他地方明确地引用它， 尤其是在模板中

# path路由匹配
urlpatterns = [
    path('admin/', admin.site.urls),
    path('books/<int:book_id>', views.book_info_post)
    path('books/<str:book_id>', views.book_info_post)
]

url反转

在代码中需要使用类似于url, 可通过reverse函数可获取当前路由的url

reverse(viewname, urlconf=None, args=None, kwargs=None, current_app=None)

  • viewname 是一个 URL 模式名称
  • kwargs 设置动态路由携带的参数
# 应用模块 views.py 视图文件
from django.urls import reverse;

# 不携带任何参数的路由地址 与  匹配?携带数据一致
reverse("ceshi_ce")

# 匹配动态路由反转成url地址
reverse("ceshi_ce", kwargs={"ceshi_id": ceshi_id})

# 如应用模块在 urls.py 存在命名空间 
app_name = "ceshi"
reverse("ceshi:ceshi_ce", kwargs={"ceshi_id":ceshi_id}) # 请携带命名空间的值

ORM模型

通过 models.py文件，用于定义数据库表对应的模型类 , 这些模型类继承自django.db.models.Model

class Book(models.Model):
    title = models.CharField(max_length=100)  # 书籍标题，字符类型字段，最大长度100
    author = models.CharField(max_length=50)  # 作者，字符类型字段，最大长度50
    publication_date = models.DateField()  # 出版日期，日期类型字段
    # 价格，十进制小数类型，总位数5位，小数点后2位
    price = models.DecimalField(max_digits=5, decimal_places=2)
  1. 在settings.py中，配置好 DATABASES，做好数据库相关的配置，
  2. 在 app 中的 models.py中定义好模型，这个模型必须继承自 django.db.models
  3. 将这个app添加到settings.py的INSTALLED_APP 中
  4. 在命令行终端，然后执行命令 python manage.py makemigrations 来生成迁移脚本文件
  5. 同样在命令行中，执行命令 python manage.py migrate 来将迁移脚本文件映射到数据库中

数据类型

  • CharField: 用于存储较短的字符串，在 MySQL 中通常会映射为 VARCHAR 类型。需要指定 max_length 参数来限定字符串的最大长度
    • name = models.CharField(max_length=50)
  • IntegerField: 对应 MySQL 中的 INT 类型，用于存储整数
    • quantity = models.IntegerField()
  • FloatField: 对应 MySQL 中的 FLOAT 类型，用于存储浮点数
    • value = models.FloatField()
  • DecimalField: 用于存储十进制小数，在 MySQL 中映射为 DECIMAL 类型。需要指定 max_digits（数字总位数）和 decimal_places（小数点后的位数）两个参数
    • models.DecimalField(max_digits=8, decimal_places=2) # 在MySQL中对应DECIMAL(8, 2)
  • DateField: 用于存储日期（年、月、日），在 MySQL 中映射为 DATE 类型
    • start_date = models.DateField()
  • DateTimeField: 存储日期和时间（年、月、日、时、分、秒），在 MySQL 中映射为 DATETIME 类型
    • models.DateTimeField(auto_now_add=True)
    • 设置 auto_now_add 为 True 可以让字段在对象首次创建时自动设置为当前时间
    • 设置 auto_now 为 True 则每次对象保存时都会更新该时间字段
  • BooleanField: 在 MySQL 中一般映射为 TINYINT 类型（值通常为 0 表示 False，1 表示 True），用于表示布尔值
    • is_completed = models.BooleanField()
  • ForeignKey: 用于建立与其他模型之间的关联关系，在 MySQL 中会创建对应的外键约束
    • article = models.ForeignKey(Article, on_delete=models.CASCADE)
