Hook注入

Hook注入: 它允许你在程序执行过程中"钩住"某些关键点，插入自定义的代码逻辑。在爬虫中，这通常用于修改或监控网页的行为(查询对应的关键字,比如:加密,解密代码位置---- 用来定位和分析加密/解密代码位置的重要技术)

  • 通过js代码注入控制台 (网页涮新之后就会失效)
  • 通过第三方插件, 每次都会触发(如果没有关闭很容易出现bug)

注入操作

  1. 打开控制台 => 点击sources(源代码)
  2. 在右上的点击Snippets(代码片段) => 再点击new snippets => 生成代码片段文件
  3. 在代码片段中写入对应的js文件(比如: headershook)
  4. 设置完对应代码之后, ctrl + enter执行

注入代码不同网站有不同的写法

爬虫中Hook注入的常见用途

headershook 处理前端请求时携带的请求头信息(注意: 请求头参数是小写)

//  用于拦截和修改 XMLHttpRequest的请求头设置行为
(function() {
    // 1. 保存原始的 setRequestHeader 方法
    var _setRequestsHeaders = window.XMLHttpRequest.prototype.setRequestHeader;
    // 2. 重写 setRequestHeader 方法
    window.XMLHttpRequest.prototype.setRequestHeader = function (key, value) {
        // 3. 当请求尝试设置名为 "x"的请求头时，会触发 debugger语句，让浏览器暂停执行（方便开发者调试）
        if (key == "x") {
            debugger; // 触发调试器断点
        }
        // 4. 调用原始的 setRequestHeader 方法
        return _setRequestsHeaders(this, arguments); // 注意：这里拼写错误
    }
})();

JSVMP

JSVMP 的全称是 JavaScript Virtual Machine Protection，即 JavaScript 虚拟机保护, 即使混淆的意思

它是一种高级的JavaScript代码混淆和加密技术, 将原本简单明了的JavaScript代码逻辑，转换成一套自定义的、类似虚拟机字节码的指令集和一套用来解释执行这些指令的虚拟机（解释器）

你可以把它想象成：

  • 原本的JS代码：就像是一本用中文（JavaScript）写的小说，谁都能直接读懂。
  • 经过JSVMP保护的代码：就像把这本小说翻译成了一套只有特定机器才能懂的密码（自定义指令集），并且把解密和阅读这套密码的机器（虚拟机）也一并给了你。你想知道小说内容，就必须用这台特定的机器去运行这些密码。

JSVMP是混淆的核心代码

比如eval混淆,AA,OO混淆,Fuckjs都是属于jsvmp. eval混淆和 AA/OO混淆是业内对 JSVMP技术输出结果

JSVMP 的主要目的

它的出现几乎只有一个目的：对抗爬虫和逆向分析。

混淆

理解混淆的代码: 混淆其实就是通过对应的方式将代码进行加密处理

一般混淆的代码为: 变量的值和名, 代码的执行流程

// 非常基础的混淆代码演示
var bufan = "123"
var haha = "456"
// 取值
window["bufan"]
window["haha"]

// 进行加密处理 通过btoa将字符转化base64字符  btoa("bufan") === 'YnVmYW4='
arr = ['YnVmYW4=','aGFoYQ==']
window[atob(arr[0])]  // 通过atob(base64格式数据) 还原

// 变量名加密完整版

function xxx(x){
	return atob(x)
}

window[xxx(xx[1])]

eval混淆

eval混淆是一种JavaScript代码保护技术，它通过eval()函数动态执行代码来增加逆向分析的难度

eval混淆的核心是使用eval()函数执行经过编码或加密的代码字符串=> 执行并解析字符串

字符串编码/加密：

  • Base64编码：eval(atob("YWxlcnQoJ0hlbGxvJyk="))
  • 十六进制编码：eval("\x61\x6c\x65\x72\x74\x28\x27\x48\x65\x6c\x6c\x6f\x27\x29")
    • 每个字符转换为 ASCII 码十六进制
  • Unicode转义：eval("\u0061\u006c\u0065\u0072\u0074\u0028\u0027\u0048\u0065\u006c\u006c\u006f\u0027\u0029")
// btoa函数将代码转化正base64
btoa('alert("hello")') // 'YWxlcnQoImhlbGxvIik='

eval(atob('YWxlcnQoImhlbGxvIik='))

AA和OO混淆

AA混淆和OO混淆是JavaScript代码保护中常用的两种混淆技术，它们都通过重命名变量和函数来增加代码阅读难度，但在实现方式和混淆效果上有所不同,

  1. 标识符替换：将所有有意义的变量/函数名替换为随机生成的短名称
  2. 字符串加密：将字符串存储在数组中并通过索引引用
  3. 控制流混淆：打破代码的自然结构，增加跳转逻辑
  4. 插入僵尸代码：添加永远不会执行的无意义代码块
基本概念对比
特性AA混淆OO混淆
命名风格使用"AA"、"BB"、"CC"等连续字母命名使用"OO"、"XX"、"YY"等重复字母命名
复杂度相对简单通常更复杂，常结合其他混淆技术
起源早期混淆工具常用后期发展出的更高级形式
典型特征变量名呈字母递增序列变量名多为重复字母组合

    _0x5e6f = _0x5e6f - 0x0;
    var _0x9i0j = _0x1a2b[_0x5e6f];
    return _0x9i0j;

console[_0x3c4d('0x1')](_0x3c4d('0x0') + ', ' + _0x3c4d('0x2'));

console[BB('1')](BB('0') + ', ' + BB('2'));

JJ混淆

J编码（JJEncode）是一种JavaScript混淆技术，它通过将代码转换成仅使用特定字符（如$、+、!等）的形式来绕过检测

  • 字符替换：将原始JavaScript代码中的字符转换为由$、+、!、(、)等少量字符组成的表达式。
  • 自执行函数：混淆后的代码通常是一个自执行函数，通过构造字符并执行来还原原始代码。
  • 依赖JavaScript类型转换：利用JS的弱类型特性，通过字符拼接和类型转换生成可执行的代码

FuckJs混淆

JSFuck 是一种极端的 JavaScript 混淆技术，它仅使用 6 个字符 [, ], (, ), !, +来编写完整的 JavaScript 代码。它的核心思想是 利用 JavaScript 的类型转换和字符串索引，动态构造出所有需要的字符，最终拼装成可执行的代码

将人类可读的代码变得完全不可读，但功能保持不变，以此对抗分析、调试和逆向工程

特殊变种：JSFuck

  • 极端技术：仅使用 [, ], (, ), !, +六个字符来编写任何 JS 代码。
  • 原理：利用 JS 类型转换，从 "false", "true", "undefined"等字符串中索引字符，拼出完整代码。
  • 特点：代码长度暴增，但肉眼完全无法理解。

补环境

“补环境” 的完整说法是 “补充浏览器环境” 或 “伪造浏览器环境”。

它的核心思想是：在非浏览器环境（如Python的js2py、execjs，或Node.js）中执行JavaScript代码时，通过手动模拟实现浏览器提供的各种原生对象和方法，从而欺骗目标JS代码，让其误以为自己正在真实的浏览器中运行，从而顺利执行并得出正确结果。

  • 简单的网址可以通过扣代码完成 复杂的网址通过伪装环境会简单很多

如何知道网站检测什么?

  • 通过调试, 异常捕获, 本地环境运行
// 基础浏览器环境模拟
if (typeof window === 'undefined') {
    global.window = {
        document: {},
        navigator: {},
        location: {},
        screen: {},
        history: {},
        // 更多浏览器属性...
    };
    global.document = window.document;
    global.navigator = window.navigator;
}

当我们辛苦将浏览器环境的加密或者解密入口找到，把加密或者解密的JS的代码拷贝到本地，由node解释器驱动执行的时候，会因为拷贝的JS代码中包括只能由浏览器调用的API，现在被node执行就会报错，为了解决这个问题，我们需要在拷贝的代码环境中模拟补充需要的前端对象，所以我们就非常有必要掌握浏览器接口对象常用的

  • window ==> BOM
  • document ==> DOM
  • element 节点信息
  • location 地址栏信息
  • navigator 浏览器信息
  • history 浏览器历史信息
  • screen 屏幕信息
  • localStorage和sessionStorage：本地存储对象

然后通过proxy代理检查整个加密或解密代码 使用了什么环境,后进行补充
