进程与线程的区别

进程概述

进程: 是操作系统中最核心的概念之一，可以理解为 “正在运行的程序”。但它不仅仅是静态的代码，而是程序在运行时的动态实体，包含了程序执行所需的所有资源和状态

对于操作系统来说，一个任务就是一个进程

  • 打开一个记事本就启动了一个记事本进程
  • 打开两个记事本就启动了两个记事本进程
  • 一个Word就启动了一个Word进

你同时打开两个软件：浏览器 和 音乐播放器

  • 浏览器是一个 进程，它有自己的内存、代码、数据、打开的文件（如网页缓存）。
  • 音乐播放器是另一个 进程，它也有独立的内存、音频解码器、播放列表等。

这两个进程互不干扰，即使浏览器崩溃了，音乐播放器仍然能正常运行。这就是进程的 独立性

总结

  1. 进程是程序的一次执行实例，拥有独立的地址空间和系统资源（如内存、文件句柄等）
  2. 不同进程之间相互隔离，一个进程崩溃通常不会影响其他进程。
  3. 操作系统以进程为单位分配资源（CPU、内存等），进程是资源分配的基本单位
    • 进程是资源单位 (几个进程就享受几个进程的资源)

线程概述

线程是操作系统能够进行运算调度的最小单位，它被包含在进程之中,是进程中的实际执行单位, 每个进程最少有一个线程

比如: 音乐播放器进程, 我可以一边听歌一边下载歌曲, 如果只有一个线程那你就不能同时操作,必须等待另一个完成才能执行

一个进程默认有 1个主线程（如 main()函数）

可以创建 多个线程，每个线程执行不同的函数（如一个线程处理网络请求，另一个线程更新数据）

通过 multiprocessing 模块实现多进程编程

这样很难直接的看出来, 最好是进行请求网页内容,进行处理, 以下内容会同时请求4个页面的数据

  • 同步和异步 apply/apply_async
    • 每次循环回将空闲掉了的子进程去池子外获取新的任务执行
    • 所以异步体现了并发：4个进程都在执行任务，其中一个执行完毕后，下一个会继续补上
from multiprocessing import Pool  # 导入进程池模块
import time

def dwon(i):
    print(f"开始执行{i}")
    time.sleep(1)
    print(f"结束{i}")
    return f"返回值{i}"

# 使用apply同步是会阻塞,需要先执行上一个才会执行下一个,一个一个的执行
if __name__ == '__main__':
    # 创建包含4个工作进程的进程池,由四个进程来处理对应的任务, 每个进程处理完成之后才会处理下一个
    po = Pool(4)
    # 向进程池提交4个任务
    po.apply_async(dwon, args=(1,))  # 异步提交任务
    po.apply_async(dwon, args=(2,))  # 异步提交任务
    po.apply_async(dwon, args=(3,))  # 异步提交任务
    po.apply_async(dwon, args=(4,))  # 异步提交任务
    # 先用四个,在设置六个,讲清楚进程Pool(4)的设置
    po.apply_async(dwon, args=(5,))  # 异步提交任务
    po.apply_async(dwon, args=(6,))  # 异步提交任务
    print("=========start-===========")
    po.close()  # 关闭进程池（禁止继续提交新任务）
    po.join()  # 等待所有子进程执行完毕
    

# apply_async 异步执行
from multiprocessing import Pool
import time

def task(name):
    print(f"任务 {name} 开始")
    time.sleep(2)
    print(f"任务 {name} 结束")
    return f"结果-{name}"

if __name__ == '__main__':
    # with设置自动调用 close() 和 join()
    with Pool(2) as pool:
        # 异步提交两个任务（主进程不阻塞）
        result1 = pool.apply_async(task, args=("A",))
        result2 = pool.apply_async(task, args=("B",))
        print("主进程继续执行其他操作...")
        # 获取结果（此时会阻塞）
        print(f"任务A结果: {result1.get()}")
        print(f"任务B结果: {result2.get()}")

进程与线程使用场景

如果程序想利用计算机的多核优势, 让CPU同时处理一些任务,适合用多进程开发

如果程序不利用计算机的多核优势, 适合用多线程开发

进程适用于: CPU 密集型（如计算、图像处理 , 视频编码 )

线程适用于: I/O 密集型（如网络请求、文件读写）

特性线程（Thread）进程（Process）
资源占用共享进程内存，创建/切换开销小独立内存空间，创建/切换开销大
并行能力受 GIL 限制（Python 中无法真正并行）可跨 CPU 核心并行
适用场景I/O 密集型（网络请求、文件读写）CPU 密集型（计算、数据处理）
通信方式共享内存（需同步机制）IPC（队列、管道、共享内存）
崩溃影响线程崩溃可能导致整个进程崩溃进程崩溃不影响其他进程

协程概述

在 Python 中，协程（Coroutine） 是一种轻量级的并发编程模型，用于高效处理 I/O 密集型任务（如网络请求、文件读写）。它通过 协作式多任务（非抢占式）实现高并发，避免了线程/进程的切换开销

  • 单线程内运行：所有协程共享同一个线程，由事件循环（Event Loop）调度
  • 非阻塞异步：遇到 I/O 操作时主动让出控制权，不阻塞线程。
  • 低开销：协程切换在用户态完成，无需操作系统介入。

协程基础

Python 通过 asyncio库原生支持协程，关键语法

  • coroutine: 协程对象在被创建后不会自动执行, 必须显式地 await它，或者通过事件循环（Event Loop）来调度运行
  • asyncio.run()： 是 Python 3.7+ 推荐的运行协程的方式
    1. 创建一个新的事件循环
      • 相当于一个无限循环,可以把一些函数注册到这个事件循环上,当满足某些条件时,函数就会执行
    2. 运行传入的协程函数直到完成
    3. 关闭事件循环
import asyncio
# 通过async定义一个协程函数（异步函数）
async def get_data(url):
    print('请求：',url)

# 调用协程函数，创建协程对象（此时未执行）
c = get_data('baidu.com')
print(c,"携程对象")  # 输出：<coroutine object get_data at 0x...> 携程对象

# 运行协程
asyncio.run(c)

#---------------------------
async def get_data(url):
    print('请求：',url)
async def data_run():
    await get_data("你好")
    print('请求')
asyncio.run(data_run())

在 Python 3.10+ 中，如果主线程没有显式创建事件循环，get_event_loop()会直接报错

Python 3.7-3.9：自动创建并返回一个新循环（但此行为已被弃用）

  • get_event_loop()：获取当前线程的默认事件循环（如果没有，可能会报错）。
  • new_event_loop()：创建一个新的循环，不管当前是否有默认循环。
    • python 3.10+ 中需要这样设置
import asyncio

#----------------------------------3.7-3.9
# 创建执行协程对象 => 它像一个任务调度员，负责协调和管理多个异步任务的执行
loop = asyncio.get_event_loop()  # ✅ 不传参数
# 执行对应协程函数:  作用是运行一个协程（coroutine）直到其完成
loop.run_until_complete(c)
#------------------------------------
# 推荐使用这种
loop = asyncio.new_event_loop() # 创建执行协程对象 => 创建新的事件循环
asyncio.set_event_loop(loop)  # 设置为当前线程的事件循环
loop.run_until_complete(c) # 执行协程

task并发

import asyncio

# 1. 定义协程函数
async def get_data(url):
    print('请求：', url)  # 无实际异步操作

# 2. 创建协程对象（此时未执行）
c = get_data('baidu.com')

# 3. 创建新事件循环并设置为当前循环
loop = asyncio.new_event_loop()
asyncio.set_event_loop(loop)

# 4. 将协程包装为 Task
task = loop.create_task(c)
print("执行前：", task)  # 输出: <Task pending>

# 5. 运行事件循环直到任务完成
loop.run_until_complete(task)
print("执行后：", task)  # 输出: <Task finished>

future并发

ensure_future()：将协程转为可调度的 Task

import asyncio

# 1. 定义一个协程函数
async def get_data(url):
    print('请求：', url)

# 2. 创建协程对象（此时未执行）
c = get_data('baidu.com')

# 3. 创建新事件循环并设置为当前循环
loop = asyncio.new_event_loop()
asyncio.set_event_loop(loop) 

# 4. 将协程包装为 Future/Task（确保可调度）
future = asyncio.ensure_future(c)
print("执行前：", future)  # 输出任务状态（未执行）

# 5. 运行事件循环直到任务完成
loop.run_until_complete(future)
print("执行后：", future)  # 输出任务状态（已完成）

练习

wait()函数：

  • 给任务列表中的每一个任务对象赋予一个可被挂起的权限！当cpu执行的任务对象遇到阻塞操作的时候，当前任务对象就会被挂起，则cup就可以执行其他任务对象，提高整体程序运行的效率！
  • 挂起任务对象：让当前正在被执行的任务对象交出cpu的使用权，cup就可以被其他任务对象抢占和使用，从而可以执行其他任务对象。
  • 注意：特殊函数内部，不可以出现不支持异步模块的代码，否则会中断整个异步
import asyncio
import time

async def fun1(index):
    print("你好")
    # time.seep(1) 不能设置同步, 设置
    await asyncio.sleep(3)
    print(index)
    pass

async def fun2(index):
    print("我好")
    await asyncio.sleep(3)
    print(index)
    pass

async def fun3(index):
    print("大家好")
    await asyncio.sleep(3)
    print(index)
    pass

loop = asyncio.new_event_loop()
asyncio.set_event_loop(loop)

if __name__ == "__main__":
    f1 = asyncio.ensure_future(fun1(1))
    f2 =  asyncio.ensure_future(fun2(2))
    f3 = asyncio.ensure_future(fun3(3))
    loop.run_until_complete(asyncio.wait([f1,f2,f3]))

日志库

from loguru import logger : pip install loguru

  • logger.add(sink,format,rotation,retention,level="INFO")
    • sink : 存储日志文件格式与地址 (格式为.log后缀)
    • level: 存储对应以上日志的级别
      • TRACE 5 DEBUG 10 INFO 20 SUCCESS 25
      • WARNING 30 ERROR 40 CRITICAL 50
    • format: 设置日志格式
      • format="{time:} | {level} | {message}"
      • format="{time:HH:mm:ss } | {level} | {message}" 设置时分秒
    • rotation: 当文件大小到达指定大小时, 会自动创建新文件继续记录日志
      • 默认会在原文件名后追加时间戳（如 file.2023-08-20_12-00-00.log）
    • retention: 自动清理超过指定时间的旧日志文件
      • retention="7 days"
from loguru import logger
