Elasticsearch数据库介绍

  • Elasticsearch是一个开源的分布式、RESTful风格搜索引擎和分析引擎，基于 Apache Lucene 构建 , 它能够处理大规模请求实时数据、分析和可视化，以其高性能、可扩展性和易用性著称 , 简称ES

  • 在爬虫项目中使用Elasticsearch，主要是利用其强大的数据索引和搜索能力，从而提升爬虫数据的管理和检索效率

搜索引擎是什么

就是在任何场景下，找寻你想要的信息，用户输入想要搜索的关键词，然后期望找到这个关键词相关的信息,

主要解决问题:

高效全文搜索

传统数据库问题：MySQL等关系型数据库的LIKE查询效率极低，无法处理大规模文本搜索

Mysql当没有可用索引或查询条件无法利用索引时,MySQL必须进行全表扫描

  1. 需要进行全文扫描
  2. 逐行读取数据库表中的每一行记录, 每一行都要进行扫描,检索里面有没有 “python” 这些字,并且开始一一比对文本值，如果指定的字段的文本过于长，扫描比对的时间就长，无法实现高效率

mysql主要是用于做数据存储和管理 , 无法做到高效的全文搜索, 速度和效率太低

全文检索是什么

全文检索是一种让计算机在海量文本数据中，像人类一样理解语义并快速找到相关信息的技术

全文检索的核心在于倒排索引(拆分词条)，实现毫秒级全文检索，支持中文分词、同义词、模糊匹配等, ES支持这些操作

  • 传统搜索的问题（如MySQL的LIKE) 像大海捞针，必须逐行扫描
    • 翻开每本书，每页每行找“%独行%”这个字 → 极慢！

倒排索引就像一本书最后的“索引”页。它记录了每个词条出现在哪些文档中，以及出现的位置和频率

ES全文检索

以上过程就是ES的全文检索，如果数据有100万条，倒排索引中未必扫描100万次，就算倒排索引扫描100万次，跟数据库相对，ES性能更高，因为数据库中要匹配是否包含关键词，ES的倒排索引只要比对词条是否一样

如何拆分词条,倒排索引不需要设置, 由es自动规划,

# 文档是es的称呼 与 mysql数据的一行是对应的
文档1 (id=1): "Python爬虫教程：快速抓取网页数据"
文档2 (id=2): "Elasticsearch全文检索实战指南"
文档3 (id=3): "Python与Elasticsearch构建搜索引擎"

Lucene是什么

Lucene是一个开源的信息检索工具包(java API)，它包含了很多极优算法，索引结构、读写索引工具、排序索引等功能，而Elasticsearch底层是基于这些包，对其进行了扩展，提供了比Lucene更为丰富简洁的查询语言，可以非常方便的通过Elasticsearch的HTTP接口与底层Lucene进行逻辑交互。

一句话概括: Elasticsearch是Lucene面向企业搜索应用的扩展，极大的缩短研发周期，因为Elasticsearch是在它基础上扩展的应用程序,提供了简单易用的restful接口，可以直接拿来使用

Elasticsearch的基本结构

Elasticsearch是基于Lucene的全文检索库，本质也是存储数据，很多概念与我们的MySQL类似

和mysql对比关系

索引（indices）---------------- Table 数据表

映射（Mapping）---------------- 对应表结构（Schema）

字段（Field）---------------- Columns 列 （字段名）

文档（Document）---------------- Row 行（数据记录）

类型（type） 存储的类型

详细说明

  • 索引(indices)：每个索引 （即数据库）的名字必须小写
    • 不能包含 /, \, *, ?, ", <, >, |, ``（空格）， ,， `#
  • 类型（type）： 设置对应类型
  • 文档（document）：存入索引库的原始数据。比如每一条商品信息，就是一个文档
  • 字段（field）：文档中的属性(字段)

语法

创建索引(数据表)的请求格式：

  • 请求方式：PUT
  • 请求路径：/索引库名
  • 请求参数：json格式：

创建用户索引

PUT /hotel 创建用户索引（相当于创建 hotel 库)

# 创建用户索引（相当于 users 表）
PUT /users  # users是索引名 必须小写
{  # mappings  这里相当于表结构定义
  "mappings": {                     
    "properties": {            # properties定义字段（列）
       # id,username 是字段  存储的数据是文档
      "id": {"type": "integer"},	
      "username": {"type": "keyword"},
      "email": {"type": "keyword"},
      "age": {"type": "integer"},
      "created_at": {"type": "date"}
      "msg": {"type": "text"}
    },
  }
}
# type指定字段类型  keyword不会被分词

type类型

  • text 文本类型：会被分词器拆分，建立倒排索引，支持全文搜索
  • keyword 关键词类型：不分词，整体作为单个词项，用于精确匹配和聚合
  • integer 整数类型：存储整数值，支持范围查询和数值计算
  • float 浮点类型：存储单精度浮点数，用于小数计算
  • long 长整型：存储大整数值，如时间戳和大数量统计
  • double 双精度类型：存储双精度浮点数，用于高精度小数计算
  • date 日期类型：存储日期和时间，支持多种格式和时间范围查询
  • boolean 布尔类型：存储 true/false 值，用于开关状态
  • object 对象类型：存储JSON对象，支持嵌套结构
  • nested 嵌套类型：存储对象数组，保持数组元素的独立性
  • geo_point 地理位置类型：存储经纬度坐标，支持地理距离查询
  • ip IP地址类型：存储IPv4和IPv6地址，支持IP范围查询
  • completion 自动补全类型：优化前缀搜索，用于搜索建议功能

{
  "mappings": {
    "properties": {
      "title": {
        "type": "text",           // 文本类型，会被分词
        "fields": {               // 多字段特性，允许同一个字段被索引成不同的方式
          "keyword": {            // 子字段，使用keyword类型，用于精确匹配和排序
            "type": "keyword"
          }
        }
      },
      "description": {
        "type": "text"
      },
      "price": {
        "type": "float"           // 浮点数类型
      },
      "quantity": {
        "type": "integer"         // 整数类型
      },
      "created_at": {
        "type": "date",           // 日期类型
        "format": "yyyy-MM-dd HH:mm:ss||yyyy-MM-dd||epoch_millis" // 可以接受多种日期格式
      },
      "is_available": {
        "type": "boolean"         // 布尔类型
      },
      "tags": {
        "type": "keyword"         // 关键字数组，每个标签都是keyword类型
      },
      "author": {                 // 对象类型
        "properties": {
          "name": {
            "type": "text"
          },
          "email": {
            "type": "keyword"
          }
        }
      },
      "comments": {               // 嵌套类型，用于对象数组，每个对象独立索引
        "type": "nested",
        "properties": {
          "user": {
            "type": "keyword"
          },
          "content": {
            "type": "text"
          }
        }
      },

      }
    }
  }

Python API实践操作

验证是否连接成功

from elasticsearch import Elasticsearch
# 连接到本地ES
es = Elasticsearch("http://localhost:9200")

# 测试连接
if es.ping():
    print("✅ 连接成功")
else:
    print("❌ 连接失败")

创建索引

  • es.indices.create (index,body,ignore)
    • index (必需): 要创建的索引的名称。
    • body (可选): 索引的配置，包括设置和映射
      • 不设置: 使用默认, 根据后续插入第一个文档数据，Elasticsearch 会根据文档的字段自动推断字段类型，并创建映射
    • ignore (可选): 要忽略的HTTP状态码，例如设置ignore=400可以忽略索引已存在的错误。
      • z注意使用可能报错: 已更改为options(ignore_status=400)
# 创建索引
py_es = es.indices.create(index="py_es",ignore=400)
print(py_es)

# 或者通过以下方式创建索引
simple_mapping = {
    "mappings": {
        "properties": {
             # 文本字段，可全文搜索
            "title": {"type": "text"},
            "content": {"type": "text"},  
             # 关键字字段，精确匹配
            "author": {"type": "keyword"}, 
            "create_time": {"type": "date"},# 日期字段
            "views": {"type": "integer"}    # 数字字段
        }
    }
}
response = es.indices.create(
    index="simple_index",
    body=simple_mapping,
).options(ignore_status=400)

添加数据

  • es.index(index,id,body)
    • index（必需） 作用：指定文档要存储的索引名称
    • id（可选） 指定文档的唯一ID
      • 如果不提供：Elasticsearch 会自动生成一个唯一ID
    • body（必需） 要存储的文档内容，必须是字典格式
      • body={"name": "张三", "age": 25}
# 添加数据

查询数据

  • es.search(index="py_index01",body =query )
    • query 字典查询参数
      • match_all : 查询所有文档数据
      • match: 基础匹配查询（会分词）
      • term: 精确查询字段值
      • terms: 查询 指定多个字段值查询
      • Wildcard : 模糊匹配
        • * 匹配任意字符
        • ? 匹配单个字符
      • Regexp 正则匹配
# match_all  查询所有文档数据 
query={ 
    "query":{  
        "match_all":{}  
    } 
}
r = es.search(index="py_index01",body= query)

# 基础匹配查询（会分词）    # title字段包含这些词
query={"query":{"match":{"title":"Elasticsearch 教程" }}}

# 在多个字段中搜索: 以下是在msg和username字段中查询
query = {
    "query": {
        "multi_match": {
            "query": "看电影",
            "fields": ["msg", "username"]  
        }
    }
}

# term主要用于精确匹配哪些值
query={  "query":{  "term":{   "age":20 }  } }
r = es.search(index="py_index01",body= query)
print(r)

# 根据terms查询 指定多个字段值查询
query={ "query":{ "terms":{ "age":[20,18] } } }
r = es.search(index="py_index01",body= query)
print(r)

# wildcard模糊匹配
query = { "query": { "wildcard": {  "username": "小*" }}}

# 正则匹配
query = { "query": { "regexp": {  "username": "张.*明" }}}

IK分词器

IK分词是什么？

IK分词器是ES的一个插件，主要用于把我们要查询的数据拆分成一个个关键字，我们在搜索时，ElasticSearch会把数据进行分词，然后做匹配。默认的中文分词器会把每一个中文拆分，比如“李四在吃饭”，会拆分成“李”，“四”，“在”，“吃”，“饭”, 显然，这并不符合我们的要求，会影响我们最终的搜索结果，所以ik分词器（中文分词器）能解决这个问题。

核心特点

两种模式：ik_smart（细粒度切分）和 ik_max_word（最细粒度切分）

在可视化界面操作查看效果

GET _analyze
{
  "analyzer": "ik_smart", # 选择IK智能分词
  "text": "张三在吃饭， 李四在睡觉"
}

在es中使用分词器

{
  "mappings": {
    "properties": {
      "content": {
        "type": "text",
        "analyzer": "ik_max_word",    # 索引时的分词器
        "search_analyzer": "ik_smart"  # 搜索时的分词器
      }
    }
  }
}
