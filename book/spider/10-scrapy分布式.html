分布式爬虫

分布式是什么

简单的说 分布式就是不同的节点（服务器，ip不同）共同完成一个任务

  • 加快项目的运行速度，但是需要的资源(硬件&网络)依然还是原有的
  • 单个节点的不稳定性不影响整个系统的稳定性

与进程,线程类似, 就是多个人共同做一件事情

  • 比如: 三个进程分别负责 1-3 4-6 7-9的页面
    • 某一个进程崩溃了, 其他的不会有影响,但是这个进程的数据会停止
  • 但是分布式的话,其他的节点会帮你处理

scrapy-redis

scrapy是一个框架，他本身是不支持分布式的，但可以借助一个叫scrapy-redis的组件，这个组件正是利用了redis可以分布式的功能，集成到Scrapy中，使得爬虫可以进行分布式

  • scrapy-redis是一套基于 redis 数据库、运行在 scrapy 框架之上的组件
  • scrapy_redis这个组件可以帮助我们的的scrapy和redis数据库做交互
    • Scrapy_redis在scrapy的基础上实现了更多，更强大的功能，具体体现在：
    • 通过持久化请求队列和请求的指纹集合来实现：
      • 断点续爬
      • 分布式快速抓取

  • 断点续爬, 当某个爬虫出现问题时,其他爬虫任务可以帮助处理
  • 三个不同的爬虫数据怎么处理, 需要有一个共同的数据地址
    • 比如: 数据库

通过老板:

让三个员工合作去爬取这个网站的数据, 三个员工三台电脑, 进行讲解

讲解2

  • 每个爬虫程序会将请求地址放到集合中进行去重,
  • 集合的请求地址
    • 没有被请求过将会移动到任务队列中
    • 如果已经被请求过的url请求地址, 不会放置到任务队列中
  • 后续爬虫程序中的调用器从任务队列中获取对应的请求地址进行请求

redis_key： 起始请求地址，通过这个起始请求地址去启动整个分布式项目

总结

  1. 爬虫运行后，每一个分布式爬虫会去redis数据库中拿数据，该数据为键值对形式，抢到redis_key, 那个爬虫程序抢到redis_key就开始执行
  2. 其他的爬虫程序会转移阵地去任务队列里面抢任务，
    • 抢到redis_key的爬虫将,翻页的请求,详情的请求,等等其他请求都会通过调度器放置到集合,再从集合移动到任务队列
  3. 当任务队列中存在请求时,其他爬虫程序就开始工作执行,
  4. 各个爬虫程序的调度器将得到的请求对象放入到有序集合中去重，能够放进去，说明之前未被执行过，则将这个请求对象顺利的放入任务队列中
  5. 最终通过scrapy的各个模块的工作实现数据的抓取
  • 通过item管道将数据存储到数据队列中(其实就是redis数据库中)

所以是上图中：

  • redis_key： 起始任务，本质是通过键名所存储的一个字符串，我们通过这个字符串去启动整个分布式项目（爬虫运行起来的第一步）
  • 集合：集合的特性是不存储重复数据，主要作用于去重
  • 任务队列： 是存放公共的待请求对象，所以的单个节点爬虫都去任务队列中提取任务
  • 数据队列： 用于存放所以爬虫节点清洗出来的数据

补充

  • 当3个爬虫节点同时运行时，哪个爬虫的下载器闲着，闲着的下载器就通过调度器去任务队列主动获取请求对象-
  • 注意:下载器绝对不会出现抢到重复的请求对象的现象，原因是因为任务队列中的待请求对象是 pop操作(移除操作)

具体流程图

  • 在scrapy redis中，所有的待抓取的请求对象和去重的指纹都存在一个公用的redis(数据库)中
  • 所有的服务器(爬虫项目)公用一个目录中的请求对象
  • 所有的请求对象存入redis前，都会经过同一个redis集合去进行判断，判断之前是否已经存入过
  • 在默认情况下所有的数据也会保存在redis中

scrapy去重

对返回的数据进行去重, 封装一个文件

  • r.sadd('集合名称', '成员') 添加元素
  • r.sadd('tags', 'python', 'redis', 'database')
  • 获取 Set 所有成员（smembers）
    • all_tags = r.smembers('tags')
  • 检查成员是否存在（sismember）
    • is_python = r.sismember('tags', 'python')
# 封装的文件
import redis
import hashlib

class Filter():
    # 将目标数据处理为密文指
    def get_md5(self,val):
        md5 = hashlib.md5()          # 创建 MD5 哈希对象
        md5.update(val.encode('utf-8'))  # 将字符串编码为 UTF-8 字节流
        return md5.hexdigest()        # 返回 32 位十六进制哈希值


    # 将密文存入到redis 集合
    def add_data(self,url):
        red = redis.Redis(host='127.0.0.1', port=6379, db=1)  # 连接Redis
        res = red.sadd('book:set_data', self.get_md5(url))    # 向Set添加URL的MD5值
        if res == 0:  # 添加失败（已存在）
            return False
        else:         # 添加成功（新URL）
            return True


    # 判断数据是否存在集合
    def is_data(self,url):
        red = redis.Redis(host='127.0.0.1', port=6379, db=1)
        # sismember判断内容是否存在 ，以避免重复爬取
        res = red.sismember('book:set_data',self.get_md5(url))
        # res输出: 1（存在）或 0（不存在）
        return res
