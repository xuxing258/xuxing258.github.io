  • 逆向是对加密的数据, 加密逻辑进行分析

加盐校验

比如: md5 sha1 sha256加密后无法还原

md5

md5无法解密, 存在16位, 32位, 40位 (生成的是十六进制)

  • update: 加密对应数据, 数据需要为字节, 通过字符串.encode("utf-8")
    • 设置字节是为了应对各种环境进行md5计算
  • hexdigest(): 输出32位十六进制哈希值
  • digest: 输出bytes原始字节形式的哈希(二进制)
# 通过123456进行加密来判断是否为md5加密
# 加密值为 e10adc3949ba59abbe56e057f20f883e
from hashlib import md5

obj = md5()                			# 创建MD5哈希对象
obj.update("123456".encode("utf-8"))   # 输入需哈希的数据（UTF-8编码）
print(obj.hexdigest())      		# e10adc3949ba59abbe56e057f20f883e
print(obj.digest())     			#b'\xe1\n\xdc9I\xbaY\xab\xbeV\xe0W\xf2\x0f\x88>'

base64

Base64 是一种将二进制数据(字节)编码为 ASCII 字符串的编码方式

  • base64组成 A-Z a-z 0-9 - = / 组成
  • base64数据 后面一般是= 或 == 结尾
import base64
# 原始字节数据
data = "Hello World".encode('utf-8')
# Base64 编码
encoded = base64.b64encode(data).decode('utf-8')  # 输出: "SGVsbG8gV29ybGQ="

# 将以上的数据进行还原
result = base64.b64decode(encoded).decode('utf-8')
print(result)  # Hello World

在浏览器中: btoa函数可以将ASCII 或 UTF-8 字符串）编码为 Base64 格式的 ASCII 字符串

//btoa()不能直接处理 Unicode（如中文、emoji），如果字符串包含非 ASCII 字符（如 中文或 😊），需要先进行 ​​UTF-8 编码​​
btoa("123456")  // 输出 'MTIzNDU2'

atob()用于将 Base64 编码的字符串 解码回原始的二进制数据

atob('MTIzNDU2') // "123456"

非对称加密与对称加密

非对称加密

最经典和广泛使用的非对称加密算法。

使用一对密钥：一个公钥和一个私钥

  1. 公钥：公开的，用于加密（锁信箱）或验证签名。
  2. 私钥：绝对私有的，用于解密（开信箱）或创建签名。

对称加密

加密和解密使用同一把密钥

对称加密

AES

AES 是一种对称分组密码，是当今全球使用最广泛、最安全的加密算法之一, 通常会以 十六进制（Hex） 或 Base64 的形式表示

可变密钥长度：AES 支持三种不同的密钥长度，这直接决定了其安全强度：

  • AES-128：密钥长度 = 128 位 → 进行 10 轮处理
  • AES-192：密钥长度 = 192 位 → 进行 12 轮处理
  • AES-256：密钥长度 = 256 位 → 进行 14 轮处理
区分

如果没有混淆时会出现AES对象, 则可判断位AES进行加密与解密

函数: AES.encrypt: 加密 AES.decrypt: 解密

AES.encrypt()方法的参数会根据编程语言和加密库的不同而有所差异

参数类型必填说明
keybytes是密钥，长度必须为 16（AES-128）、24（AES-192）或 32（AES-256）字节。
modeAES.MODE_*是加密模式，如 AES.MODE_CBC、AES.MODE_GCM、AES.MODE_ECB（不推荐）。
ivbytes视模式而定初始化向量（16字节），CBC/GCM 等模式需要，ECB 不需要。
plaintextbytes是明文数据，需先填充为 16字节的倍数（如用 PKCS7填充）。

AES.decrypt

参数类型必填说明
keybytes是必须与加密时使用的密钥相同（16/24/32 字节）。
modeAES.MODE_*是需与加密模式一致（如 AES.MODE_CBC）。
ivbytes视模式而定必须使用加密时的 IV（CBC/GCM 需要 16 字节）。
ciphertextbytes是密文数据（需去除 IV 部分，除非 IV 已拼接在密文中）。
const plaintext = CryptoJS.AES.decrypt(
  ciphertext, 
  key, 
  { iv, mode, padding }
).toString(CryptoJS.enc.Utf8);

DES

通常会以 十六进制（Hex） 或 Base64 的形式表示

DES密钥在形式上是一个 64位（8字节） 的字符串。然而，在这64位中，每第8位（即第8、16、24、32、40、48、56、64位）被用作奇偶校验位，不参与实际的加密运算。因此，真正起作用的、用于加密的密钥只有 56位（7字节)

加密DES.encrypt() 解密 DES.decrypt

  • 明文 (message)： 要加密的是什么？通常是用户名、密码、时间戳或其他请求参数的组合。
  • 密钥 (key)： 密钥是什么？它可能是固定的（写死在JS里），也可能是动态生成的（需要其他JS函数计算）。
  • 加密模式 (mode)： 如 ECB, CBC 等。
  • 填充方式 (pad)： 如 Pkcs7, ZeroPadding 等。

非对称加密

RSA(私钥 公钥): 同一个明文可以生成不同的密文, 通常会以 十六进制（Hex） 或 Base64 的形式表示

  • 通过公钥加密, 在通过私钥进行解密

总结

对称加密与非对称加密: 生成后数据一般是十六进制展示或者 base64形式进行展示,

md5加密是十六进制加密

无线debug

debugger 是JavaScript中定义的一个专门用于断点调试的关键字，只要遇到它，JavaScript的执行便会在此处中断，进入一种调试模式。有了 debugger 这个关键字，该 debugger 会被网站开发者利用，使其成为我们正常调试的拦截手段

案例演示

  • 目标网址：https://antispider8.scrape.center/
  • 进入该网站，开始我们爬虫正常的抓包控制台，你会发现弹出一个自动debug，如下图：

在我们没设置任何断点的情况下，它就直接进入了断点模式。这时候我们可以点击 Resume script execution(恢复脚本执行)按钮，尝试跳过这个断点继续执行，如下图所示：

然而不管我们点击多少次按钮，它仍然一次次地进入断点模式，无限循环下去，我们称这样的情况为无限debugger。那我们如何解决这种干扰手段呢？

解决手段

首先要做的是找到无限 debugger 的源头。在 Sources 面板中可以看到，debugger 关键字出现在一个 JavaScript 文件里，这时点击左下角的格式化按钮，那我们仔细观察该文件，如下图所示：

截图中setInterval方法表示循环，循环执行 debugger 语句。无限 for 循环、无限 while 循环也都能实现改需求，原理都大同小异

解决方法

禁用断点: 刷新后不会进入debugger,但是后续自身也无法使用断点

局部警用

在 debugger 语句所在的行数上单击鼠标右键，此时会出现一个快捷菜单，如下图所示：

操作完后会发现当前行数前多了一个？符号，同时 Breakpoints 也出现了刚才添加的断点位置，这时再次点击 Resume script execution(恢复脚本执行)按钮，就可以发现我们不会再进入无限 debugger 模式了

局部禁用

我们也可以选择另外一个选项 Add conditional breakpoint（添加条件判断）跳过debug，这个模式我们可以设置进入断点的条件,如下图所示： 会弹出一个输入框, 在里面输入false即可跳过

利用fiddler解除无限debug

  1. 找到对应的的debugger文件进行下载
  2. 下载本地后进行删除debug或修改,并保存,
  3. 在通过fiddler进行自动响应

记住: 浏览器一定要清除缓存, 浏览器有些网站会进行自动缓存

修改源代码解决

在Sources中选择Overriders点击 新建, 设置一个桌面的文件夹存储位置

  1. 在对应的js文件中,右键选择Override content
  2. 然后就可以在其中进行修改,删除debugger
