协程概述

协程（Coroutine） 是一种轻量级的并发编程模型，可以在单个线程内实现并发, 协程之间通过yield或await进行切换，而不是线程切换，因此开销更小。

  • 单线程内运行：所有协程共享同一个线程，由事件循环（Event Loop）调度
  • 非阻塞异步：遇到 I/O 操作时主动让出控制权，不阻塞线程。
  • 低开销：协程切换在用户态完成，无需操作系统介入。

进程和线程都是受操作系统管理, 协程是受用户管理

协程基础

pip install asyncio 通过 asyncio库原生支持协程

使用async def定义协程函数

import asyncio
# 通过async定义一个协程函数（异步函数）
async def get_data(url):
    print('请求：',url)
  
# 直接调用
main_xie("https://www.baidu.com") # 报错

# 调用获取协程对象c
c = main_xie("https://www.baidu.com")
print(c,"携程对象")  
# <coroutine object main_xie at 0x00000270417B36B0>

coroutine: 协程对象在被创建后不会自动执行, 需要通过事件循环（Event Loop）来调度运行

执行协程

旧API (Python 3.6及之前)

  • asyncio.get_running_loop() 获取当前事件循环
  • asyncio.ensure_future() 将协程包装成任务,在后台执行并发
  • loop.run_until_complete() 运行协程任务

新API (Python 3.7+)

  • asyncio.get_event_loop() 获取当前事件循环
  • asyncio.create_task() 将协程包装成任务,在后台执行并发
  • asyncio.run() 运行协程任务

python37以前

import asyncio
async def get_data(url):
    print('请求：',url)

c = get_data('baidu.com')
print(c,"携程对象")  

# 创建执行协程对象: 获取事件循环(任务调度中心)来操作协程
loop = asyncio.get_event_loop() 
# 执行对应协程函数:  作用是运行一个协程直到其完成
loop.run_until_complete(c)
  • get_event_loop()：获取当前线程的默认事件循环
  • run_until_complete : 执行指定的协程，直到这个协程完全执行完毕

新python3.7+

import asyncio

async def get_data(url):
    print('请求：',url)

c = get_data('baidu.com')
print(c,)  

# 运行协程
asyncio.run(c)
  • asyncio.run()： 是 Python 3.7+ 推荐的运行协程的方式
    • 用于运行异步程序。它负责管理事件循环，执行传入的协程，并最终关闭事件循环
    • asyncio.run() 只调用一次，是整个异步程序的入口点

并发

await

await 关键字的作用是挂起当前协程，直到等待的对象完成。在挂起期间，事件循环可以运行其他协程。但是，是否切换以及切换到哪个协程，是由事件循环调度的

await 暂停挂起当前协程执行, 在等待期间可以执行其他协程完成任务

await具有等待功能

  • asyncio.gather 同时启动所有协程，并发执行
    • 返回值顺序：结果顺序与输入顺序一致
import asyncio
import time
# 记录当前时间
s = time.time()

async def task1():
    print("Task1 开始")
    # 这行代码会让出控制权2秒钟
    await asyncio.sleep(2)  # 比如网络请求
    print("Task1 完成")

async def task2():
    print("Task2 开始")
    # 这行代码会让出控制权1秒钟
    await asyncio.sleep(1)  
    print("Task2 完成")
	
# asyncio运行异步函数入口函数
async def main():
    # 将协程包装任务,在后台并发执行
    f1 = asyncio.create_task(home1())
    f2 = asyncio.create_task(home2())
    # 同时启动所有协程，并发执行 可以获取返回值
    await asyncio.gather(f1, f2)
    pass
    
if __name__ == '__main__':
    asyncio.run(main())
    print("耗时", time.time()-s)

import asyncio
import time
# 记录当前时间
s = time.time()

async def task1():
    print("Task1 开始")
    # 这行代码会让出控制权2秒钟
    await asyncio.sleep(2)  # 比如网络请求
    print("Task1 完成")

async def main():
    task_list = []
    for i in range(5):
        task = asyncio.create_task(home1())
        task_list.append(task)
        # *进行解构列表
    await asyncio.gather(*task_list)
    pass

# 创建一个协程对象, 协程对象不会自动执行,需要通过事件循环来进行调用
if __name__ == '__main__':
    asyncio.run(main())
    print("耗时", time.time() - s)

future并发

使用ensure_future原因是: 需要兼容旧版本或处理未知类型的可等待对象，否则优先使用 asyncio.create_task()

ensure_future和create_task设置并发操作

import asyncio
import time
# 记录当前时间
s = time.time()

async def task1():
    print("Task1 开始")
    await asyncio.sleep(4)  # 暂停2秒
    print("Task1 完成")

async def task2():
    print("Task2 开始")
    await asyncio.sleep(4)  # 暂停1秒
    print("Task2 完成")


async def main():
    # ensure_future 创建协程任务对象
    f1 = asyncio.ensure_future(task1())
    f2 = asyncio.ensure_future(task2())
    # 并发执行多个协程任务对象
    await asyncio.wait([f1, f2])

if __name__ == '__main__':
    asyncio.run(main())
    print("耗时", time.time()-s)

import asyncio
import time

s = time.time()

    list_msg = []

        a = task1()

def parse_data(html):
    xml = etree.HTML(html)
    href = xml.xpath('//div[@class="stbody "]//p/text()|//div[@class="stbody first"]//p/text()')  #
    content = '\n'.join(href)
    print("链接：",content)  # []
    # print('解析完成')
    return content


def save_data(data,filename):
    filename = filename+'.txt'
    with open(filename,'w',encoding='utf-8')as f:
        f.write(data)
# 启动方法
def run(url,filename):
    print("参数：",url,filename)
    html =get_data(url)  # 请求一级页面
    href = parse_data(html)  # 解析一级页面链接
