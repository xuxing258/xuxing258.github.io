cookie逆向案例

如何确定和定位cookie加密数据位置

  1. 先全部清空cookie内容, 判断能否请求成功,
    • 请求成功则表示不是cookie加密
    • 不成功可查看缺少什么参数,以及查看后端返回的内容
  2. 点击分页查看那个cookie数据进行了变换
    • cookie是有时间的, 如果时候很长并且没有改变内容则无需进行破解
    • 时间很短并且还每次请求都发生变化,是一定需要进行破解的

常见问题

PyExecJS

安装的PyExecJS不是PyExecJS2会出现问题

  • PyExecJS最常见问题（如你之前用execjs调用 Node.js 时的编码错误),因为 Windows 默认编码是gbk，而现代程序普遍用utf-8。
  • 当使用execjs出现gbk格式不对是; 可使用以下
  • 注意: pyexecjs安装时是安装的pip install pyexecjs2

functools和subprocess都是 Python 的标准库模块，不需要额外安装

# -*- coding: utf-8 -*-
import subprocess
from functools import partial

# 全局修改subprocess.Popen的默认行为，强制其所有调用都自动使用encoding='utf-8'参数，无需每次手动指定。
subprocess.Popen = partial(subprocess.Popen, encoding='utf-8')
