微任务和宏任务

  • 宏任务:常见的定时器,用户交互事件等等.（宏任务就是特定的这些个任务,没什么特殊含义）
  • 微任务:Promise相关任务,MutationObserver等（一样，只是一种称呼而已！！！）

定时器时间相同的情况下

原则一
  • 万物皆从全局上下文准备退出，全局的同步代码运行结束的这个时机开始,全局的同步代码都执行完了,剩下的都是异步代码,
function app() {
    setTimeout(() => {
        console.log("1-1");
    });
    console.log("1-2");
    Promise.resolve().then(() => {
        console.log("1-3");
    });
}
app();

// 从上往下执行  先执行同步 在执行异步
// 先执行 1-2
原则二
  • 同一层级下(不理解层级，可以先不管，后面会讲),微任务永远比宏任务先执行
  • 微任务执行完毕才会执行宏任务

即Promise.then 比 set Timeout 先执行

原则三
  • 同一层级下,微任务永远比宏任务先执行,
  • 同一层级微任务内部具有微任务 先执行内部微任务 以此类推未有微任务为止, 再次执行初始层级的宏任务
  • 宏任务时间相同 是同层级 按顺序执行 => 再执行微任务的宏任务

安装问题

node安装 电脑桌面壁纸管理 会阻止node路径 导致不能使用node

mongodb 安装

mongodb无法启动 无法链接

connecting to: mongodb://127.0.0.1:27017/?compressors=disab1edl&gssapiServiceName=mongodb

  1. 出现这个问题 查看任务管理器 中的 服务器 mongoodb 是否能启动
  2. 打开服务 查看mongodb服务是否启动

解决一

  • 通过新建data文件启动 mongod --dbpath 存储文件夹地址 --port 27017

解决二

  • 通过任务管理器查看mongodb服务, 打开服务之后打开mongodb属性
  • 设置登录本地系统登录 进行设置即可

通过axios下载

dom.onclick = async function(){
    let data = await axios({
        method:"get",
        url:"http://localhost:8000/person",
        responseType: 'blob' // 重要，指定响应类型为blob
    })
    // 创建一个Blob对象，
    const blob = new Blob([data.data], { type: 'text/markdown' });
    // 创建一个指向该Blob的URL
    const url = window.URL.createObjectURL(blob);
    // 创建一个a标签
    const a = document.createElement('a');
    a.style.display = 'none';
    a.href = url;
    // 设置下载的文件名
    a.download = 'example.md'; // 你可以
    // 触发a标签的点击事件
    a.click()_
}
