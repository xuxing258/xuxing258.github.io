npm

npm（Node Package Manager）是一个用于管理和分发JavaScript模块的默认软件包管理工具, 是Node.js的标准软件包管理器，随同Node.js安装一起提供

通过npm，开发者可以轻松地安装、更新、卸载和管理Node.js模块（也称为软件包）。这些模块可以是开源的，由其他开发者编写并在npm注册中心发布，也可以是自己编写的模。

初始化

  • 检测版本 npm -v

  • 初始化项目 npm init npm init -y

    命令会在目录下生成一个package.json的文件，用来记录项目的各种信息。

    如果npm init出现权限不足, 与安装nodemon一样进行设置权限, 即可使用

{
  "name": "node",      项目名称
  "version": "1.0.0",  当前文件版本号
  "main": "index.js",  入口文件
  "scripts": {         测试或命令运行 npm test
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "keywords": [],     关键词
  "author": "",        作者
  "license": "ISC",   协议
  "dependencies": {     生产
    "body-parser": "^1.19.0",
    "express": "^4.17.1"
  }, 
  "devDependencies": {   开发
    "jsondom": "^0.4.1"
  },
  "description": ""   描述
}

基础指令

npm install 包名 简写 npm i 包名 ，可在包名后面接上@版本号。

-save -S 生产环境 --save-dev -D 开发环境 -g 全局环境

npm uninstall 包名 简写 npm un 包名

npm update [包名]

npm list 列出当前项目或全局安装的所有 npm 包

npm cache clean --force 清除npm缓存

npm test 默认测试

目录信息

  • package.json 项目的配置信息以及所需要的各种模块, 还能通过npm install这个命令，自动下package.json文件里所需的模块及依懒;
  • node_modules 模块文件源码位置,同时也会将模块的依懒安装,
  • package-lock.json 安装之后锁定包的版本,对于任何 npm 修改node_modules或package.json, 它描述了生成的确切树，以便后续安装能够生成相同的树,

换源

//查看源
npm config get registry
//更换源
npm config set registry https://registry.npmjs.org
//淘宝源
npm config set registry https://registry.npmmirror.com

npm发布

  1. 注册npm账号
  2. 注册完成, 退出重新进入 进行邮箱验证
  3. 项目中通过 npm init -y 初始化并设置信息
  4. 上传时:
    • 先登录npm官网账号
    • 通过控制台cmd发布
      • npm login
    • 输入登录账号密码
      • 账号:
      • 密码: 密码按上时光标不会闪烁
  5. 登录成功 会返回一个npm地址
  6. 在cmd中输入代码 npm publish 代码将会发送到npm上

nodemon

windows安装

npm i nodemon -g  //全局安装
nodemon 文件名  //启动

mac安装

npm i nodemon -g  //全局安装
  • 无法启动 一般是电脑禁止运行了
  • 管理员身份打开powerShell或者cmd
  • 输入set-ExecutionPolicy RemoteSigned
  • 选择Y 或者A ，就好了

更多解决方案: https://github.com/remy/nodemon/blob/master/faq.md

yran

用于管理Node.js包依赖关系的工具，类似于npm。使用yarn可以更快地安装和管理依赖项

npm install -g yarn //安装
yarn -v  // 显示版本

npm 和 yarn区别

  • yarn init：创建一个新包
  • yarn run：运行 package.json 中定义的脚本
  • yarn publish：发布一个包
  • yarn cache clean：从缓存文件夹中删除所有数据

这些命令使两个管理器之间的切换变得容易，但有一些不同的命令可能会导致混淆。

  • yarn：安装依赖
  • yarn add [package]：安装一个包
  • yarn add --dev [package]：安装包作为开发依赖项
  • yarn remove [package]：卸载一个包
  • yarn remove [package]：卸载开发依赖包
  • yarn upgrade：更新的依赖关系
  • yarn upgrade [package]：更新包

yarn 还有一些独特的命令，这些命令在 npm 下没有相同的。例如，why 命令显示需要一个包的原因：它可能是一个依赖项、一个本地模块或一个项目依赖项。

换源

yarn config get registry
# 国内淘宝源
yarn config set registry https://registry.npm.taobao.org/
# 官方源
yarn config set registry https://registry.yarnpkg.com

pnpm

npm i pnpm -g  	//全局安装
pnpm  -v  		//启动
  • pnpm install 初始化 pnpm i 自动下载依赖
  • pnpm add 包名 添加 pnpm add sax@3.0.0 安装指定版本
  • pnpm remove 包名 删除
  • pnpm up 包名 更新
