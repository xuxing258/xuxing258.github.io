模块化

  • 模块化是一种组织和管理代码的方法，它允许将代码分割成独立的模块，每个模块负责特定的功能。模块化使得代码更易于维护、测试和重用，同时也提高了开发效率
  • 模块分为内置模块,自定义模块, 第三方模块
  • 在 Node模块系统中，每个js文件都被视为一个独立的模块，每个文件包含了一组相关的功能。一个模块可以导出（export）一些内容，供其他模块使用，也可以导入（import）其他模块的内容来使用
  • node中每个 Js 文件的所有代码都是被一个立即执行函数包裹住
// 函数的 arguments参数  中的callee属性 指向自身函数
console.log(arguments.callee + "" )    
console.log(arguments )

function (exports, require, module, __filename, __dirname) { 
	// 该立执行参数函数包裹这整个js的内容  其中有中5种形参
}

函数形参

  • __filename 文件的绝对路径
  • __dirname 文件夹的绝对路径
  • require 专门用来请求模块的=>内置模块,第三方模块,自定义模块
  • module 模块对象
  • exports 跟ES6的export一样的

导入

require是一个用于导入模块的函数。它是Node.js中模块系统的核心部分，用于加载其他JavaScript文件或模块，并使其可在当前模块中使用

require函数接受一个模块标识符（通常是一个字符串），并返回该模块导出的内容

模块标识符可以是相对路径或绝对路径，也可以是包名或核心模块的名称

导入文件时会执行导入文件一次

导出

exports是一个特殊的对象，用于导出模块的内容。通过将函数、变量或对象赋值给exports对象的属性，可以将它们暴露给其他模块引用和使用

// Js文件
let a = 10;
let b = 10;
let c = 10;
export {a,b,c}

// 导出
exports.mins = 10   
exports.fun = (a,b) => { return a + b }

// 默认导出
module.exprots = { name:'222" }
  • 文件中使用module.exports 导出数据 ,其他exports 无法导出
    • 单文件中只生效一次module.exports后写覆盖
  • exports 只是引用了 module.exports 的对象, 本质是通过module.exports 导出数据

exports引用 module.exports对象

let module = {
    exports : {  a:1  }
}
let exports = module.exports 
module.exports ===  exports // true

// 转化成node导出代码,  此时外部导入还可获取数据
exports.x = { a:1 } 

// exports原始对象是导出的, 设置新对象之后,不再进行导出,因为它不再绑定到 module.exports
exports = { a:1 }
console.log(module.exports == exports)  // false 
// 修改之后将导入数据文件不会获取到任何数据

path

  • path.extname(path文件路径) 获取文件的后缀名 | 扩展名
  • path.parse(__filename) 序列化路径,返回一个对象
  • path.basename(path) 方法会返回 path 的最后一部分
  • path.dirname(path) 方法会返回 path 的目录名
  • path.isAbsolute(path) 方法检测 path 是否为绝对路径
  • path.join() 方法会将所有给定的 path 片段连接到一起（使用平台特定的分隔符作为定界符）
  • path.relative(from, to) 用于将绝对路径转为相对路径
    • from 当前文件夹绝对路径
    • to 目标文件或文件夹绝对路径
  • path.resolve([...paths]) 方法接受一个或多个路径片段作为参数，并根据这些片段解析出一个绝对路径
    • 从左往右
