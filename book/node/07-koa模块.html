koa模块

Koa是由Express背后的团队设计的一种新的Web框架，旨在为Web应用程序和API提供更小，Koa在其核心内未捆绑任何中间件

app.js是整个koa入口文件，封装了context，request，response，以及最核心的中间件处理流程。

koa 需要配合 async 使用

http://koajs.cn/#introduction koa使用地址

使用koa模块
npm  install koa  // 安装koa模块
const Koa = require("koa")//引用Koa  返回一个构造函数
const app = new Koa()  // 执行构造函数

//  返回根页面数据
app.use( ctx =>{
    ctx.body  = "111"
})
 
 
 //创建并返回了一个 HTTP 服务器  监听
 app.listen(5000,()=>{
   console.log("5000端口成功运行"
 })
koa的API

  1. ​ app.use 为应用添加指定的中间件

  2. ​ app.listen 创建并返回了一个 HTTP 服务器 监听

自定义中间件

调用next()函数时，函数挂起并控件传递给定义的下一个中间件

let ss = 44
app.use( async (context,next) =>{
    // context.body  = "111"
    ss = ss + 66
    await next()
})

app.use( async (context,next) =>{
    context.body  = ss
})
配置路由

https://github.com/koajs/router/blob/master/API.md 路由api

原生路由
function route( url ) {
    let view = '这是404页面'
    switch ( url ) {
        case '/':
            view = '这是index页面'
            break
        case '/index':
            view = '大哥'
            break
        case '/todo':
            view = '真的烦'
            break
        case '/404':
            break
        //default:
        //break
    }
    return view
}

app.use( async context =>{
    // context.request.url
    let url = context.request.url
    let html = await route( url )
    context.body = html
})

cookie

kao的cookie属性

maxAge 一个数字，表示 Date.now()到期的毫秒数

signed 是否要做签名

expires cookie有效期

pathcookie 的路径，默认为 /'

domain cookie 的域

secure false 表示 cookie 通过 HTTP 协议发送，true 表示 cookie 通过 HTTPS 发送。

httpOnly    表示是否可以通过javascript来修改，设成true会更加安全  true就是不可以

overwrite 一个布尔值，表示是否覆盖以前设置的同名的Cookie（默认为false）。 如果为true，在设置此cookie时，将在同一请求中使用相同名称（不管路径或域）设置的所有Cookie将从Set-Cookie头部中过滤掉

// 设置cookie
ctx.cookies.set(name, value, [options])
//列子
ctx.cookies.set("luyao","shitger ",{
        maxAge : 20000
 })
// 获取cookie
ctx.cookies.get(name, [options])
// 不允许中文,可以将中文转换成unicode的格式

koa-session 中间件并配置

//配置

app.keys = ['some secret hurr'];   // 用来加密cookie

const  CONFIG  =  { 
  key : 'koa.sess' ,  // cookie 名称
    
  maxAge : 86400000 , // 过期时间
    
  rolling : false ,  // rolling为true，也就是每次都更新session  
    
  httpOnly : true,  // 表示是否可以通过javascript来修改，设成true会更加安全  true就是不可以
    
  signed : true ,  // signed设为true的时候会自动给cookie加上一个sha256的签名,从而防止cookie被篡改
  
  autoCommit : true , // 自动提交到响应头
    
  overwrite : true ,  // 是否允许重写
  
  refresh : false ,  // 在 session 即将过期时更新 session，所以我们可以始终保持用户登录状态 (默认为 false
    
  secure : true ,  //将cookie标记为仅与HTTPS一起使用。
} ;
//启动配置
app.use(session(CONFIG, app));

// 读取 session
cxt.session.name

// 设置 session   
cxt.session.name = value

// 销毁 session
ctx.session = null
