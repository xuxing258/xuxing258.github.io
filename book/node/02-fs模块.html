fs模块

fs模块全称 File System(文件系统)

注意: fs模块路径是 node的路径是以启动服务的路径为基准的,而不是以当前文件的路径为基准;

所有文件系统操作都具有同步、回调和基于 promise 的形式。

模块方法

  • 异步形式总是把完成回调作为其最后一个参数
    • 回调参数取决于具体方法
      • 第一个参数总是预留给异常, 如果操作被成功地完成，则第一个参数会为 null 或 undefined
    • 第二个参数返回获取数据 或 undefined

删除文件

  • fs.unlink(path, callback): 异步地删除文件。
  • fs.unlinkSync(path): 同步地删除文件。
    • path: 文件地址及后缀
fs.unlink('./66.js', (err) => { console.log('已成功地删除文件') });

let data = fs.unlinkSync('./1.css')

注意: 一般同步删除文件, 超大文件建议异步删除文件

读取文件

  • fs.readFile(path[, options], callback): 异步地读取文件的全部内容。
  • fs.readFileSync(path[, options]): 同步地读取文件的全部内容。
    • path：文件地址及后缀。
    • options（可选）：一个对象，包含读取选项，如编码方式等
      • 不指定编码方式，则返回原始的 Buffer 对象
fs.readFile('./66.js',"utf-8", (err,data) => {
    console.log(err,"报错信息",data,"获取的utf-8格式数据");
});

let data = fs.readFileSync("./66.js","utf-8")  // 字符串

注意: 读取路径格式使用 / 最好不要使用**\**

写入文件

  • fs.writeFile(file, data[, options], callback): 异步地将数据写入文件，如果文件已存在则覆盖。
  • fs.writeFileSync(file, data[, options]): 同步地将数据写入文件，如果文件已存在则覆盖。
    • file：文件地址及后缀。
    • data：要写入文件的数据，可以是字符串或Buffer对象。
    • options（可选）：一个对象，包含写入选项，如编码方式等。
    • callback：写入完成后的回调函数，接收一个错误参数。
fs.writeFile("./feng.js","写入数据",function(error){ console.log(error,"报错信息或者 null")})

// 再原有的基础追加
fs.writeFile("./feng.js","写入数据",{ flag : "a" },function(error){
    console.log(error,"报错信息或者 null");
})

文件追加

  • fs.appendFile(file, data[, options], callback): 异步地将数据追加到文件末尾，如果文件不存在则创建。
  • fs.appendFileSync(file, data[, options]): 同步地将数据追加到文件末尾，如果文件不存在则创建。
fs.appendFile("./feng.js", "写入数据", function (error) {
    console.log(error, "报错信息或者 null") 
})

创建文件夹

  • fs.mkdir(path[, options], callback): 异步地创建目录。
  • fs.mkdirSync(path[, options]): 同步地创建目录
    • path：要创建的目录的路径。
    • options（可选）：一个对象，包含创建目录的选项，例如权限等。
    • callback：创建完成后的回调函数，接收一个错误参数。
fs.mkdir("./6661",function(err){console.log(error,"报错信息或者 null")})

注意: 如果目录已经存在，fs.mkdir会抛出一个错误。 使用同步

删除文件夹

  • fs.rmdir(path,callback) : 异步地删除指定的空文件夹（目录)
  • fs.rmdirSync(path) : 同步地删除指定的空文件夹（目录）
    • path：要删除的目录路径。
    • callback：删除完成后的回调函数，接收一个错误参数
fs.rmdir("./666",function(error){ console.log(error,"报错信息或null");})

fs.rmdirSync("./666")

注意: 如果目录中包含文件或其他子目录，将会导致删除操作失败并抛出错误

删除文件或文件夹

  • fs.rm(path[, options], callback): 异步地删除指定的空文件夹（目录)或文件
  • fs.rmSync(path[, options]): 同步地删除指定的空文件夹（目录)或文件
    • path：要删除的文件或目录的路径。
    • options（可选）：一个对象，包含删除选项，例如递归删除子目录等。
      • recursive 设置为 true，则会递归删除目录及其所有子目录和文件。
      • 如果不提供配置对象或将 recursive 设置为 false，则仅能删除空目录或文件，并且不能删除非空目录
    • callback：删除完成后的回调函数，接收一个错误参数。
fs.rm("./22",{recursive: true},(err)=>{ console.log(err)})

fs.rmSync("./123",{recursive: true})

读取文件夹

  • fs.readdir(path, callback): 异步地读取目录下的文件和子目录列表。
  • fs.readdirSync(path): 同步地读取目录下的文件和子目录列表。
    • path：要读取的目录的路径。
    • callback：读取完成后的回调函数，接收两个参数：错误参数和一个包含目录内容的数组。
fs.readdir("filePath",callback)

let a = fs.readdirSync("文件夹路径")

文件或目录是否存在

  • fs.existsSync(path): 同步地检查文件或目录是否存在。
    • path：文件或目录的路径。
    • 如果路径存在则返回 true，否则返回 false。
let a = fs.existsSync("文件路径") // 返回true 或 false

文件或目录信息

  • fs.stat(path, callback) : 异步地获取指定路径的文件或目录的详细信息
  • fs.statSync(path) 同步获取文件或目录信息完成，并返回获取到的信息
    • path：文件或目录的路径。
    • callback：获取信息完成后的回调函数，接收两个参数：错误参数和一个描述文件或目录的 fs.Stats 对象
fs.stat("./1.txt",function(error,stats){ console.log(error,stats) })

let stats = fs.statSync("C:/Users/86151/Desktop/新建文件夹")

console.log('文件或目录大小：', stats.size);
console.log('最后访问时间：', stats.atime);
console.log('最后修改时间：', stats.mtime);

重命名或移动文件或文件夹

  • fs.rename(oldPath, newPath, callback): 异步地将文件重命名。
  • fs.renameSync(oldPath, newPath): 同步地将文件重命名。
    • oldPath：要重命名或移动的文件或目录的旧路径。
    • newPath：重命名或移动后的文件或目录的新路径。
    • callback：操作完成后的回调函数，接收一个错误参数
  • 将文件从 oldPath 重命名为 newPath。 返回 undefined
fs.rename("./66.js","6666.js",function(error){ console.log(error) })

fs.renameSync("./66.js","6666.js")

注意: 在重命名或移动文件或目录时，要求新路径不能与现有的文件或目录重复，且新路径所在的目录必须已存在。否则将抛出错误

监听文件|文件夹状态

  • fs.watch(path[, options], callback) 方法用于监视指定路径下文件和子目录的更改
    • path：要监视的文件或目录的路径。
    • options（可选）：一个对象，用于指定监视的选项，
      • persistent（布尔值，指定是否持久监视，默认为 true）
      • recursive（布尔值，指定是否递归监视子目录，默认为 false）等。
    • callback：当文件或目录发生更改时调用的回调函数，接收两个参数：eventType（字符串，表示事件类型，例如 'rename'、'change' 等）和 filename（字符串，表示发生更改的文件或目录的名称）。
  • fs.watchFile(path[, options], callback) 方法用于监视单个文件的更改
    • path：要监视的文件的路径。
    • options（可选）：一个对象，用于指定监视的选项，
      • persistent（布尔值，指定是否持久监视，默认为 true）
      • interval（以毫秒为单位的时间间隔，在该间隔内检查文件的更改，默认为 5007）等。
    • callback：当文件发生更改时调用的回调函数，接收两个参数：curr（表示文件的当前 fs.Stats 对象）和 prev（表示文件的先前 fs.Stats 对象）。
fs.watch("./6666",function(eventType, filename){
  console.log(eventType, filename);
})

fs.watch("./123.txt",function(eventType, filename){
  console.log(eventType, filename);
})

fs.watchFile("./6666/333.txt",function(curr,prev){
  console.log(curr,"当前的状态对象",prev,"以前的状态对象");
})

可读可写流

  1. 可读流在任意时刻,可堵了会处于三种状态
  2. readable.readableFlowing === null || true || false
  3. 为 null 时，没有提供消费流数据的机制，所以流不会产生数据
  4. 为true 时，可读流开始主动地产生数据并触发事件
  5. 为 false 时，暂时停止事件流动但不会停止数据的生成 readable.pause() 关闭
  6. data事件 : 数据传送时调用readable.pipe()， readable.resume() 或绑定监听器到 'data' 事件时 ，可读流流会转换到流动模式
  7. 在可读流上调用 'pipe' 事件 => 通过管道流入到可写流的位置
let read =  fs.createReadStream("./1.flac");         // 创建可读流
let ws =  fs.createWriteStream("./123/2.flac");      // 创建可写流
// 简写方法  一步到味,不需要你进行任何操作
read.pipe(Write)

let read =  fs.createReadStream("./1.flac");         
let ws =  fs.createWriteStream("./123/2.flac");     

// data事件   每次发送65486字节的数据给你  分批次发送  =>  最大值65486
read.on("data",(value)=>{
    ws.write(value)
    // console.log(read.readableFlowing);  // 查看可读流的情况 true发送中
})
// 数据发送完 触发
read.on("end",()=>{
    read.pause()  // 关闭管道 
    console.log("数据发送完成触发",read.readableFlowing);
})

readfile 和 可读可写流的区别

readFile 函数异步读取文件的全部内容，并存储在内存中，然后再传递给用户。

createReadStream 使用一个可读的流，逐块读取文件，而不是全部存储在内存中。

与 readFile 相比，createReadStream 使用更少的内存和更快的速度来优化文件读取操作。如果文件相当大，用户不必等待很长时间直到读取整个内容，因为读取时会先向用户发送小块内容。
