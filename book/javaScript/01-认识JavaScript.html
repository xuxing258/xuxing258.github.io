了解JavaScript

什么是JavaScript

JavaScript是一种高级的、动态的、解释型的编程语言。主要用于在网页上实现交互性和动态功能，但也可以在服务器端（使用Node.js）和其他环境中使用

JavaScript能做什么

任何能够用JavaScript实现的应用系统，最终都必将用JavaScript实现

  • 前端领域（本阶段主要学习的内容）

    ajax出现之后 JavaScript有了在WEB领域立足的土壤和根基，时至今日。JavaScript已经是WEB前端领域最重要的基石，一切现代化的WEB项目都离不开JavaScript。

    • 操作HTML页面（结构、样式、UI、动画......）
    • 响应浏览器事件
    • 用户行为交互
    • 数据验证
    • 前后端数据交互

  • 后端领域

    V8 JIT NodeJS 让JavaScript可以在服务端崭露头角，打破了JavaScript只能寄生在浏览器上的魔咒。CouchDB mongodb等基于JSON格式的NoSQL类型的数据库诞生，让JavaScript也可以在DB操作上大展身手。

    • WEB服务框架: express/KOA
    • NoSQL 数据库: mongodb CouchDB
    • 博客系统 : Ghost/hexo
    • 自动化构建领域: Gulp/Grunt

  • 桌面应用

    JavaScript还可以介入的桌面应用开发，主流有 electron Node-webkit hex React Navite

  • 游戏开发

    JavaScript在网页上也可以实现简单的游戏。使用HTML5的Canvas元素和WebGL技术，可以创建交互式的图形和游戏

  • 移动应用开发

    目前来说主流使用框架如React Native和Ionic，JavaScript可以用于开发跨平台的移动应用

前端领域JavaScript的组成

  • ECMAScript，描述了该语言的语法和基本对象；
  • 文档对象模型（DOM），描述处理网页内容的方法和接口；
  • 浏览器对象模型（BOM），描述与浏览器进行交互的方法和接口。

常见问题

JavaScript和Java有什么关系

本质上讲 JavaScript和 Java没有关系，只是JavaScript诞生之初 网景与sun合作想要推出 web端的脚本语言。JavaScript 的基本语法和对象体系，是模仿 Java 而设计的。但是，JavaScript 没有采用 Java 的静态类型。除此之外JavaScript和Java在语言层面上可以说是大相径庭。处于某种商业意图，这门语言才从一开始的LiveScript 改名为 JavaScript。

JavaScript与ECMAScript有什么关系

ECMAScript 只用来标准化 JavaScript 这种语言的基本语法结构。

JavaScript与jQuery有什么关系

jQuery就是由JavaScript编写的，本身就是一大段js代码，可以理解为一个简化了DOM操作的js库。

JavaScript与Node.js有什么关系

Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境，是一个让 JavaScript 运行在服务端（后端）的开发平台。它让 JavaScript 成为与PHP、Python、Perl、Ruby 等服务端语言平起平坐的脚本语言。

javascript基础认识

什么是编程语言?

  • 编程语言是一种用于描述计算机程序的形式化语言
  • 编程语言是一种标准化的交流技巧用于向计算机发出指令并让计算机执行特定的任务

最初使用的语言是----- 二进制语言(机器语言) 110001001001100010

  1. 进化汇编语言

    • 使用助记符代替机器指令的操作码，用符号或标号代替指令或操作数据
      • 如JMP MOV 一些特殊的指令操作计算机指令

  2. 进化高级语言

    • 编译型语言
    • 解释型语言

高级语言

计算机不能直接理解高级语言，只能直接理解机器语言，所以必须要把高级语言翻译成机器语言，计算机才能执行高级语言编写的程序

编译型语言

  • c c++ Delphi Pascal、Fortran
    • 编译型语言在执行之前需要将源代码编译成可执行文件才能执行
    • 编译型语言程序执行之前，存在专门的编译过程，会对源代码进行词法分析、语法分析、语义分析、代码优化等操作，生成可执行文件(二进制文件), 可执行文件可以在目标机器上独立地运行，不需要依赖编译器或解释器。
    • 执行速度通常较快，因为代码已经被转换为机器代码
    • 缺点: 一般是不支持跨平台

解释型语言

  • JavaScript python php Java Basic
    • 每次执行程序都需要通过解释器进行解析和执行。解释器会逐行读取源代码，并将其转换成机器码 ，然后根据代码的含义和语义执行相应的操作
      • 每执行一次都要翻译一次
    • 效率比较低，依赖解释器，跨平台性好
  • 解释型语言在开发完成后仍然是逐行解析和执行源代码

浏览器组成

浏览器 = 外壳 + 内核（渲染引擎 + 解析引擎）

外壳 :用户界面: 包括地址栏、后退/前进按钮、书签目录等

渲染引擎负责网页中内容的显示,HTML+CSS

JS引擎创建一个环境，在这个环境中Js引擎可以解释、编译和执行JS代码

js代码写在哪里

  • 页面script标签

    <script>代码</script> （推荐） 或

    <script type="text/javascript">代码</script> 或

    <script language="javascript">代码</script>

  • 外部文件引入

    创建：xxx.js

    引入：<script src="路径/xxx.js"></script>

写js代码需要注意什么

  • 严格区分大小写
  • 语法字符使用半角字符（字符串中不需要满足这个规则）
  • 完整语句后面使用 ; 结束符（可以不加，看个人或团队开发习惯而定）
  • 缩进对齐

注释写法

  • 单行注释

    //这里是注释

  • 多行注释

    /*
这里是注释
这里是注释
*/
