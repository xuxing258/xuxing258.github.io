Class

Class（类）是面向对象编程中的一个重要概念，它用于创建对象的蓝图或模板。在许多编程语言中都有类的概念

在JavaScript中，类是通过使用class关键字来定义的。类定义包含了属性和方法的声明，用于描述对象的行为和特征。实例化类时，会创建一个具体的对象，该对象具有类中定义的属性和方法

面向对象

编程方法

  • 面向过程
    • 以过程为中心，强调具体的操作步骤和流程，注重解决问题的具体算法和实现过程。它将问题分解为一系列的步骤或函数，通过依次执行这些步骤来完成任务。
    • 例如，计算两个数的和，可以定义多个函数分别实现输入数据、进行加法运算和输出结果等具体步骤。
  • 面向对象
    • 以对象为中心，将问题看作是由相互交互的对象组成的系统。强调对现实世界中的事物进行抽象和封装，将数据和操作封装在对象中。
    • 比如，把一辆汽车看作一个对象，它具有属性（如颜色、型号、速度等）和方法（如启动、加速、刹车等）

数据与操作的关系

  • 面向过程
    • 数据和操作是分离的。数据通常作为函数的参数传递，操作通过函数对数据进行处理。
    • 例如，在进行数学计算时，数据（如数字）作为参数传递给加法函数，函数执行加法操作并返回结果。
  • 面向对象
    • 数据和操作被封装在对象中，对象具有自己的状态（由数据表示）和行为（由方法实现）。对象通过调用自身的方法来操作内部数据，实现对自身状态的改变。
    • 例如，汽车对象可以通过调用加速方法来改变自身的速度属性

可维护性和可扩展性

  • 面向过程
    • 当程序规模较大时，由于函数之间的耦合度较高，一个函数的修改可能会影响到其他多个函数，导致维护成本较高。
    • 扩展性方面，添加新功能可能需要修改多个函数，容易引入新的错误。
  • 面向对象
    • 由于对象的封装性，使得代码的修改相对独立，对一个对象的修改通常不会影响到其他对象。同时，通过继承和多态等特性，可以方便地扩展现有代码。
    • 例如，要添加一种新类型的汽车，只需要创建一个新的汽车子类，继承父类的属性和方法，并添加特定于新类型汽车的属性和方法即可。

代码复用性

  • 面向过程
    • 主要通过函数的复用实现代码复用。可以将一些通用的函数提取出来，在不同的地方调用。但是，函数的复用程度相对较低，因为函数通常是针对特定的任务设计的。
    • 例如，一个排序函数可以在多个需要排序的地方调用，但如果需求稍有变化，可能就需要修改函数。
  • 面向对象
    • 通过类的继承、多态和组合等方式实现更高程度的代码复用。子类可以继承父类的属性和方法，并且可以根据需要重写或扩展这些方法。同时，可以将多个对象组合在一起，形成更复杂的对象。
    • 例如，一个图形绘制程序中，可以定义一个抽象的图形类，然后派生出各种具体的图形类（如圆形、矩形、三角形等），这些子类可以复用父类的绘制方法，并实现自己特定的绘制逻辑。

问题解决思路

  • 面向过程
    • 通常从具体的问题出发，分析问题的解决步骤，然后用函数来实现这些步骤。注重的是解决问题的流程和算法。
    • 例如，要解决一个生产线上的产品质量检测问题，会分析检测的具体流程，如读取产品数据、进行数据分析、判断质量是否合格等，然后分别编写函数来实现这些步骤。
  • 面向对象
    • 首先对问题进行抽象，识别出其中的对象以及对象之间的关系，然后通过对象之间的交互来解决问题。注重的是对现实世界的模拟和抽象。
    • 比如，对于同样的产品质量检测问题，会考虑将产品、检测设备、质检员等抽象为对象，定义它们的属性和方法，通过对象之间的消息传递来实现质量检测过程

  }

  • MyClass是类的名称，
  • constructor是构造函数

constructor

  • constructor方法是类的默认方法，通过new命令生成对象实例时，自动调用该方法。
  • 一个类必须有constructor方法，如果没有显式定义，一个空的constructor方法会被默认添加。
  • constructor方法默认返回实例对象（即this），完全可以指定返回另外一个对象。
    • 普通类型不会改变,引用类型才改变
  • 可以执行一些初始化操作，例如设置对象的属性或执行
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
    // 默认
    return this;
  }
  // 类的方法
  sayHello() {
    console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
  }
}

let person1 = new Person("Alice", 25);
person1.sayHello();  // 输出: "Hello, my name is Alice and I am 25 years old."

原型

每个类（class）都有一个原型（prototype）对象，原型对象是类的一个特殊对象，它包含了类的方法和属性，可以被类的所有实例共享。当你创建一个类时，类的方法和属性会被定义在类的原型对象上

class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
    greet() {
        console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
    }
}

// 属性不相同 但是可使用相同的方法
const person1 = new Person("Alice", 30);
const person2 = new Person("Bob", 25);
// 使用的是同一个函数   true
person1.greet == person2.greet 

var a = {  aa(){ } }
var a1 = { aa(){}}
// false
a.aa === a1.aa

原型的属性和方法会被实例继承,

  • 可以避免在每个实例对象中重复定义相同的方法，节省了内存空间

new

使用 new 关键字调用构造函数将执行以下操作

  • 创建一个新的空对象：
    • 使用 new 关键字调用类的构造函数会创建一个新的空对象，该对象将成为类的实例。
    • 这个对象是一个空的 JavaScript 对象，没有任何属性和方法。
  • 将新对象的原型链接到类的原型（即实现继承）：
    • 在 JavaScript 中，每个对象都有一个原型（prototype），它是对象的属性和方法的来源。当使用 new 关键字调用类的构造函数时，新创建的对象将通过原型链与类的原型关联起来，从而继承类的属性和方法。
  • 将构造函数的作用域设置为新对象的作用域：
    • 在构造函数内部，this 关键字引用的是新创建的对象。
    • 通过将构造函数的作用域设置为新对象的作用域，构造函数可以访问并操作新对象的属性和方法。
  • 执行构造函数内的代码，以初始化对象的状态：
    • 构造函数是一个特殊的方法，在创建类的实例时自动调用。通过在构造函数内部编写代码，可以对新对象进行初始化操作，例如设置属性的初始值或执行其他必要的操作。
  • 返回新对象作为类的实例：
    • 构造函数执行完毕后，将返回新创建的对象作为类的实例。这意味着可以将该对象分配给一个变量，以便稍后使用和操作该实例
/*
	一个新的空对象被创建。
    这个空对象被关联到该函数的原型（prototype）。
    执行函数内部的代码，同时 this 被指向这个新创建的对象。
    如果函数没有显式返回一个对象，则返回这个新对象。
*/
class Person {
  constructor(name) {
    this.name = name;
  }
  sayHello() {
    console.log(`Hello, my name is ${this.name}.`);
  }
}

let person1 = new Person("Alice");

原型属性

proto属性

实例对象具有 __proto__ 属性，用于指向其原型;

类具有 prototype 属性，用于指向其原型;

class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }
    greet() {
        console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
    }
}

// 在类的原型上添加一个属性
Person.prototype.species = "Human";
const person1 = new Person("Alice", 30);
const person2 = new Person("Bob", 25);

console.log(person1.__proto__ === Person.prototype); // 输出: true
console.log(person2.__proto__ === Person.prototype); // 输出: true

  • 原型的属性和方法会被实例继承,

    • 可以避免在每个实例对象中重复定义相同的方法，节省了内存空间

原型链

在访问实例的属性或方法时，如果实例自身没有定义该属性或方法，JavaScript 引擎会沿着原型链向上查找，直到找到对应的属性或方法或者到达原型链的顶端

原型链的顶端

在 JavaScript 中，每个对象都有一个原型属性 __proto__ , 指向其构造函数的原型对象。原型对象本身也是一个对象，也有自己的原型，这样就形成了一个原型链

class Person(){	}	
let xiaoming = new Person();

// 原型对象的原型对象
console.log(xiaoming.__proto__.__proto__);

// 原型对象的原型对象的构造函数是谁
console.log(xiaoming.__proto__.__proto__.constructor); //Object

// 那我们看看还能不不能再向上查原型对象
console.log(xiaoming.__proto__.__proto__.__proto__);
// 结果为null

原型链终点

在 JavaScript 中，原型链的终点是 Object.prototype, 几乎所有的对象都是从 Object 对象通过原型继承而来的，包括自定义对象和内置对象。Object.prototype 是所有对象的顶层原型，即原型链的最终层级

let arr = new Array()
arr.__proto__.__proto__  // object

let num = new Number()
num.__proto__.__proto__  // object

继承

允许一个类继承另一个类的属性和方法。子类可以通过继承来重用父类的代码，并可以添加、修改或覆盖父类的行为

类的继承

使用 extends 关键字实现继承

class Animal {
  constructor(name) {
    this.name = name;
  }
  eat() {
    console.log(`${this.name} is eating.`);
  }
}

class Dog extends Animal {
  bark() {
    console.log(`${this.name} is barking.`);
  }
}

let dog = new Dog("Buddy");
dog.eat();  // 继承自父类 Animal 的方法
dog.bark(); // 子类 Dog 自己的方法

子类存在 constructor 方法时super函数，必须在使用 this 关键字之前和构造函数返回之前被调用

super

当在子类的构造函数中使用 super 调用父类的构造函数时，super 会执行父类的构造函数，并将当前子类实例作为 this 上下文。这确保了从父类继承的属性可以正确地被初始化到子类实例上

class Parent {
  constructor(name) {
    this.name = name;
  }
}

class Child extends Parent {
  constructor(name, age) {
    super(name); // 调用父类的构造函数
    this.age = age;
  }
}

let child = new Child("Alice", 10);
console.log(child.name); // 输出: "Alice"
console.log(child.age);  // 输出: 10

同名方法

  • 父子属性或方法同名时,使用子级方法和属性
  • 子类的方法中使用 super 调用父类的同名方法
  • 使用父级属性条件:
    • 父子属性不能重复, 子级可通过this.父级属性使用
class Animal {
    constructor() {
        this.name = "哈哈哈"
    }
    greet() {
        console.log(this.name ,"Hello, I'm the parent.");
    }
}

class Dog extends Animal {
    constructor() {
        super();
        this.name = "嘻嘻嘻"
    }
    greet() {
        super.greet()
        console.log(`${this.name} is barking.`);
    }
}

let dog = new Dog();
dog.greet();

静态方法

定义在类本身上而不是类的实例上的方法。它们在类级别上执行操作，而不需要通过类的实例进行调用。静态方法通常用于实现与类相关的实用功能或工具函数

使用static关键字来定义静态方法, 就表示该方法不会被实例继承，而是直接通过类来调用

class Person{
    constructor(name,age){ 
        this.name = name;
        this.age = age;
    }
    showName(){
        return `这是通过构造对象调用方法`;
    }
    static aaa(){
        return `这是静态方法,是通过类来调用的方法`;
    }
}

var p1 = new Person('wuwei',18)
console.log(p1.showName);
console.log(Person.aaa());

新特性

私有方法只能在定义它的类内部访问，专用方法名称以开头#

class MyClass {
  #privateMethod() {
    console.log('This is a private method.');
  }

  publicMethod() {
    this.#privateMethod();  // 在类内部调用私有方法
  }
}

let myObject = new MyClass();
myObject.publicMethod();  // 输出: "This is a private method."
myObject.#privateMethod();  // 语法错误，无法访问私有方法

构造函数

构造函数是一种特殊的函数，用于创建和初始化通过 new 关键字创建的对象。构造函数通常用于定义对象的属性和方法，并且可以通过 this 关键字将属性和方法绑定到将要创建的对象

// 定义一个构造函数 Person
function Person(name, age) {
    this.name = name;
    this.age = age;
    
    this.greet = function() {
        console.log(`Hello, my name is ${this.name} and I am ${this.age} years old.`);
    };
}

// 使用构造函数创建对象
const person1 = new Person("Alice", 30);
const person2 = new Person("Bob", 25);

person1.greet(); // 输出：Hello, my name is Alice and I am 30 years old.
person2.greet(); // 输出：Hello, my name is Bob and I am 25 years old.

构造函数this

构造函数中的 this 关键字表示正在创建的实例对象。

当使用 new 关键字调用构造函数时，构造函数内部的 this 将绑定到新创建的对象上

function Person(name, age) {
  this.name = name;
  this.age = age;

  this.greet = function() {
    console.log(`Hello, my name is ${this.name} and I'm ${this.age} years old.`);
  };
}

let person = new Person("Alice", 25);
person.greet(); // 输出: Hello, my name is Alice and I'm 25 years old.

原型

每个构造函数对象都有一个原型，它充当对象的模板，包含可以被对象实例共享的属性和方法

构造函数可以通过原型继承属性和方法。原型形成了对象之间的原型链，使得对象可以访问其原型中定义的属性和方法，以及原型链上其他级别的属性和方法

通过构造函数的 prototype 属性来定义构造函数的原型, 在构造函数的原型上添加属性和方法

function Person(name) {
    this.name = name;
}
let person1  = new Person("Alice")

// 原型添加数据
Person.prototype.greet = function() {
  console.log(`Hello, my name is ${this.name}`);
};

let person1 = new Person("Alice", 25);
let person2 = new Person("Bob", 30);

person1.greet(); // 输出: Hello, my name is Alice 
person2.greet(); // 输出: Hello, my name is Bob

注意: 构造函数和类基本一致
