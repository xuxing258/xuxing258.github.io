作用域

作用域（Scope）在编程中指的是变量、函数和对象在代码中可访问的范围。它规定了在何处以及如何查找和访问变量

全局作用域

  • 全局作用域是在代码的最外层声明的变量和函数所在的作用域。在全局作用域中声明的变量和函数可以在代码的任何地方进行访问
  • 在浏览器环境中，全局作用域中函数会成为全局对象 window 的属性和方法。可以使用 window 对象来访问全局变量和函数
  • 全局的 let 和 const 变量不会挂载在特定的对象上，它们仅在全局作用域中存在
  • 在全局作用域中，函数的声明会被提升到作用域的顶部
let globalVariable = 'Hello, world!';

function globalFunction() {
  console.log(globalVariable);
}

console.log(globalVariable); // 输出 "Hello, world!"
globalFunction(); // 输出 "Hello, world!"

局部作用域

局部作用域对于封装代码块中的变量和函数非常有用，可以避免命名冲突和变量污染的问题。

同时，通过控制变量的作用域，可以提高代码的可读性和维护性

函数作用域

局部作用域是在函数内部声明的变量和函数所在的作用域,

在函数内部声明的变量和函数只能在函数内部进行访问

function myFunction() {
  var localVariable = 'Hello, world!';
  console.log(localVariable);
}

myFunction(); // 输出 "Hello, world!"
console.log(localVariable); // 报错，localVariable 不在当前作用域内
​```

在上面的示例中，变量 `localVariable` 在函数内部声明，因此只能在函数内部进行访问。在函数外部访问该变量会导致报错。

块级作用域

引入的 let 和 const 关键字可以在块级作用域中声明变量。块级作用域是指由花括号 {} 包裹的代码块（如循环、条件语句的代码块）

外部不能调取内部的变量,内部可以调用外部的变量

if (true) {
  var x = 10; // 在块级作用域之外仍然可访问
  let y = 20; // 仅在块级作用域内可访问
  const z = 30; // 仅在块级作用域内可访问
}

console.log(x); // 输出 10
console.log(y); // 报错，y 不在当前作用域内
console.log(z); // 报错，z 不在当前作用域内
​```

在上面的示例中，变量 `x` 在块级作用域之外声明，因此可以在块级作用域外部进行访问。但变量 `y` 和 `z` 在块级作用域内部声明，只能在块级作用域内部进行访问。

作用域链

作用域链（Scope Chain）是 JavaScript 中用于查找变量和函数的机制。

当代码中访问一个变量或函数时，JavaScript 引擎会按照作用域链的顺序逐级向上查找，直到找到对应的标识符或达到全局作用域。

作用域链变量查找特点: 当在函数中访问一个变量时，JavaScript 引擎首先在当前函数的作用域中查找该变量。如果找到了对应的变量，则使用该变量的值。如果没有找到，则会向上一级作用域继续查找，直到找到对应的变量或达到全局作用域。

作用域销毁

作用域的销毁是由垃圾回收机制负责的，它负责释放不再被使用的作用域所占用的内存空间

  • 函数执行完成： 当函数执行完毕后，函数作用域中的局部变量和函数会被标记为不再需要，然后由垃圾回收机制自动回收它们占用的内存空间

  • 函数的销毁

    在函数作用域中定义的函数，当函数执行完毕后，这些函数也会被销毁。函数对象会被标记为不再需要，并在垃圾回收时被回收。

  • 全局变量会在您关闭页面是被删除。

注意: 作用域的销毁是由垃圾回收机制自动处理的，开发者无需手动进行销毁操作。垃圾回收机制会周期性地检查不再需要的变量和作用域，并进行回收释放内存

this

this 用于引用当前执行上下文中的对象。它的值取决于函数的调用方式和上下文

全局上下文中的 this：

当在全局作用域中使用 this 时，它将引用全局对象（在浏览器环境中通常是 window 对象）

console.log(this); // 在浏览器中，输出全局对象 window

函数调用中的 this

在函数内部，this 的值取决于函数的调用方式。

  • 函数作为普通函数调用时，this 指向全局对象或 undefined（在严格模式下）。

    function myFunction() {
  console.log(this);
}
myFunction(); // 在浏览器中，输出全局对象 window

  • 函数作为对象的方法调用时，this 指向调用该方法的对象。

    var obj = {
  name: 'John',
  greet: function() {
    console.log('Hello, ' + this.name + '!');
  }
};

obj.greet(); // 输出 "Hello, John!"

  • 使用 call、apply 或 bind 方法显式设置函数的执行上下文时，this 的值会被显式指定。

    • call 方法的语法是：function.call(thisArg, arg1, arg2, ...)
    • apply 方法的语法是：function.apply(thisArg, [argsArray])
    • bind 方法的语法是：function.bind(thisArg, arg1, arg2, ...)
    myFunction.call()
dom.onclick = function(){}.call()

箭头函数中的 this

箭头函数具有词法作用域中的 this 值，它继承自外部作用域。箭头函数没有自己的 this 绑定，并且无法通过 call、apply 或 bind 方法来改变

var obj = {
  name: 'John',
  sayHello: function() {
    setTimeout(() => {
      console.log('Hello, ' + this.name + '!');
    }, 1000);
  }
};
obj.sayHello(); // 输出 "Hello, John!"

严格模式

严格模式（Strict Mode）是JavaScript的一种执行模式，通过启用严格模式，可以对代码进行更严格的解析和执行，以减少一些常见的错误，并使代码执行更加安全和可靠

"use strict";
  • 变量必须先声明：在严格模式下，所有变量必须使用 var、let 或 const 关键字声明，否则会抛出错误。
  • 禁止变量泄漏到全局作用域：在严格模式下，忘记使用 var 或 let 声明变量会将其限定在当前作用域，不会自动成为全局对象的属性。
  • 禁止删除变量、函数和函数参数：在严格模式下，使用 delete 操作符删除变量、函数或函数参数会抛出错误。
  • 禁止使用重复的参数名：在严格模式下，函数的参数名称必须唯一，重复的参数名会导致错误。
  • 禁止使用八进制字面量：在严格模式下，八进制字面量（如 0123）将被视为无效，必须使用完整的十进制表示。
  • 禁止使用 with 语句：在严格模式下，with 语句被禁用，因为它会引起代码可读性和性能问题。
  • 其他变化：严格模式还会对 this 的值进行更严格的限制，禁止对只读属性赋值，禁止对不可扩展对象添加属性等。
