Iterator

迭代器（Iterator）是一种对象，用于提供一种逐个访问集合或序列中元素的方法, 迭代器提供了一种统一的方式来遍历数据结构，例如数组、字符串、Set、Map等.

迭代器对象具有以下两个主要方法：

  1. next() 方法：返回一个具有两个属性的对象，即 { value, done }。value 表示当前迭代的值，done 表示迭代是否已完成。
  2. Symbol.iterator 方法：返回迭代器对象自身。这使得迭代器对象可以通过 for...of 循环或使用 ... 扩展运算符进行迭代。
let arr = [11,12,3]
// 获取数组上的Iteator  然后执行返回一个 Iterator{}
let iterator = arr[Symbol.iterator]() 
// 里面具有一个next()方法  

console.log(iterator.next()); // 输出: { value: 11, done: false }
console.log(iterator.next()); // 输出: { value: 12, done: false }
console.log(iterator.next()); // 输出: { value: 3, done: false }
console.log(iterator.next()); // 输出: { value: undefined, done: true }

Iterator本质，就是一个指针对象。

过程是这样的：

（1）创建一个指针对象，指向当前数据结构的起始位置。

（2）第一次调用指针对象的next方法，可以将指针指向数据结构的第一个成员。

（3）第二次调用指针对象的next方法，指针就指向数据结构的第二个成员。

（4）不断调用指针对象的next方法，直到它指向数据结构的结束位置。

（5）next() 函数执行 value值 done判断跌代是否结束 停止循环

普通函数实现Iterator

let  obj = [11,12,3]
function myIter(obj){
  let i = 0;
  return {
    next(){
      let done = (i>=obj.length);
      let value = !done ? obj[i++] : undefined;
      return {
        value,
        done,
      }
    }
  }
}

当 for of执行的时候，循环过程中引擎就会自动调用这个 对象上的迭代器方法， 依次执行迭代器对象的 next 方法,将 next 返回值赋值给 for of 内的变量，从而得到具体的值

数据结构

Set和Map是两种不同的数据结构

Set

Set对象允许你存储任何类型（无论是原始值还是对象引用）的唯一值。

类似于数组，但是不允许重复的元素。您可以使用Set来存储一组独特的值，并且可以执行添加、删除和检查值是否存在的操作

构造函数

new Set(iterable) 传入一个可迭代对象，它的所有元素将不重复地被添加到新的 Set 中

const set = new Set([1, 2, 3, 4, 5]);
  • 创建一个空的Set
    • const mySet = new Set();
  • 添加元素到Set
    • mySet.add(value);
  • 删除元素从Set
    • mySet.delete(value);
  • 检查值是否存在于Set
    • mySet.has(value);
  • 获取Set的长度
    • mySet.size;
  • 清空Set
    • mySet.clear();
  • 迭代Set中的元素
    • mySet.forEach((value) => { console.log(value) });

WeakSet

允许您存储对象引用，并且这些引用是弱引用。与Set不同，WeakSet中的元素只能是对象，并且对这些对象的引用是弱引用，这意味着如果没有其他地方引用该对象，它可能会被垃圾回收

  1. WeakSet中的元素必须是对象，不能是原始值（如数字、字符串等）。
  2. 对象是弱引用，如果没有其他地方引用该对象，它可能会被垃圾回收。
  3. WeakSet没有迭代方法，因此无法直接获取所有元素。
  4. WeakSet不可迭代，也没有size属性。

常用方法

  • 添加对象到WeakSet
    • weakSet.add(obj);
  • 从WeakSet中删除对象
    • weakSet.delete(obj);
  • 检查对象是否存在于WeakSet
    • weakSet.has(obj);
const weakSet = new WeakSet();
const obj1 = { name: 'Object 1' };
const obj2 = { name: 'Object 2' };

weakSet.add(obj1);
weakSet.add(obj2);

Map

Map 对象保存键值对, 与普通的对象相比，Map提供了更灵活的键类型，可以使用任意数据类型作为键，并且保留了键值对的插入顺序。

Map 对象是键值对的集合。Map 中的一个键只能出现一次；它在 Map 的集合中是独一无二的。

构造函数

new Map(iterable) 一个元素是键值对的数组或其他可迭代对象

const myMap = new Map([ [1, "one"],[2, "two"], [3, "three"] ])
  • 创建一个空的Map
    • const myMap = new Map();
  • 添加键值对到Map
    • myMap.set(key, value);
  • 删除键值对
    • myMap.delete(key);
  • 获取键对应的值
    • myMap.get(key);
  • 检查键是否存在于Map
    • myMap.has(key);
  • 获取Map的长度
    • myMap.size;
  • 清空Map
    • myMap.clear();
  • 迭代Map中的元素
    • myMap.forEach((value) => { console.log(value) });

WeakMap

WeakMap是JavaScript中的一种特殊映射（Map）数据结构，它类似于Map，但键是弱引用，值可以是任意类型的对象。与Map不同，WeakMap中的键只能是对象，并且对这些对象的引用是弱引用。这意味着，如果没有其他地方引用该对象，它可能会被垃圾回收。

WeakMap的特点如下：

  1. WeakMap中的键必须是对象，不能是原始值（如数字、字符串等）。
  2. 对象是弱引用，如果没有其他地方引用该对象，它可能会被垃圾回收。
  3. WeakMap没有迭代方法，因此无法直接获取所有键值对。

常用方法：

  • 添加键值对到WeakMap
    • weakMap.set(key, value);
  • 获取键对应的值：
    • weakMap.get(key);
  • 检查键是否存在于WeakMap：
    • weakMap.has(key);
  • 删除键值对
    • weakMap.delete(key);

示例：

const weakMap = new WeakMap();

const obj1 = { name: 'Object 1' };
const obj2 = { name: 'Object 2' };

weakMap.set(obj1, 'Value 1');
weakMap.set(obj2, 'Value 2');

console.log(weakMap.get(obj1)); // 输出: "Value 1"
console.log(weakMap.has(obj2)); // 输出: true

请注意，由于键的弱引用特性，WeakMap中的键不能被迭代或计算其大小。WeakMap主要用于需要存储对象引用的场景，并且在对象不再需要时自动进行垃圾回收的情况下非常有用

Generator

生成器（Generator）是一种特殊的函数，它可以在执行期间暂停和恢复。生成器函数使用 function* 语法进行定义，并且可以通过使用 yield 关键字来暂停函数的执行并产生一个值。每次调用生成器函数时，都会返回一个迭代器对象，该对象可以用于控制生成器的执行

function* myGenerator() {
  // 生成器函数的主体

  yield value; // 暂停函数的执行并产生一个值

  // 可以有更多的 yield 语句和其他逻辑

  return value; // 结束生成器的执行，并返回一个值
}

生成器函数的执行过程如下：

  1. 创建迭代器对象：调用生成器函数时，会返回一个迭代器对象。这个迭代器对象用于控制生成器函数的执行。
  2. 执行生成器函数：当迭代器的 next() 方法被调用时，生成器函数开始执行，直到遇到第一个 yield 语句。
  3. 暂停执行并产生值：当生成器函数遇到 yield 语句时，函数的执行会暂停，并将 yield 后面的值作为 next() 方法的返回值。
  4. 恢复执行：当再次调用迭代器的 next() 方法时，生成器函数会从上一次暂停的位置继续执行，直到再次遇到下一个 yield 语句或函数结束。
  5. 迭代终止：当生成器函数执行完成并遇到 return 语句时，迭代器的 next() 方法会返回一个具有 value 和 done 属性的对象，其中 value 是 return 语句指定的值，done 为 true。
function* myGenerator() {
  yield 1;
  yield 2;
  yield 3;
  return 4;
}

const iterator = myGenerator();

console.log(iterator.next()); // { value: 1, done: false }
console.log(iterator.next()); // { value: 2, done: false }
console.log(iterator.next()); // { value: 3, done: false }
console.log(iterator.next()); // { value: 4, done: true }
console.log(iterator.next()); // { value: undefined, done: true }

在这个示例中，生成器函数 myGenerator 使用 yield 语句产生了一系列的值。每次调用迭代器的 next() 方法时，生成器函数会执行到下一个 yield 语句，并返回相应的值。当生成器函数执行完毕时，迭代器的 done 为 true。

Module

模块（Module）是指将代码划分为独立且可重用的单元，每个模块都有自己的作用域，并且可以导出和导入功能以实现模块之间的交互和共享代码。模块化是一种组织和管理代码的方法，使得大型应用程序更易于开发、维护和扩展。

通过使用 import 和 export 关键字来实现模块的导入和导出

export

命名导出（Named Exports）：通过指定导出的名称，将特定的功能导出为模块的公共接口

// 导出一个函数
export function sayHello(name) {
  console.log(`Hello, ${name}!`);
}

// 导出一个常量
export const PI = 3.14159;

// 导出时重命名
export { sayHello as multiplyNumbers };

默认导出（Default Export）：通过 default 关键字，将模块的默认功能导出。每个模块只能有一个默认导出

// 导出默认功能
export default function sayHello(name) {
  console.log(`Hello, ${name}!`);
}

import

  • import命令具有提升效果，会提升到整个模块的头部，首先执行
  • 由于import是静态执行，所以不能使用表达式和变量，这些只有在运行时才能得到结果的语法结构。 是静态引入 一定会执行的引入的
  • import模块只导入一次,并且执行对应文件

命名导入（Named Imports）：通过指定要导入的名称，将其他模块导出的功能导入到当前模块中

// 导入单个功能
import { add } from './myModule.js';

// 导入多个功能
import { PI, sayHello } from './myModule.js';

// 导入并重命名
import { multiply as multiplyNumbers } from './myModule.js';

// 导入所有导出的功能（命名导入的方式）
import * as myModule from './myModule.js';

默认导入

import sayHello from './myModule.js';

sayHello('John'); // 输出: Hello, John!

动态导入

import('./1.js').then(res => {   console.log(res) })
或
let module = await import('./module2.js');
