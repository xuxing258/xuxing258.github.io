浮动

浮动（float）是一种布局属性，用于控制元素在其容器中的位置。通过设置元素的浮动属性，可以使其脱离文档流，向左或向右浮动，从而使周围的内容环绕在其周围

浮动样式最初是作为一种解决多列布局问题的替代方法而引入的

浮动样式的原理

  • 普通文档流

    浏览器在默认情况下规定一个块元素在父元素内的排列规则：

    • 从上往下排列
    • 从左开始排列
    • 一个块元素占一行

  • 浮动之后的文档流

    • float: left | right ; 设置浮动
    • 元素被设置为浮动时, 它会从正常的文档流中脱离出来，然后根据浮动属性的设置向左或向右浮动到容器的左侧或右侧, 也就是说它不再占据正常的布局空间
    <style>
    .wrap{width:800px; height:400px; border:10px solid blue;}

    .box1{ width:250px; height:80px; background-color:#ed7d31;}
    .box2{width:400px; height:100px; background-color:#70ad47;}
    .box3{width:200px; height:200px; background-color:#7030a0;}
    .flo{ float:left }
</style>
    <div class="wrap">
        <div class="box1 flo"></div>
        <div class="box2"></div>
        <div class="box3"></div>
  </div>

当一个元素被设置为浮动后，其后面的元素会自动填充，是因为浮动元素脱离了正常的文档流，其他元素会忽略浮动元素的位置而进行布局

浮动特点

  • 元素脱离文档流：当一个元素被设置为浮动时，它会脱离正常的文档流
  • 文字环绕效果：浮动元素不仅可以影响块级元素的布局，还可以影响行内元素（如文本）。文本内容会环绕在浮动元素的周围，形成文字环绕的效果。
  • 浮动元素的位置将由浮动属性的设置决定。
    • 按照指定方向移动，遇到父级或者相邻的浮动元素会停止不动
    • 上一个兄弟元素是普通文档流元素,那么这个浮动元素相对垂直位置不变
    • 相邻的浮动元素会并排成一行
  • 使行内元素具有宽高。支持margin 相当于使用了display:block; 不支持auto
  • 浮动元素所在位置父级容器宽度不足时,将会换行显示

清除浮动

clear 用于浮动和非浮动元素

当元素设置clear时: 某个方向不能有浮动元素, 如果指定方向有浮动元素时本元素会自动换行

clear取值： left 清除左浮动 | right 清除右浮动 | both 清除所有浮动

  • left: 不允许元素的左侧有浮动元素 只会对float:left生效
    • 对于设置float:right的浮动元素无效
  • right: 不允许元素的右侧有浮动元素 只会对float:right生效
    • 对于设置float:left的浮动元素无效
  • both: 不允许元素的左侧或右侧有浮动元素 只会生效一个

注意: 清除浮动不代表元素没有浮动功能, 清除浮动是对应方向不存在浮动元素3

<div class="box">
    <div class="box1"></div>
    <div class="box2"></div>
    <div style="clear: both;">45</div>
    <div class="box3"></div>
    <div class="box4"></div>
</div>

高度塌陷

当父级元素没有设置高度时: 高度塌陷（Height Collapse）是指当一个容器中的所有子元素都设置为浮动元素时，容器的高度无法自动撑开以适应子元素的高度，导致容器的高度变为0或非常小

解决高度塌陷方法：

  • 方法1 -- 子元素加clear

    在浮动元素后面加一个空的子元素，并给其CSS属性clear；

    <div style="clear:both"></div

  • 方法2 -- 父元素加宽高

    直接规定父元素的宽高，就不存在塌陷问题了。缺点是，得手动计算合适的宽高，并且扩展性不好。

  • 方法3 -- 父元素利用伪元素after

    当前最主流的方式，利用伪元素，相当于给父级多加了一个子元素，同时我们可以将样式公用，所有需要清除浮动的父元素加上一个相同的class名clear就可以解决：

    .container::after{ content: ""; display: block; clear:both; }

    单标签可以设置伪元素但是没有效果, 单标签没有正文内容

  • 方法3 -- 父元素BFC(Block formatting context)化

    父元素满足下列条件之一即可：

    • float属性不为none
    • position不为static和relative
    • overflow不为visible（较常用，通常设置overflow : hidden来解决高度塌陷）
    • display为inline-block / table-cell / table-caption / flex / inline-flex

BFC

BFC（Block Formatting Context）是CSS中的一个概念，用于描述一个独立的块级格式化上下文。BFC是一个对元素内容进行布局和渲染的区域，它具有一些特定的规则和属性，这些规则和属性可以影响元素及其周围内容的布局和显示

触发BFC会在页面生成一个隔离的独立容器，容器里面的子元素不会影响到外面的元素。反之也如此。

当一个元素形成BFC时，它的浮动子元素会被包裹在BFC内部，使得父元素的高度可以正确计算

补充

  • 注意：元素浮动之后，不再支持margin:auto，只支持margin确切的值。

  • 推荐使用浮动来做横向布局而不是inline-block

    line-block布局的基线对齐处理较为麻烦，浮动不用担心这个问题；

    line-block布局空格会被解析显示在页面中，浮动不用担心这个问题。

注意

  • 元素卡主了
    • 拉动父级宽度缩小 ,元素不换到下一行 .高度问题
    • 其中有某个元素高出一点就会卡主元素
  • 浮动与行内块元素区别
    • 不是基线对齐
    • 没有空格解析
