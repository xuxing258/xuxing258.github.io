基础标签

  • <header>：用于表示页面或区块的页眉部分。
  • <footer>：用于表示页面或区块的页脚部分。
  • <nav>：用于表示导航链接的区域。
  • <article>：用于表示独立的文章、帖子或新闻等内容结构。
  • <section>：用于表示文档中的节或段落。
  • <aside>：用于表示与页面主要内容相关但又不是其主要部分的内容。
  • <figure>：定义一段独立的内容，通常包含图片、图表、照片、代码示例等
  • <main>：用于表示文档中的主体内容。
  • <video> : 用于表示视频标签
  • <audio> : 音频标签

全屏

HTML5规定了任何元素都可以全屏显示, 使用元素的requestFullscreen方法来请求全屏显示,

使用全屏API时，不同的浏览器可能有不同的前缀（如moz、webkit、ms），因此需要根据浏览器的兼容性进行适当的前缀处理

Element.requestFullscreen()   // 元素全屏
  • 谷歌浏览器 webkit
  • 火狐浏览器 moz
  • 欧朋 o
  • IE ms
  • Safari浏览器 webkit 内核 -webkit-
Element.webkitRequestFullscreen()  // 全屏

兼容各浏览器

var aBox = document.getElementsByTagName("div")[0]
aBox.onclick = function () {
  if (aBox.webkitRequestFullScreen) {
    aBox.webkitRequestFullScreen()
  } else if (aBox.mozRequestFullScreen) {
    aBox.mozRequestFullScreen()
  } else if (aBox.oRequestFullScreen) {
    aBox.oRequestFullscreen()
  } else {
    aBox.msRequestFullscreen()
  }
}

FileReader 转换成图片

对象允许Web应用程序异步读取存储在用户计算机上的文件（或原始数据缓冲区）的内容，使用 File 或 Blob 对象指定要读取的文件或数据

  • File 文件 => 来源 input 或者 拖拽上等
  • Blob 文件 => 文本或二进制的格式
let rander =new FileReader()  // 构造函数
// 读取指定 file 或 Blob 数据
rander.readAsDataURL(参数)   //参数 = File ||   Blob || 二进制

  • rander.readAsDataURL( ) 开始读取指定的不可变、原始数据的类文件对象一旦完成; 自身中result属性中将包含一个data: URL格式的 Base 64 字符串以表示所读取文件的内容;

    读取操作完成的时候,触发onload事件中,在自身有一个result属性保存着所读文件内容

完整列子

let inpt = document.querySelector("#inpt")
inpt.addEventListener("change",function(event){
  // 获取数据位置
  let value = this.files[0]
  // 创建读取数据函数
  let rander =new FileReader()  // 构造函数
  // 读取指定 file 或 Blob 数据
  rander.readAsDataURL(value)   //参数 = File ||   Blob
  // rander读取成功数据 触发事件
  rander.onload=function(){
    // 创建图片标签
    let oImg = new Image()
    oImg.src = rander.result
    oImg.width = 100;
    oImg.height = 200;
    oImg.onload=function(){
      box.appendChild(oImg)
    }
  }
})

FileReader.readAsArrayBuffer()

开始读取指定的内容Blob，一旦完成，该result属性包含一个ArrayBuffer代表文件的数据。

FileReader.readAsBinaryString()

开始读取指定 的内容Blob，完成后，该result属性包含来自文件的原始二进制数据作为字符串。

FileReader.readAsDataURL()

开始读取指定 的内容Blob，完成后，该result属性包含data:表示文件数据的URL。

FileReader.readAsText()

开始读取指定的内容Blob，完成后，该result属性包含作为文本字符串的文件内容。可以指定可选的编码名称。

blob转换

blob创建一个指向 类型化数组的URL

  let blob =  new Blob([文件])    
  //  参数需要数组格式  文字 || 二进制 ||

  • 构造函数返回一个新的 Blob 对象。 blob的内容由参数数组中给出的值的串联组成;

  • URL.createObjectURL(blob) 静态方法会创建一个 DOMString,

    其中包含一个表示参数中给出对象的URL。这个 URL 的生命周期和创建它的窗口中的 document 绑定,

let inpt = document.querySelector("#inpt")
inpt.addEventListener("change",function(event){
    // 获取数据位置
  let value = this.files[0]
    // 创建读取数据函数 并读取
  let rander =new Blob([value])  // 构造函数
  let qq =URL.createObjectURL(rander)//转换成url格式
  let oImg = new Image()
  oImg.src = qq
  oImg.width = 100;
  oImg.height = 200;
  oImg.onload=function(){
    box.appendChild(oImg)
  }
})

DOMString就是String。规范解释说DOMString指的是UTF-16字符串编码
