认识react

React是一个用于构建用户界面的JavaScript库。它由Facebook开发并开源，是目前最流行的前端框架之一

  1. 英文官网: https://reactjs.org/
  2. 中文官网: https://react.docschina.org/

由Facebook开源

  1. 起初由Facebook 的软件工程师 Jordan Walke 创建
  2. 于2011年部署于Facebook的newsfeed
  3. 随后在2012年部署于Instagram。
  4. 2013年5月宣布开源。
  5. 近十年“陈酿”React 正在被腾讯、阿里等一线大厂广泛使用。

为什么要学

  1. 流行度和广泛应用：React是一个非常流行的JavaScript库，由Facebook开发和维护，被许多大型公司和项目广泛采用
  2. 组件化和可重用性
  3. 虚拟DOM和性能优化
    • React使用虚拟DOM（Virtual DOM）来管理和更新界面的变化，而不直接操作实际的DOM
  4. 单向数据流和状态管理
    • React采用单向数据流的思想，父组件通过props向子组件传递数据，子组件通过回调函数将事件传递给父组件
  5. 跨平台开发：React不仅可以用于Web开发，还可以用于构建原生移动应用（React Native）和桌面应用（Electron）

React的特点

  1. 组件化
    • 组件化是一种软件开发的方法论，旨在将复杂的软件系统拆分成相互独立、可重用和可替换的组件(HTML+css+js)
  2. 使用虚拟DOM+优秀的Diffing算法，尽量减少与真实DOM的交互,找到最小的差异集合，避免不必要的 DOM 更新。这样可以提高性能，减少不必要的计算和渲染开销
  3. 单向数据流
    • React采用单向数据流的思想，父组件通过props向子组件传递数据，子组件通过回调函数将事件传递给父组件。这种数据流的方式使得数据变化的追踪更加清晰，组件之间的依赖关系更加可控，从而简化了状态管理和数据流动的复杂性
  4. JSX语法
    • React引入了JSX语法，它是一种将HTML和JavaScript结合的语法扩展
  5. 数据操作页面
    • 数据变化时,页面发生变化
  6. 声明式编码 (不讲)
    • 声明式编码（Declarative programming）是一种编程范式，通过描述问题的结果或目标，而不是详细指定解决问题的步骤和过程。在声明式编码中，你只需声明期望的结果，由系统自动推导和处理如何实现这些结果
    • 通过封装的函数(Api)来实现目的,

<script>
    const domContainer = document.querySelector('#root');
    // 传入真实DOM节点通过 ReactDOM.createRoot 将 传入的真实DOM节点 渲染到转化成根节点
    const root = ReactDOM.createRoot(domContainer);

API
  • ReactDOM 对象是 react-dom.js 提供的进行 DOM 操作的对象
  • React 对象是react.development.js提供API对象
  • React.createElement(type, [props], [...children])
    • type：表示要创建的元素的类型
    • props：一个包含元素属性的对象
    • ...children：表示元素的子元素,可以是其他 React 元素、文本内容
    • 创建虚拟DOM节点
  • ReactDOM.createRoot(domContainer);
    • domContainer是真实的DOM节点
    • 作用: 在浏览器的 DOM 节点中创建根节点以显示 React 内容
  • root.render(element)
    • element 是 React 生成的虚拟 DOM，
    • 作用: React 的根节点中渲染为 element 节点并显示

虚拟DOM是真实DOM的映射,将真实DOM映射成用树型结构的JS对象来描述真实的DOM结构的信息,树结构的JS对象包含了对应真实DOM结构的信息

  1. 数据处理方式
  • 是对真实 DOM 的一种轻量级内存映射
  1. 虚拟DOM可以提高性能
    • 虚拟 DOM 通过在 JavaScript 内存中进行操作，减少了直接操作真实 DOM 的开销 ,减少浏览器的重绘和回流，从而提高性能
  2. 更新机制
    • 在真实 DOM 中，当数据发生变化时，浏览器会重新计算整个页面的布局和渲染。而在虚拟 DOM 中，React 使用差异化算法来比较前后两个虚拟 DOM 树的差异，并只更新真实 DOM 中实际发生变化的部分。这使得更新过程更加高效
  3. 批量更新
  • 虚拟 DOM 允许进行批量更新，即将多个状态变化合并为单个更新操作。在 React 中，多个状态变化可以合并为一个更新，然后一次性应用到虚拟 DOM 和真实 DOM，减少了重复渲染的次数

jsx

JSX 是一种在 JavaScript 代码中编写类似 HTML 的语法扩展, 浏览器默认是不识别的,需要使用以下语法解析

<!-- 解析JSX语法代码转为JS代码的库 -->
<script src="https://unpkg.com/babel-standalone@6/babel.min.js"></script>
<!-- 使用 JSX，方法是在为其添加 type="text/babel" 属性 -->
<script type="text/babel">  
   	const domContainer = document.querySelector('#root');
    const root = ReactDOM.createRoot(domContainer);
    let VDOM = <h1>Hello,World</h1>

	root.render(VDOM);
</script>

为啥要设置type="text/babel"

为了支持直接在浏览器中使用 JSX 语法，需要使用 Babel 编译器实时将 JSX 转换为普通的 JavaScript 代码。为了告诉浏览器正确处理 JSX，可以在 <script> 标签上设置 type="text/babel" 属性

用途

  1. jsx就是原生js的语法糖, 解决我们写原生js的困难
  2. 通过更加简单和高效，并提供了更好的可读性、可组合性和动态性

JSX语法规则

  1. JSX支持多行（换行），如果需要换行，需使用() 包裹，防止bug出现
  2. 虚拟dom只能存在一个根标签,标签可以相互嵌套，形成层次结构
  3. 定义虚拟dom时,不需要书写引号
  4. 内联样式使用 style={{key:value}}的形式去写
  5. React 元素的样式使用驼峰命名法
  6. 样式属性名：**class → className ** 使用 ClassName定义
  7. 嵌入 JS 表达式 使用 {表达式} ; 字符串模板的写法
    • JavaScript中的表达式和语句是有区别的.
      • 表达式一定会产生一个值,接受值可以放在任何需要一个值的地方
  8. 标签必须闭合 或
    1. <开头的代码, 以标签的语法解析:html同名标签转换为html同名元素,若html中无标签对应的同名元素者报错
    2. 若大写字母开头, react就去渲染对应的组件,若组件没有定义,则会报错

循环

页面的构建离不开重复的列表结构，比如歌曲列表，商品列表等，我们知道vue中用的是v-for，react这边如何实现呢？

  1. 数组能自动遍历,能实现但是会报key错误
    • 用于 React 的虚拟 DOM 算法, 使用 key 时会基于 key 值的变化来渲染, 使用唯一值
  2. 对象不能自动遍历,对象不能做为 react的合法节点
下面这些都是表达式:
    (1). a
    (2). a+b
    (3). demo(1)  
    (4). arr.map() 
    (5). test()

语句(代码)

// 设置初始数据,  如直接使用:会将每一项自动循环展示
let arr = ["楚铮", "Zx", "886", "南一"]     
{arr.map(item => item)} 或  {arr}

// 设置虚拟DOM显示, key报错
arr.map(item => { return <span>{item}</span> })
// 解决key问题
arr.map((item, index) => { return <span key={index}>{item}</span> })
