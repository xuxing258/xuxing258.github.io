路由

单页面

  • 整个应用只有一个完整的页面

  • 点击页面中的链接不会刷新页面，只会做页面的局部更新。

多页面

  • 一个项目是由多个完整的html页面组成，不在有主、分页面。
  • 页面跳转是所有的资源都要重新加载，页面之间的切换会出现卡顿空白的问题，不容易实现切换动画等

简介

  • 通过点击显示对应内容不会发生跳转页面, 通过a标签进行跳转路由
    • 原理是通过浏览器的历史记录, history进行处理的跳转 或 使用浏览器的 URL 锚点（hash）来实现路由
  • 路由就是一组key-value的对应关系。
  • react路由
    • key为路由地址,value是component 用于展示页面内容
    • 工作过程:当浏览器的路径改变时，对应的组件就会显示。
  • 安装npm install react-router-dom

路由原理

路由的模式

  • HashRouter:
    • 基于hash 模式: URL 中的 # 符号后面的部分被视为 hash
      • HashRouter 通过hashchange事件监听浏览器的 hash 变化，并根据变化的 hash 来判断当前应该渲染哪个路由
    • 比较丑：在域名后，先拼接/#，再拼接路径；也就是利用锚点，实现路由的跳转；
  • Browser Router:
    • 基于history模式： 即 pushState、replaceState 和 onpopstate监听方法 ，来管理浏览器的 URL 和历史记录。这使得可以在不加载新页面的情况下，通过更新 URL 来实现路由的变化
  • 路由组件必须在HashRouter与BrowserRouter之中使用

注意: createBrowserRouter路由表 和 createHashRoute路由表 与 RouterProvider 是一体的

注意: useRoutes路由表 与 <HashRouter> 和 <BrowserRouter>是一体的

  • path: 路由地址
  • element: 路由组件

}]);

RouterProvider 路由组件出口

import { RouterProvider } from "react-router-dom";
import router from "./route/index.js";
// 组件中使用
const root = ReactDOM.createRoot(document.getElementById('root'));
root.render( <RouterProvider router={router}> </RouterProvider>);

注意: 后续API(Link, route) 只能写在RouterProvider包裹的组件中才能被调用, 根组件将会是路由组件

创建多路由

  • children: 子路由(嵌套路由)
    • 嵌套路由path设置不需添加"/", 添加会被认为一级路由
    • 二级路由展示需要设置渲染出口, 需要设置Outlet组件
const router = createBrowserRouter([
  {
    path: "/",
    element: <App></App>,
  },
  {
    path: "/about",
    element: <About></About>,
    children: [
      {
        path: "child",
        element: <ChildRen />,
      },
    ],
  },
]);
  • Link组件: to属性设置路由地址
<Link to="/about">关于</Link>
  • Outlet组件: 展示子路由需要设置路由展示出口
<!--About组件 -->
<div>
    <div>About</div>
 	<Link to="/about/child">跳转到二级路由</Link>
	<Outlet />
</div>

动态路由

  • 路由地址本身是不进行改变的, 如/a 只能匹配 /a, 但是动态路由可以匹配/c 或 /d等等, 可以匹配多个
// 创建路由表
const router = createBrowserRouter([
  {
    path: "/about",
    element: <About></About>,
    children: [
      {
        path: ":child",
        element: <ChildRen />,
      },
    ],
  },
]);
  • path: "*", 匹配任意地址,用于404处理
{
   path: "*",
   element: <Hot></Hot>,

编程式导航

允许通过useNavigate() 逻辑代码进行跳转, 返回navigate(url,options)函数

  • url传递一个To值
  • options
    • state:{ 传递数据 }
    • 如果在跳转时不想加历史记录，可以添加额外参数replace为true
      • replace属性用于控制跳转模式（push 或 replace，默认是push）
    • 在历史堆栈中传递进入的历史记录。例如，navigate(-1)相当于点击后退按钮
import { useNavigate } from 'react-router-dom';

export default function About() {
  const navigate = useNavigate();
  const nav = () => { navigate('/about/child')  }
  return ( <div>  <button onClick={nav}>跳转</button> </div> )
}

navigate("detail", { eplace: false })

function forwrad(){  nvavigate(1) }
<button onClick={back}>后退</button>
<button onClick={forwrad}>前进</button>

懒加载

让组件在未被使用时组件不需进行被加, 使用该组件时才会加载

// 该组件是动态加载的, 只有在使用时才会加载
 const Home = lazy(() => import('./child/Home.jsx'))
// 需要配合`Suspense ` 内置组件使用

使用Suspense 组件数据未加载完成显示备用数据

  • fallback 属性 : 真正的 UI 未渲染完成时代替其渲染的备用 UI
// 加载显示
import {  Suspense } from 'react';
<Suspense fallback={<div>Loading...</div>}>
     <Home />
</Suspense>
