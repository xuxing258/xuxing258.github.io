类组件

  1. React.Component 是 React 框架提供的一个基类，用于创建类组件
    • 通过继承 React.Component 类，并实现一些特定的方法和属性来定义组件的行为
  2. 类组件必须提供 render 方法render 方法必须有返回值，表示该组件的 UI 结构
  3. 组件中render方法中的this为组件实例对象
const root = ReactDOM.createRoot(document.getElementById('root'));

//类组件  类组件继承 React.Component 父类，从而使用父类中提供的方法或属性
class MyComponent extends React.Component {
    
    // render是需要渲染的虚拟DOM节点和数据  其中可以进行处理数据
    render() {
        // render中的this是谁? —MyComponent的实例对象。
        console.log(this);
        return <h1>成功</h1>
    }
}
root.render(<MyComponent1 />);

核心属性state

通过this.setState覆盖原来的属性; 重新调用render函数; 传递对象或函数;

class MyComponent extends React.Component {
    state = { bol: true };

    // 事件
    handlerClick() {  // 不能执行this指向出现问题  没有绑定指向
        let { bol } = this.state
        
        // 代码不会重新渲染组件：
        this.state.bol = 'Hello';
        
        // 注意修改必须通过setState进行修改
        this.setState({ "bol": !bol })
    }
	
	// 正确版本
	 handlerClick = () => { //  箭头函数本身没有this,借助外部this
        let { bol } = this.state
        // 注意修改必须通过setState进行修改
        this.setState({ "bol": !bol })
     }

    render() {
        return (
        	<div onClick={this.handlerClick}>
            	今天我正在学习: {this.state.bol ? "vue" : "react"}
			</div>
        )
    }
}
root.render(<MyComponent />);

核心属性refs

  1. Ref 是使用属性创建React.createRef()并附加到 React 元素的属性中
    1. 调用之后返回一个容器,该容器可以存储被ref所识别的节点
  2. 当 ref 属性用于 HTML 元素时，构造函数中使用 React.createRef()创建的 ref 接收底层 DOM 元素作为其 current 属性
  3. 当 ref 属性用于自定义 class 组件时，ref 对象接收组件的挂载实例作为其 current 属性
class Demo extends React.Component {
