组件

组件是构建用户界面的基本构造块。组件是可重用和独立的代码单元，用于封装特定的功能和UI元素。通过组合和嵌套组件，可以构建复杂的用户界面

定义组件

  1. 当 组件被渲染时，React 会自动调用函数组件，并将其返回的 JSX 元素插入到 App 组件的渲染结果中
  2. 函数组件中this为undefined
    • 开启严格模式 use strict;
const root = ReactDOM.createRoot(document.getElementById('root'));

核心属性state

state是一个特殊的属性，用于存储和管理组件内部的数据。状态是一个可变的对象，当状态发生变化时，React 会自动重新渲染组件，并更新相应的 UI

useState参数

const [state, setState] = useState(initialState);

  • initialState： state 初始化的值可以是任何类型的值
    • 如果传递函数作为 initialState，则它将被视为 初始化函数
  • useState 返回一个由两个值组成的数组
    • 当前的 state。在首次渲染时，它将与你传递的 initialState 相匹配
    • 返回的 set 函数允许你将 state 更新为不同的值并触发重新渲染

useState初步使用

用于在函数组件中定义和管理数据

  1. useState 的初始值(参数)只会在组件第一次渲染时生效
    • useState 以后的每次渲染，useState 获取到都是最新的状态值，React 组件会记住每次最新的状态值
const {useState} = React;

function MyComponent() {
    let [data,setData] = useState(0)
    function handleClick() {
        setData(data + 1)
    }
    return (
        <div>
            <button onClick={handleClick}>Click me --- {data}</button>
    	</div>
    );
}

useState更新过程

函数组件使用useState后的执行过程，以及状态值的变化

  • 组件第一次渲染
    1. 从头开始执行该组件中的代码逻辑
    2. 调用 useState(0) 将传入的参数作为状态初始值，即：0
    3. 渲染组件，此时，获取到的状态 count 值为： 0
  • 组件第二次渲染
    1. 点击按钮，调用 setCount(count + 1) 修改状态，因为状态发生改变，所以，该组件会重新渲染
    2. 组件重新渲染时，会再次执行该组件中的代码逻辑
    3. 再次调用 useState(0)，此时 React 内部会拿到最新的状态值而非初始值，比如，该案例中最新的状态值为 1
    4. 再次渲染组件，此时，获取到的状态 count 值为：1

  • 注意: data数据在重新渲染之后才会拿到最新值

更改state的对象和数组

  • 将对象和数组放入状态中。在 React 中，状态被认为是只读的，因此 你应该替换它而不是改变现有对象
const [person, setPerson] = useState({
    title: 'Blue Nana',
    city: 'Hamburg',
})

function MyComponent() {
    const [person, setPerson] = useState({
        title: 'Blue Nana',
        city: 'Hamburg',
    })

    function handleClick() {
		person.title = 'Taylor';
        // 覆盖
        setPerson({
          ...person,
          title: 'Taylor'
        });
    }
    return (
        <div>
            <button onClick={handleClick}>Click me --- {person.title} --</button>
    	</div>
    );
}

核心属性props

组件的 props 是组件之间传递数据的一种机制。通过 props，您可以将数据从一个组件传递到另一个组件，并在接收组件中使用这些数据

  • 在父组件中，可以将数据作为属性传递给子组件。子组件可以通过 props 对象来访问传递的属性值
  • props 是只读的，即子组件无法直接修改父组件传递的属性值
  • props可以设置默认值
  • 传递虚拟DOM节点
// 子组件 
function MyChild(props){  return ( <p>  子组件  </p> ) }
// 向子组件传递数据
function MyComponent() {
    const [obj,setObj] = useState({q:1,w:2,e:3})
    return (<div> <MyChild  name="路遥" age="18" obj={obj}/> </div>  );
}

// 子组件根据数据进行修改    
function MyChild({name,age = 100}){
    const [name1,setName1] = useState(name)
    const handlerClick = ()=>{  setName1("哈哈") }
    return (
        <p>
        	子组件--{name1}
        	<button onClick={handlerClick}>点击</button>
        </p>
	)
}

    • 共用同一组件, 通过传递不同的参数, 组件展示的效果(内容)不相同,
    • 共用同一组件,通过传递不同的参数或插槽, 组件展示的效果内容不相同,
