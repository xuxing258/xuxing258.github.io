nexTick

Vue 在更新 DOM 时是异步执行的。当你修改了响应式数据，Vue 不会立刻更新 DOM，而是将这些修改放进一个队列里，等到所有执行完毕时，再批量更新 DOM。这样做的目的是为了提升性能，避免频繁的 DOM 操作

nextTick 接收一个回调函数作为参数，并将这个回调函数延迟到页面DOM数据更新后才执行

使用场景：想要操作 基于最新数据生成的DOM 时，就将这个操作放在 nextTick 的回调中

<button @click="numFun">点击++</button>
<div class="dagda" ref="qwe" :contenteditable="num"></div>

<script>
    numFun(){
        this.num = !this.num
        this.$refs.qwe.focus() // 不会成功 修改模板的时候,没有挂载到页面
        
        setTimeout(()=>{   //成功,挂载之后在执行
          this.$refs.qwe.focus()
        },200)
        
        this.$nextTick(function(){  // DOM更新结束后执行
           this.$refs.qwe.focus()
        })
    },
</script>

过渡与动画

Transition 组件 是 Vue.js 提供的一个内置组件,

  • <Transition>用于在元素或组件的 进入/离开 时添加动画效果。它可以帮助你轻松实现淡入淡出、滑动、缩放等动画效果，适用于单个元素或组件的过渡场景

Transition组件

Transition 组件通过包裹需要动画的元素或组件，自动在元素的进入和离开时添加动画类 。进入或离开可以由以下的条件之一触发：

  • 由 v-if 所触发的切换
  • 由 v-show 所触发的切换
  • 仅支持单个元素或组件作为其内容
  • 由特殊元素 <component> 切换的动态组件
  • appear属性: 元素在初始渲染时就应用过渡或动画效果

添加动画效果实现

<!--实现隐藏显示,但没有动画效果-->
<button @click="bol=!bol">显示/隐藏</button>
<h1 v-show="bol">准备的动画</h1>

<button @click="bol = !bol">显示/隐藏</button>
<transition>
    <!-- 通过布尔值进行控制 -->
    <h1 v-show="bol">准备的动画</h1>
</transition>

<!-- 初始化时出现动画效果,执行进入时的效果 -->
<transition appear>
    <h1 v-show="bol">准备的动画</h1>
</transition>


<script>
 bol: true,
</script>

<style scoped>
  /* 定义进入生效动画 */    /* 进入的过程中, */
  .v-enter-active{
    animation: moveLogo 1s;
  }
    
  /* 定义离开时生效动画 */    /*离开的过程中 */
  .v-leave-active{
    animation: moveLogo 1s reverse;
  }

        transform: translateX(-100%);

</style>

过渡效果

<button @click="bol = !bol">显示/隐藏</button>

    }

    /*离开的起点*/

        transform: translateX(0);
    }
    /*离开的终点*/
    .hello-leave-to{
        transform: translateX(-100%);
    }
</style>

TransitionGroup组件

TransitionGroup 是一个内置组件 设置多元素过度会在一个 v-for 列表中的元素或组件被插入，移动，或移除时应用动画

  1. 可容纳多个元素设置不同的效果
  2. 列表中的每个元素都必须有一个独一无二的 key , 其他的无所谓
  3. 使用v-show设置过渡效果会出现问题, v-if没有问题
<button @click="bol = !bol">显示/隐藏</button>
    
<!-- vue3会报错  仅支持单个元素或组件作为其插槽内容-->
<transition name="hello" appear>
    <h1 v-show="bol" key="1">第一个</h1>
    <h1 v-show="bol" key="2">第二个</h1>
</transition> 

<!-- 解决问题 需要设置key属性-->
<transition-group name="hello" appear>
    <h1 v-show="bol" key="1">第一个</h1>
    <h1 v-show="bol" key="2">第二个</h1>
</transition-group>


<!-- 这样还是单元素,效果是单一的-->
<transition name="hello" appear>
    <div v-show="bol">
        <h1>第一个</h1>
        <h1>第二个</h1>
    </div>
</transition>

<!-- 设置不同的效果,展示不同的内容 -->
<transition-group name="hello" >
    <h1 v-for="item in arr" :key="item">{{ item }}</h1>
</transition-group>
<script>
methods: {
    hander(){
      this.num++
      this.arr.push(this.num)
    },
    hander123(){
      this.arr.splice(2,1)
    }
  }
</script>

<style scoped>
  h1{
    width: 300px;
    height: 50px;
    line-height: 50px;
    text-align: center;
    background: green;
  }
  /* 效果相同可以进行合并css代码 */
  .hello-enter-from,.hello-leave-to{
    transform: translateX(-100%);
  }
  /* 进入的过程中,离开的过程中 */
  .hello-enter-active,.hello-leave-active{
    transition:all  1s linear;
  }
  .hello-enter-to,.hello-leave-from{
    transform: translateX(0);
  }
</style>

js钩子

这些钩子可以与 CSS 过渡或动画结合使用，也可以单独使用。

<Transition
  @before-enter="onBeforeEnter"
  @enter="onEnter"
  @after-enter="onAfterEnter"
  @enter-cancelled="onEnterCancelled"
  @before-leave="onBeforeLeave"
  @leave="onLeave"
  @after-leave="onAfterLeave"
  @leave-cancelled="onLeaveCancelled"
>
  <!-- ... -->
</Transition>

export default {
  // ...
  methods: {
    // 在元素被插入到 DOM 之前被调用
    // 用这个来设置元素的 "enter-from" 状态
    onBeforeEnter(el) {},

    // 在元素被插入到 DOM 之后的下一帧被调用
    // 用这个来开始进入动画
    onEnter(el, done) {
      // 调用回调函数 done 表示过渡结束
      // 如果与 CSS 结合使用，则这个回调是可选参数
      done()
    },

    // 当进入过渡完成时调用。
    onAfterEnter(el) {},
    onEnterCancelled(el) {},

    // 在 leave 钩子之前调用
    // 大多数时候，你应该只会用到 leave 钩子
    onBeforeLeave(el) {},

    // 在离开过渡开始时调用
    // 用这个来开始离开动画
    onLeave(el, done) {
      // 调用回调函数 done 表示过渡结束
      // 如果与 CSS 结合使用，则这个回调是可选参数
      done()
    },

    // 在离开过渡完成、
    // 且元素已从 DOM 中移除时调用
    onAfterLeave(el) {},

    // 仅在 v-show 过渡中可用
    onLeaveCancelled(el) {}
  }
}

动画和过渡总结

  1. 图示

  2. 写法:

    1. 准备好样式

      • 元素进入的样式
        • v-enter-from: 进入的起点
        • v-enter-active: 进入过程中
        • v-enter-to: 进入的终点
      • 元素离开的样式
        • v-leave-from:离开的起点
        • v-leave-active:离开过程中
        • v-leave-to:离开的终点

    2. 使用<transition>包裹要过度的元素，并配置name属性:

      <transition name="hello">
	<h1 v-show="isShow">你好啊!</h1>
</transition>

    3. 备注:若有多个元素需要过度，则需要使用:<transition-group>，且每个元素都要指定 key值。

vue/cli

vue.config.js配置单个:

devServer :{
	proxy : "http://localhost:5000"
}
  • 优点:配置简单，请求资源时直接发给前端(8080)即可
  • 缺点:不能配置多个代理，不能灵活的控制请求是否走代理。
  • 工作方式:若按照上述配置代理，当请求了前端public中不存在的资源时，那么该请求会转发给服务器（优先匹配前端资源)
axios.get("/qqq")

vue.config.js配置多个

//配置多个的时候  一定要配置pathRewrite替换代理名    将代理名替换成字符串
module.exports = {
    devServer: {
        proxy: {
            '/api': {  //代理名
                target: 'http://localhost:3000',  //代理目标地址
                ws: true,             // webSocket 实时通讯
                secure: false,        //  如果是https接口，需要配置这个参数
                changeOrigin: true,   //  是否跨域
                pathRewrite: {        //  /api 替换
                  '^/api': ''
                }
            },
            '/dage': {
                target: 'http://localhost:6000',  //代理目标地址
                secure: false,        //  如果是https接口，需要配置这个参数
                changeOrigin: true,   //  是否跨域
                pathRewrite: {        //  /api 替换
                  '^/dage': ''
                }
            },
        }
}
}

  • 优点:可以配置多个代理，且可以灵活的控制请求是否走代理。

  • 缺点:配置略微繁琐，请求资源时必须加前缀。

插槽slot

  • 我们已经了解到组件能够接收任意类型的 JavaScript 值作为属性，但组件要如何接收模板内容呢
  • slot 元素是一个插槽出口 (slot outlet)， 标示了父元素提供的插槽内容 (slot content) 将在哪里被渲染。
<FancyButton>
  Click me! <!-- 插槽内容 -->
</FancyButton>

<!--组件-->
<button class="fancy-btn">
  <slot></slot> <!-- 插槽出口 -->
</button>

具名插槽

  • 组件中可设置多处插槽出口, 组件通过 slot的name 属性设置为具名插槽
  • 为具名插槽传入内容，在传递模版处通过 template元素设置v-slot 指令，并设置目标插槽的名字
    • v-slot 简写 #，因此 <template v-slot:header> 可以简写为 <template #header>
<BaseLayout>
  <template v-slot:header>
    <!-- header 插槽的内容放这里 -->
  </template>
</BaseLayout>

<div class="container">
  <header>
    <slot name="header"></slot>
  </header>
  <main>
    <slot></slot>
  </main>
  <footer>
    <slot name="footer"></slot>
  </footer>
</div>

作用域插槽

插槽内容可以访问到父组件的数据作用域，因为插槽内容本身是在父组件模板中定义的

  1. 插槽内容无法访问子组件的数据。
  2. Vue 模板中的表达式只能访问其定义时所处的作用域
  3. 作用域插槽: 插槽内容能够访问子组件中的数据

插槽的内容可能想要同时使用父组件域内和子组件域内的数据

(在slot上设置进行传递数据,将子组件数据发送到插槽处使用)

<!-- <MyComponent> 的模板 -->
<div>
  <slot :count="1">默认插槽</slot>  
  <slot name="cente" :text="greetingMessage">具名插槽</slot>
</div>

<!-- 默认插槽通过组件  v-slot=slotProps   slotProps可以随便定义 -->
<MyComponent v-slot="slotProps">
  {{ slotProps.text }} {{ slotProps.count }}
</MyComponent>


<!-- 具名插槽通过tempalte  #name名称=hero  hero可以随便定义 -->
<MyComponent>
  <template #cente="hero">
		{{hero}}
    </template>
</MyComponent>
