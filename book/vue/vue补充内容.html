Vue 的响应式更新精确到组件级别

Vue 对于响应式属性的更新，只会精确更新依赖收集的当前组件，而不会递归的去更新子组件，这也是它性能强大的原因之一

<template>
   <div>
      {{ msg }}
      <ChildComponent />
   </div>
</template>

<!--  
我们在触发 this.msg = 'Hello, Changed~'的时候，会触发组件的更新，视图的重新渲染。
但是 <ChildComponent /> 这个组件其实是不会重新渲染的，这是 Vue 刻意而为之的
-->

vue(响应式+依赖收集) = 你把你要渲染的数据放在data里，需要更新那些组件的心智负担交给我，我知道那些组件需要更新。 react(setXXX setState) = 你只要调用这个两个方法，我就帮你从父到子的递归渲染，那些组件不需要渲染我把开关给你了(shouleComponentUpdate memo)，你自己看着办
