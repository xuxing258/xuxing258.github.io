Vuex

Vuex 是什么？

Vuex 是一个专为 Vue.js 应用程序开发的状态(数据)管理模式。它采用集中式存储管理应用的所有组件的状态(任意组件通信)，并以相应的规则保证状态以一种可预测的方式发生变化.

  1. Vuex 的状态存储是响应式的。当 Vue 组件从 store 中读取状态的时候，若 store 中的状态发生变化，那么相应的组件也会相应地得到高效更新
  2. 你不能直接改变 store 中的状态。改变 store 中的状态的唯一途径就是显式地提交 (commit) mutation。

任意组件通信

Vuex工作原理

  1. state：页面状态管理容器对象。集中存储Vue components中data对象的零散数据，全局唯一，以进行统一的状态管理。页面显示所需的数据从该对象中进行读取，利用Vue的细粒度数据响应机制来进行高效的状态更新。
  2. render: state数据渲染页面
  3. Vue Components：Vue组件。展示页面，负责接收用户操作等交互行为，执行dispatch方法触发对应action进行回应。
  4. dispatch：操作行为触发方法。将组件设置的数据发送给actions
  5. actions：处理Vue Components接收到的所有交互行为,包含同步/异步操作,
  6. commit：状态改变提交操作方法。将actions接收或处理后数据发送给mutations
  7. mutations：状态改变操作方法。该方法进行同步操作, 它会接收到一个事件 ==> 事件中参数一个state,一个cations数据, 然后进行操作修改state
    vue允许在组件内调用commit 直接执行mutations去修改state内容,
  8. Mutate: 修改state数据
  9. Backend API 后端API 也需要dispatch进行触发,应为是请求所以是异步,
  10. Devtools 检测工具与mutations进行对应
  11. dispatch commit state actions mutations 都是$store的内容

vuex

actions

actions负责执行同步和异步逻辑并将数据发送给mutations

const actions = {
    //接受一个与 store 实例具有相同方法和属性的 context 对象 与 dispatch发送的数据
    jia(context,value){
        context.commit("JIA",value)
    },  
    // 同步逻辑过大,负载均衡
    jian(context,value){
        let ss = 10;
        ss += value
        context.dispatch("JIANJIAN",ss)
    },
    JIANJIAN(context,value){
        context.commit("JIAN",value)
    }
}

mutations

  • mutations 中最好不要执行代码逻辑, 避免混乱,
  • 通过actions执行逻辑, mutations负责处理state数据
  • mutations 中可以执行异步,但是不推荐,需要符合规范逻辑
const mutations = {
    //变量命名大写
    JIA(state,value){
        setTimeout(() => {
            state.num += value
        }, 1000);
    },
}
// 能修改成功, 但是不利于观察数据,最好不要使用---避免出现不知道神马问题,符合规范;

        ...mapGetters(["newAge"])

     mounted(){
         //演示: mapState({计算属性函数名:vuex中state属性名})  返回一个对象

模块

  • 由于使用单一状态树，应用的所有状态会集中到一个比较大的对象。当应用变得非常复杂时，store 对象就有可能变得相当臃肿
  • 默认情况下，模块内部的 action 和 mutation,Getter是注册在全局命名空间的——这样使得多个模块能够对同一个 action 或 mutation 作出响应
    • Getter 同样也默认注册在全局命名空间
    • 多个模块中出现相同的处理函数名称, 触发时将会全部会触发
const moduleA = {
  namespaced: true,     // 设置  设置命名空间进行分模块
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... },
  getters: { ... },
}

const moduleB = {
  namespaced: true,   
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... }
}

const store = createStore({
  modules: {
    a: moduleA,
    b: moduleB
  }
})

store.state.a // -> moduleA 的状态
store.state.b // -> moduleB 的状态

// 未设置命名空间时 action 和 mutation使用 
this.$store.dispatch("函数","数据")
this.$store.commit("函数","数据")

命名

  • 模块具有更高的封装度和复用性( action 和 mutation )，通过添加 namespaced: true 的方式使其成为带命名空间的模块

  • 通过对应的命名进行操作对应模块的的数据和方法

  • 当模块被注册后，它的所有 getter、action 及 mutation 都会自动根据模块注册的路径调整命名

this.$store.commit("conutAbout/JIA",1)  
this.$store.dispatch("conutAbout/setData",10)
this.$store.getters["showAbout/computedArr"]

this.$store.state.conutAbout.num

actions请求数据

// 配置store核心文件
import axios  from "axios"

const actions = {
    jia(context){
        axios.get("数据地址").then(( {data} )=>{
            // 设置成功信息
            context.commit("JIA",data.result.songs)
        }).catch((error)=>{
            //设置失败信息
            context.commit("ERROR",error)
        })
    }
}

const mutations = {
    JIA(state,value){
        state.num = value
    },
    ERROR(state,value){
        //自己看怎么定义报错信息 或者其他的操作行为
    },
}
