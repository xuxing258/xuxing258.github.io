ref

  • 获取元素或子组件注册引用信息
  • 在普通的 DOM 元素上使用，引用指向的就是 DOM 元素在子组件上，引用就指向组件实例
  • 使用方法
    • 打标记
      • <div ref="xxx">替换内容</div> 或者 <Fruit ref="xxx"></Fruit>
    • 获取
      • this.$refs.xxx

expose

  • expose 选项值应当是一个包含要暴露的属性名称字符串的数组。
  • 当使用 expose 时，只有显式列出的属性将在组件实例上暴露
export default {
  // 只有 `publicMethod` `msg` 父组件才能获取使用
  expose: ['publicMethod',"msg"],
    data(){
        return :{
            msg: "123",
            you: "youyou"
        }
    }
  methods: {
    publicMethod() {
      // ...
    },
    privateMethod() {
      // ...
    }
  }
}

props

  • props 可以是数组或对象，用于接收来自父组件的数据
  • props 可以是简单的数组，或者使用对象作为替代，对象允许配置高级选项，如类型检测、自定义验证和设置默认值

props声明

  • 一个组件需要显式声明它所接受的 props，这样 Vue 才能知道外部传入的哪些是 props
  • 接收静态数据是以字符串格式
<!--想要在watermelon里面给年龄加1或其他数字-->
<Watermelon name="夏栀" sex="女" age="18"></Watermelon>

<!--直接在年龄中加数字,会被拼接-->
<p>年龄--{{age + 10}}</p> <!--显示成 年龄1810 或 1910 或 2010 不会被计算-->

<!--打开检测工具查看props内容是一个字符串,传值的时候就是一个字符串-->
<p>年龄--{{Number(age)+10}}</p> <!--会被计算成功,但是不很优秀-->
      
<!--在传值时进行绑定传值;传值形成js表达式,就可以传递其他类似-->
<p>年龄--{{age+10}}</p>

<!-- 传递动态数据 -->
<Watermelon name="夏栀" sex="女" :age="18"></Watermelon>

<script>
export default {
    name:"Watermelon",
    //接收父级传入的数据  数组格式必须是字符串
    props:["name","age","sex"],
}
</script>

单向数据流

  • props 都遵循着单向绑定原则，props 因父组件的更新而变化，自然地将新的状态向下流往子组件，而不会逆向传递;
    • 这避免了子组件意外修改父组件的状态的情况，不然应用的数据流将很容易变得混乱而难以理解
  • 父组件修改传值props,子组件也会发生变化重新渲染
    • 修改父组件其他数据,不会让子组件重新触发; 只有修改传递的props数据才会触发
<Watermelon  :eldestBrotherGood="msg"></Watermelon>
<button @click="addChange">点击</button>

<script>
    methods:{
      addChange(){
        this.msg="点击父组件内容修改子组件的内容"
      }
    },
    data(){
      return {
        msg : "hello Vue"
      }
    }
</script>

子组件修改props数据

  • 修改能成功,但是会提示报错信息,
  • 子组件更新会重新渲染,因为修改的props数据是父组件传递的,也会导致父组件更新,父组件更新props数据又会导致子组件更新渲染;
    • 很容易变得混乱而难以理解
<!--动态传值-->
<Watermelon  :eldestBrotherGood="msg"></Watermelon>

<!--子组件-->
<p>{{eldestBrotherGood}}</p>
<button @click="addChange">点击</button>

<script>
    props:{
      eldestBrotherGood:String,
    },
    methods:{
      addChangeEldest(){
   
        this.eldestBrotherGood = "修改" 
      }
    }
</script>

解决子修改问题

  • prop 被用于传入初始值；而子组件想在之后将其作为一个局部数据属性。
  • 在这种情况下，最好是新定义一个局部数据属性，从 props 上获取初始值即可
export default {
  props: ['initialCounter'],
  data() {
    return {
      // 计数器只是将 this.initialCounter 作为初始值
      // 像下面这样做就使 prop 和后续更新无关了
      counter: this.initialCounter
    }
  }
}
  • 需要对传入的 prop 值做进一步的转换。在这种情况中，最好是基于该 prop 值定义一个计算属性：
export default {
  props: ['size'],
  computed: {
    // 该 prop 变更时计算属性也会自动更新
    normalizedSize() {
      return this.size.trim().toLowerCase()
    }
  }
}

对象数组操作

当对象或数组作为 props 被传入时，虽然子组件无法更改 props 绑定，但仍然可以更改对象或数组内部的值。这是因为 JavaScript 的对象和数组是按引用传递，而对 Vue 来说，禁止这样的改动虽然可能，但有很大的性能损耗，比较得不偿失,

你应该尽可能避免这样的更改，除非父子组件在设计上本来就需要紧密耦合。在大多数场景下，子组件应该抛出一个事件来通知父组件做出改变。

provide

功能: 用于提供为后代组件注入值

  • provide 选项应当是一个对象或是返回一个对象的函数。这个对象包含了可注入其后代组件的属性
    • 父组件不需要知道哪些子组件使用它 provide 的属性
    • 子组件不需要知道 inject 的属性来自哪里
  • provide 不是数据响应式:
    • 祖先组件传递响应式数据: 子组件---孙子组件不会发生变化
  • provide 书写
    • 对象格式
    • 函数格式
  • inject 接收一个字符串数组，或一个对象
    • inject:['属性名'],
    • inject:{ key 是本组件的绑定名 : value 是接收祖先属性名 },
  • provide与组合式computed使用可转化成响应式
<script>
 import { computed } from 'vue'
  export default {
    name:"App",  
    data(){
      return {
        msg:{
            qwe:"夏栀"
        },
      }
    },
     // 向后代传值  不能绑定数据进行更改,也就是说不能使用this.user
      provide: {
        user: 'app组件值',
      },  
      // 使用函数的形式，可以访问到 `this`
      provide() {
        return {
            user: this.num
        }
      },
      // 响应式数据  this.qwe数据发生变化 后代数据发生变化
      provide(){
        return {
          info: computed(() => this.qwe)
        }
      }
  }
</script>

<!--NavChild 组件-->
<script>
    export default {
    	name："BaseAppChild",
        //后代接收到值
        inject: ['user'],
        // user 接收到数据 后续祖组件数据变化 本组件不会发生变化
    }
</script>

mixin混入

  • 混入是一种用于组件代码复用的机制。它允许你将组件的选项（如 data、methods、computed、生命周期钩子 等）抽离出来，然后在多个组件中复用。Mixin 可以有效地减少代码重复，提高代码的可维护性
  • 特点：
    • 可以定义一组通用的逻辑，然后在多个组件中混入。
    • 如果混入的选项与组件本身的选项冲突，组件本身的选项会优先
  • 通过 mixins 选项将 Mixin 混入组件中
// mixin.js 文件  混入文件内容
export default {
    data(){
        return {
            mixinName:"我是混入数据",
            name:"细心"
        }
    },
    methods:{
        addHandleChild(){
            console.log("处理点击数据");
        }
    }
}
// 单个选项混合
export const hunhe1 = { }

局部混入

  • 当组件和混入对象含有同名选项时,并在发生冲突时以组件数据优先,
  • 同名钩子函数将合并为一个数组，因此都将被调用, 另外混入对象的钩子将在组件自身钩子之前调用。
<script>
import mixin from "./mixin"
mixins:[mixin],
</script>

<!--子组件混入-->
<script>
import mixin from "../mixin"
mixins:[mixin]
</script>

全局混入

// main.js中
import mixin from "./mixin"
//全局混入, 任何组件都会有该配置的内容
app.mixin(mixin)

注意: 在 Vue 2 中，mixins 是创建可重用组件逻辑的主要方式。尽管在 Vue 3 中保留了 mixins 支持，但对于组件间的逻辑复用，使用组合式 API 的组合式函数是现在更推荐的方式

directives

  • Vue 允许注册自定义的指令,用于注册对当前组件实例可用的指令;
  • 用于直接操作 DOM 或增强模板的功能机制 , Vue 提供了一些内置指令（如 v-if、v-for、v-model）
  • 自定义指令：开发者可以根据需求自定义指令
 <!-- 定义自动聚焦指令 -->
<input v-focus />

<script>
    export default {
      directives: {
        // 在模板中启用 v-focus
        focus:{
            mounted: (el) => el.focus()
            // 当绑定元素插入 DOM 时调用
        }
      }
    }
</script>

钩子函数

app.directive('focus', {
  // 在绑定元素的 attribute 前 或 事件监听器应用前调用
  created(el, binding, vnode, prevVnode) {
    // 下面会介绍各个参数的细节
  },
  // 在元素被插入到 DOM 前调用
  beforeMount(el, binding, vnode, prevVnode) {},
  // 在绑定元素的父组件 及他自己的所有子节点都挂载完成后调用
  mounted(el, binding, vnode, prevVnode) {},
  // 绑定元素的父组件更新前调用
  beforeUpdate(el, binding, vnode, prevVnode) {},
  // 在绑定元素的父组件 及他自己的所有子节点都更新后调用
  updated(el, binding, vnode, prevVnode) {},
  // 绑定元素的父组件卸载前调用
  beforeUnmount(el, binding, vnode, prevVnode) {},
  // 绑定元素的父组件卸载后调用
  unmounted(el, binding, vnode, prevVnode) {}
})
  • el：指令绑定到的元素。这可以用于直接操作 DOM。
  • binding：一个对象，包含以下属性。
    • value：传递给指令的值。例如在 v-my-directive="1 + 1" 中，值是 2。
    • oldValue：之前的值，仅在 beforeUpdate 和 updated 中可用。无论值是否更改，它都可用。
    • arg：传递给指令的参数 (如果有的话)。例如在 v-my-directive:foo 中，参数是 "foo"。
    • modifiers：一个包含修饰符的对象 (如果有的话)。例如在 v-my-directive.foo.bar 中，修饰符对象是 { foo: true, bar: true }。
    • instance：使用该指令的组件实例。
    • dir：指令的定义对象。
  • vnode：代表绑定元素的底层 VNode。
  • prevNode：之前的渲染中代表指令所绑定元素的 VNode。仅在 beforeUpdate 和 updated 钩子中可用。
// 防抖
<button v-debounce="handlerAdd">点击</button>

debounce: {
    mounted(el, binding) {
        let timer
        el.addEventListener('click', () => {
            if (timer) {
                clearTimeout(timer)
            }
            timer = setTimeout(() => {
                binding.value()
            }, 1000)
        })
    }
}

methods: {
    handlerAdd() {
        console.log("执行");
    }
}

自定义事件

父组件可以通过 v-on (缩写为 @) 来监听事件; 名称使用连接符

<MyComponent @some-event="callback" />
<MyComponent @someEvent="callback" />

$emit()

当前组件触发一个自定义事件。任何额外的参数都会传递给事件监听器的回调函数

export default {
  // 用于声明由组件触发的自定义事件
  emits: ['check',"someEvent","some-event"],
  created() {
    // 仅触发事件
     this.$emit('check')
    // 带有额外的参数
    this.$emit('bar', 1, 2, 3)
  }
}

组件原生事件

emits 选项会影响一个监听器被解析为组件事件监听器，还是原生 DOM 事件监听器。被声明为组件事件的监听器不会被透传到组件的根元素上，且将从组件的$attrs对象中移除

<!--会被当成自定义事件-->
<HelloFeng @click="custom" />

自定义事件总结

  • 父子组件间通信，适用于:子组件===>父组件

  • 使用场景:A是父组件，B是子组件，B想给A传数据，那么就要在A中给B绑定自定义事件（事件的回调在A中)

  • 绑定自定义事件:

    • 在父组件中: <Demo @atguigu="test"/> 或 <Demo v-on:atguigu="test"/>

    • 若想让自定义事件只能触发一次，可以使用once修饰符。

  • 触发自定义事件: this.$emit( 'atguigu ' ,数据)

  • 组件上也可以绑定原生DOM事件，设置emits不接收。

import mitt from 'mitt'

const app = createApp(App)
const bus = mitt()
app.config.globalProperties.$bus = bus //相当于Vue2中的:Vue.prototype.$bus = bus

// 向外发布招聘信息    组合式API直接通过this获取app.config.globalProperties数据
<script>
   mounted(){
    this.$bus.on("handlerLuyao",(val)=>{
      console.log(val);
    })
  }
</script>

// 向handlerLuyao投简历
<script lang='ts' setup>
add(){
    this.$bus.emit("handlerLuyao","投稿")
},
</script>

emit(name,data) 
//触发事件，两个参数：name：触发的方法名，data：需要传递的参数
on(name,callback) 
//绑定事件，两个参数：name：绑定的方法名，callback：触发后执行的回调函数
off(name) 
//解绑事件，一个参数：name：需要解绑的方法名
