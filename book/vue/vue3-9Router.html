vue-router

vue-router 是 Vue.js 官方的路由管理器，用于构建单页面应用（SPA）。它允许你将 Vue 组件映射到不同的 URL，并在不刷新页面的情况下切换视图

  • key为路由地址,value是component 用于展示页面内容
  • 工作过程:当浏览器的路径改变时，对应的组件就会显示。

起步

安装 npm install vue-router

router文件夹

import { createRouter, createWebHashHistory, createWebHistory  } from "vue-router"
// vue3不推荐使用模板模式,需要创建路由组件
const Home = { template: '<div>Home</div>' }
const About = { template: '<div>About</div>' }

// 配置路由表
const routes = [
  { path: '/', component: Home },
  { path: '/about', component: About },
]

// 创建路由
export default createRouter({
  history: createWebHashHistory(), // hash模式
  history: createWebHistory(),  //  history模式
  routes, 
})

// main.js
import router from "./router/index.js"
Vue.createApp({}).use(router).('#app')

props配置

作用:让路由组件更方便的收到参数

// props传递给路由组件, 路由组件需要props进行接收
{
    path:"BBB",
    name:"BBB",
    component:B,
    children:[
        {
            path:"ddd/:id/:name",
            name:"dage",
            component:D,
    //第一种写法: props值为对象，该对象中所有的数据都会通过props传给组件;
            props : {a: 900},
            
    //第二种写法: props值为布尔值，布尔值为true，则把路由收到的所有params参数通过props传给路由组件 
            props :true,
  
    //第三种写法: props值为函数，该函数返回的对象中每一组key-value都会通过props传给路由组件
            // 进行选择 params 或者 query
            props(router){
                //获取到当前路由组件的数据 $router
                return {
                    id:router.query.id,
                    name:router.query.name,
                }
            }
      
        }
    ]            
}

编程式导航

  • 除了使用 <router-link> 创建 a 标签来定义导航链接，我们还可以借助 router 的实例方法，通过编写代码来实现路由跳转。
  • 此时如果支持 Promise，router.push 或 router.replace 将返回一个 Promise

push添加

$router 访问路由实例,调用 this.$router.push, 这个方法会向 history 栈添加一个新的记录

// 字符串
this.$router.push("/admin/message")
this.$router.push("/admin/message?id=002&name=asd)   // query
this.$router.push("/admin/message/阿萨德/asdd")       // params

// 对象
this.$router.push({ path: '/admin/message' })
// name代替path  对象传参使用name操作 
this.$router.push({ name: 'AA', query: { plan: 'private' }})
this.$router.push({ name: 'User',  params: { id: '123' } })

replace替换

router.replace 向 history 添加新记录并且替换上一个记录

// 字符串
this.$router.replace ("/admin/message")
this.$router.replace("/admin/message?id=002&name=asd)   // query
this.$router.replace("/admin/message/阿萨德/asdd")       // params

// 对象
this.$router.replace ({ path: '/admin/message' })
this.$router.replace ({ name: 'AA', query: { plan: 'private' }})
this.$router.replace ({ name:'User',params: { id: '123' } })

浏览器API

router.go(n) 参数是一个整数，意思是在 history 记录中向前或者后退多少步

// 在浏览器记录中前进一步，等同于 history.forward()
this.$router.go(1)

// 后退一步记录，等同于 history.back()
this.$router.go(-1)

// 前进 3 步记录
this.$router.go(3)

// 如果 history 记录不够用，那就默默地失败呗
this.$router.go(-100)
this.$router.go(100)

前进后退

this.$router.forward() // 前进
this.$router.back()   // 后退
this.$router.go(n)

路由模式

History 模式

history原理： 利用H5的 history中新增的两个API pushState() 和 replaceState() 和一个事件onpopstate监听URL变化; 通过浏览器的历史记录进行修改

  1. .地址干净,美观
  2. 兼容性和hash模式相比略差
  3. 应用部署上线时，后端人员支持，解决刷新页面服务端404的问题
  • 通过浏览器前进后退改变 URL 时会触发 popstate 事件
  • 通过pushState/replaceState或<a>标签改变 URL 不会触发 popstate 事件。
  • 好在我们可以拦截 pushState/replaceState的调用和<a>标签的点击事件来检测 URL 变化
  • 通过js 调用history的back，go，forward方法课触发该事件
// 每当 history 对象出现变化时，就会触发 popstate 事件。
window.onpopstate = function(e) {
  	console.log("触发")
}

let stateObj = {
    foo: "bar",
};

history.pushState(stateObj, "page 2", "bar.html");
// 点击返回的时候 打印触发

Hash模式

hash即URL中"#"字符后面的部分。

  1. 使用浏览器访问网页时，如果网页URL中带有hash，页面就会定位到id（或name）与hash值一样的元素的位置；
  2. hash还有另一个特点，它的改变不会导致页面重新加载；
  3. hash值浏览器是不会随请求发送到服务器端的；
  4. 通过window.location.hash属性获取和设置hash值。
  5. 兼容性好,
// 　　window.location.hash值的变化会直接反应到浏览器地址栏（#后面的部分会发生变化），同时，浏览器地址栏hash值的变化也会触发window.location.hash值的变化，从而触发onhashchange事件。

原理: hash通过监听浏览器的hashchange事件变化，查找对应的路由规则

  1. 通过浏览器前进后退改变 URL触发
  2. 通过<a>标签改变 URL触发
  3. 通过window.location改变URL触发
<a href='#/home'>home</a>
<a href='#/list'>list</a>
<div id='wrap'>132</div>
<script>
    let wrap = document.querySelector('#wrap');
    // 地址栏url发生变化触发函数
    window.addEventListener('hashchange', () => {
        let hash = location.hash;
        wrap.innerHTML = hash
    })
    // 初始化定位hash值
    window.addEventListener('DOMContentLoaded', () => {
        if (!location.hash) {//如果不存在hash值，那么重定向到#/
            location.hash = "/"
        } else {//如果存在hash值，那就渲染对应UI
            let hash = location.hash;
            wrap.innerHTML = hash
        }
</script>

动态添加路由

动态路由主要通过两个函数实现。router.addRoute() 和 router.removeRoute()

router.addRoute({ path: '/about', component: About })

添加嵌套路由

router.addRoute({ name: 'admin', path: '/admin', component: Admin })
router.addRoute('admin', { path: 'settings', component: AdminSettings })

router.addRoute({
  name: 'admin',
  path: '/admin',
  component: Admin,
  children: [{ path: 'settings', component: AdminSettings }],
})

在导航守卫中添加路由

router.beforeEach(to => {
  if (!hasNecessaryRoute(to)) {
    router.addRoute(generateRoute(to))
    // 触发重定向
    return to.fullPath
  }
})

删除路由

// 删除路由
router.addRoute({ path: '/about', name: 'about', component: About })
// 删除路由
router.removeRoute('about')
