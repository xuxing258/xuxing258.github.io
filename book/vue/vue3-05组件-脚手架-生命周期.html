组件

  • 组件允许我们将 UI 划分为独立的、可重用的部分，并且可以对每个部分进行单独的思考;
  • 使我们可以在每个组件内封装自定义内容与逻辑
  • 将 Vue 组件定义在一个单独的 .vue 文件中，这被叫做单文件组件
  • <template>、<script> 和 <style> 三个块在同一个文件中封装、组合了组件的视图、逻辑和样式

组件写法

  1. vue组件 html CSS JavaScript 组成
  2. 组件只关注对应的部分,
  3. html CSS JavaScript 全部复用,

组件名使用

  • PascalCase 是合法的 JavaScript 标识符
  • 使用 kebab-case 的标签解析为使用 PascalCase 注册的组件
  • 组件名尽可能回避HTML中已有的元素名称，例如: h2、H2都不行

生命周期

  • 又名:生命周期回调函数、生命周期函数、生命周期钩子,

  • 是什么? :Vue在指定时间执行的特殊名称的函数。

  • 生命周期函数的名字不可更改，但函数的具体内容是程序员根据需求编写的

  • 生命周期函数中的this指向是vm或组件实例对象。

选项式生命钩子函数

  1. beforeCreate 在组件实例初始化完成之后立即调用
    • 会在实例初始化完成、props 解析之后、data() 和 computed 等选项处理之前立即调用
  2. created 在组件实例创建完成之后调用
    • 内容已经设置完成：响应式数据、计算属性、方法和侦听器。然而，此时挂载阶段还未开始，因此 $el 属性仍不可用
  3. beforeMount 在组件被挂载之前调用
    • 此时已经完成了模版的编译，还没有渲染到界面中,
  4. mounted 在组件被挂载之后调用
    • 所有同步子组件都已经被挂载
    • 其自身的 DOM 树已经创建完成并插入了父容器中
    • 钩子通常用于执行需要访问组件所渲染的 DOM 树
  5. beforeUpdate 在组件数据发生变化,页面数据没有发送变化之前
  6. updated 在页面模板DOM更新之后
  7. beforeUnmount 在一个组件实例被卸载之前调用
    • 当这个钩子被调用时，组件实例依然还保有全部的功能
  8. unmounted 在一个组件实例被卸载之后调用。
    • 在钩子中手动清理一些副作用，例如计时器、DOM 事件监听器或者与服务器的连接。

常用的生命周期钩子

  • mounted: 发送Ajax请求、启动定时器、绑定自定义事件、订阅消息等【初始化操作】
  • beforeUnmount: 清除定时器、解绑自定义事件、取消订阅消息等【收尾工作】
