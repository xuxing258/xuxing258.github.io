vuex

通过调用 useStore 函数，来在 setup 钩子函数中访问 store

<script>
import { useStore } from 'vuex'

export default {
  setup () {
    const store = useStore()
    
    store.commit("ADD_ADD","传递数据")
	store.dispatch("add_add","传递数据")
      
  	return {
     	 num:store.state.num,
         num: computed(() => store.state.num),
    }
  }
}
</script>

Router

在 setup 里面没有访问 this，所以我们不能再直接访问 this.$router 或 this.$route。作为替代，我们使用 useRouter 函数

import { useRouter, useRoute } from 'vue-router'
const router = useRouter()  //路由对象
const route = useRoute()    //当前路由信息

function pushWithQuery(query) {
    router.push({
        name: 'search',

    })
}

ref属性

关于 ref 注册时机的重要说明：因为 ref 本身是作为渲染函数的结果来创建的，必须等待组件挂载后才能对它进行访问

使用选项式 API，引用将被注册在组件的 `this.$refs` 对象里
<p ref="p">hello</p>

组合式 API，引用将存储在与名字匹配的 ref 里
<script setup>
import { ref } from 'vue'

const p = ref()
</script>

<template>
  <p ref="p">hello</p>
</template>

无法直接获取子组件的数据信息,需要子组件进行暴露 defineExpose

// 父组件正常ref组件名

// 子组件
import { ref } from 'vue'

const a = 1
const b = ref(2)

defineExpose({
  a,
  b
})

provide

用于提供可以被后代组件注入的值

  1. 祖先组件有一个 provide 选项来提供数据，后代组件有一个 inject 选项来开始使用这些数据
    • 提供响应式数据
      • 祖先修改provide原数据,后代组件接收的provide数据会同时更新
      • 后代组件修改provide数据, 也会导致祖先组件provide原数据更新
        • 可通过readonly 设置成只读的数据
    • 提供基础数据
      • 不会响应变化

  • 具体写法：

    1. 祖组件中：

      import { provide, reactive } from 'vue';
let obj = reactive({ name:"路遥"})
provide("cat",obj)
// 祖先元素能修改, 后代不能修改
let a = ref(11)
provide("cat1",readonly(a))

    2. 后代组件中：

      import { inject } from 'vue';
const car = inject('cat')

const car1 = inject('cat1')

Teleport

  • 什么是Teleport？—— Teleport 是一种能够将我们的组件html结构移动到指定位置的技术。

  • <Teleport> 挂载时，传送的 to 目标必须已经存在于 DOM 中。理想情况下，这应该是整个 Vue 应用 DOM 树外部的一个元素。如果目标元素也是由 Vue 渲染的，你需要确保在挂载 <Teleport> 之前先挂载该元素。

  • 只改变了渲染的 DOM 结构，它不会影响组件间的逻辑关系

    <teleport to="移动位置">
	<div v-if="isShow" class="mask">
		<div class="dialog">
			<h3>我是一个弹窗</h3>
			<button @click="isShow = false">关闭弹窗</button>
		</div>
	</div>
</teleport>

异步组件

在大型项目中，我们可能需要拆分应用为更小的块，并仅在需要时再从服务器加载相关组件。Vue 提供了 defineAsyncComponent 方法来实现此功能

defineAsyncComponent

  • 定义一个异步组件，它在运行时是懒加载的。
  • 参数可以是一个异步加载函数，或是对加载行为进行更具体定制的一个选项对象。
  • 最后得到的 AsyncComp 是一个外层包装过的组件，仅在页面需要它渲染时才会调用加载内部实际组件的函数
<script>
import { defineAsyncComponent } from 'vue'
import {ref} from "vue"
let bo = ref(false)
// 将普通组件拆分仅在需要时加载
const Home = defineAsyncComponent(() =>  import('./components/Home.vue'))
</script>

<template>
    <!-- 顶层不能有任何的异步, 隐藏时不会加载Home组件,显示时加载Home组件数据 -->
    <Home v-if="bo"/>
	<button @click="bo = !bo">点</button>
</template>

加载与错误状态 (通过设置网页中network的网速进行展示对应的效果)

const AsyncComp = defineAsyncComponent({
  // 加载函数
  loader: () => import('./Foo.vue'),

  // 加载异步组件时使用的组件
  loadingComponent: LoadingComponent,
  // 展示加载组件前的延迟时间，默认为 200ms
  delay: 200,

  // 加载失败后展示的组件
  errorComponent: ErrorComponent,
  // 如果提供了一个 timeout 时间限制，并超时了
  // 也会显示这里配置的报错组件，默认值是：Infinity
  timeout: 3000
})

Suspense

<Suspense> 是一个内置组件，用来在组件树中协调对异步依赖的处理。

它让我们可以在组件树上层等待下层的多个嵌套异步依赖项解析完成，并可以在等待时渲染一个加载状态

异步依赖

  • 如果使用 <script setup>，顶层中使用 await 表达式会自动让该组件成为一个异步依赖,
  • 异步组件: 通过defineAsyncComponent引入的组件
let data = await  axios.post("http://xiaochengxu.houdu.online/buchong/get",{type:"轮播图"})
console.log(data);

处理异步

等待组件异步依赖项解析完成,让应用有更好的用户体验

// home组件
import { ref } from "vue";

let ss = ref(0);
let promise = await new Promise((res, rej) => {
  setTimeout(() => {
    res(11);
  }, 2000);
});

<Suspense >
    <!-- default 异步依懒组件加载完成显示 -->
    <template #default>
		<Home />
    </template>
    <!-- fallback 加载异步依懒组件过程中展示 -->
    <template #fallback>
        <h3>加载中.....</h3>
    </template>
</Suspense>

<script setup>
import Home from './component/Home.vue';
</script>

Suspense事件

  • pending 事件是在进入挂起状态时触发。
  • resolve 事件是在 default 插槽完成获取新内容时触发。
  • fallback 事件则是在 fallback 插槽的内容显示时触发
<Suspense @fallback="habd"></Suspense>

Suspense和异步组件使用

加载是需要时间的: 同理(通过设置网页中network的网速进行展示对应的效果) 需要注意缓存问题
