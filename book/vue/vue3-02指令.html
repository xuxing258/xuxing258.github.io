条件渲染

v-if

v-if 指令用于条件性地渲染一块内容。这块内容只会在指令的表达式返回真值时才被渲染

  1. v-if="表达式
  2. v-if v-else v-if v-else-if v-else
  3. 使用v-if v-else 或者 v-else-if 必须链接在一起使用
  4. v-if 在 <template> 元素上使用渲染多项,不占据位置

v-show

v-show 指令通过条件显示一个元素

  1. v-show 会在 DOM 渲染中保留该元素；v-show 仅切换了该元素上名为 display 的 CSS 属性
  2. v-show 不支持在 <template> 元素上使用，也不能和 v-else 搭配使用

区别

  1. v-if 是“真实的”按条件渲染，因为它确保了在切换时，条件区块内的事件监听器和子组件都会被销毁与重建
  2. v-if 也是惰性的：如果在初次渲染时条件值为 false，则不会做任何事。条件区块只有当条件首次变为 true 时才被渲染
  3. v-show 简单许多，元素无论初始条件如何，始终会被渲染，只有 CSS display 属性会被切换

总结: v-if 有更高的切换开销，而 v-show 有更高的初始渲染开销。因此，如果需要频繁切换，则使用 v-show 较好；如果在运行时绑定条件很少改变，则 v-if 会更合适

Attribute 绑定

绑定属性

双大括号不能在 HTML attributes 中使用, 想要响应式地绑定一个 attribute，应该使用 v-bind

v-bind 指令指示 Vue 将元素的 id attribute 与组件的 dynamicId 属性保持一致。如果绑定的值是 null 或者 undefined，那么该 attribute 将会从渲染的元素上移除

<div v-bind:id="dynamicId"></div>

<!-- 绑定 attribute -->
<img v-bind:src="imageSrc" />

<!-- 动态 attribute 名 -->
<button v-bind:[key]="value"></button>

<!-- 缩写 -->
<img :src="imageSrc" />

<!-- 缩写形式的动态 attribute 名 -->
<button :[key]="value"></button>

<!-- 内联字符串拼接 -->
<img :src="'/path/to/images/' + fileName" />

<!-- class 绑定 -->
<div :class="{ red: isRed }"></div>
<div :class="[classA, classB]"></div>
<div :class="[classA, { classB: isB, classC: isC }]"></div>

<!-- style 绑定 -->
<div :style="{ fontSize: size + 'px' }"></div>
<div :style="[styleObjectA, styleObjectB]"></div>

<!-- 绑定对象形式的 attribute -->
<div v-bind="{ id: someProp, 'other-attr': otherProp }"></div>

总结

  • class样式
    • 写法:class="xxx xxx可以是字符串、对象、数组。
      • 字符串写法适用于:类名不确定,要动态获取。
      • 对象写法适用于:要绑定多个样式,个数不确定,名字也不确定。
      • 数组写法适用于:要绑定多个样式，个数确定，名字也确定，但不确定用不用。
  • style样式
    • :style="{fontsize: xxx}"其中 xxx是动态值。
    • :style="[a,b]"其中a、b是样式对象。

事件处理

使用 v-on 指令 (简写为 @) 来监听 DOM 事件，并在事件触发时执行对应的 JavaScript

内联事件处理器

内联事件处理器通常用于简单场景，例如

<button @click="count++">Add 1</button>
<p>Count is: {{ count }}</p>
<!-- 内联事件中书写表达式 -->

方法事件处理器

随着事件处理器的逻辑变得愈发复杂，内联代码方式变得不够灵活。因此 v-on 也可以接受一个方法名或对某个方法的调用

<button @click="greet">Greet</button>
  1. 方法中的 this 指向当前活跃的组件实例
  2. event 是 DOM 原生事件

事件对象

  1. 函数没有传参时; 此时事件对象是函数名中第一个参数
  2. 函数传参时,需要事件对象,在函数的最后输入$event 传事件对象

事件修饰符

处理事件时调用 event.preventDefault() event.stopPropagation()

  • .stop 阻止事件冒泡；
  • .prevent 阻止默认事件；
  • .capture 使用事件捕获模式；
  • .self 只在自身触发（子元素不能触发）；
  • .once 只触发一次；
  • .passive 事件的默认行为立即触发,无需等待事件回调函数执行完毕，无视 preventDefault。
    • 根据MDN文档的passive:
      • 一个布尔值，设置为 true 时，表示 listener 永远不会调用 preventDefault()
<!-- 阻止单击事件冒泡 -->
<div @click.stop="doThis"></div>

<!-- 只有修饰符 => 同样会生效 => 阻止右键默认事件-->
<div @contextmenu.prevent></div>

<!-- 阻止默认跳转 -->
<a herf="https://www.baidu.com" @click.prevent="showInfo">

<!-- 添加事件监听器时使用事件捕获模式 -->
<div @click.capture="doThis">...</div>

<!-- 只当在 event.target 是当前元素自身时触发处理函数 -->
<!-- 即便事件是从内部元素触发的 -->
<div class="box" @click.self="repeatFun">
        <p>{{msg}}</p>
        <p @click="dbChange">点我</p>
 </div>

<!-- 点击事件将只会触发一次 -->
<div @click.once="doThis"></div>

passive

<style>
    ul{
        height: 400px;
        background-color: red;
        overflow: auto;
    }
    ul li{
        height: 200px;
        height: 200px;
    }
</style>

<div id="app">
    <!--wheel  鼠标滚轮滚动事件 -->  
    
    <!--wheel 未设置passive 鼠标滚轮触发事件,先执行回调函数完成,才会去执行浏览器默认行为  -->
    <!--事件的默认行为立即触发,无需等待事件回调函数执行完毕-->
    <ul @wheel.passive="dome">
        <li>1</li>
        <li>2</li>
        <li>3</li>
        <li>4</li>
    </ul>
</div>

<script>
    Vue.createApp({
        methods: {
            dome(){
                // 在元素上设置.passive的时候 会直接触发滚动默认行为,不需要等待函数执行完毕
                for (let index = 0; index < 100000; index++) {
                    console.log("大");
                }
            }
        }
    }).mount("#app")
</script>

按键修饰符

监听键盘事件时,只有指定的按键才会执行

按键别名

<!-- 仅会在 $event.key 为 'enter' 时调用事件处理。 -->
<input @keyup.enter="onPageDown" />

Vue 为一些常用的按键提供了别名：

  • .enter
  • .tab
  • .delete (捕获“Delete”和“Backspace”两个按键)
  • .esc
  • .space
  • .up
  • .down
  • .left
  • .right

Vue未提供别名的按键，可以使用按键原始的event.key 值去绑定，但注意要转为kebab-case（短横线命名)

<!--只要按健抬起就会触发,不友好,可以设置指定触发,使用按键修饰符-->
<input @keyUp="submit" placeholder="按键是提示">

<!-- enter 抬起时，调用 submit()  其他按键不能调用 -->
<input @keyUp.enter="submit" placeholder="enter">

<!-- up 抬起时，调用 submit()  其他按键不能调用 -->
<input type="text" @keyUp.esc="submit" placeholder="tab">

<!-- 其他同理 -->
<input type="text" @keyUp.down="submit" placeholder="down">

<!-- 添加自定义修饰符  自定义修饰符event.key的值, -->
<input @keyUp.q="fun"> 

<!--使用 event.keyCode自定义健码值 被废除了  81表示q -->
<input type="text" @keyUp.81="fun">

<!--自定义修饰符大小写  光标按键pageDown按下时，使用-连接符 并小写 -->
<input @keyUp.page-down="fun">

<!--自定义组合 按下q w 任意键都会触发-->
<input @keyUp.q.w="fun" />

<!--理论上 按下tab键触发 但是焦点转走了 不会被触发了 除非使用keydown -->
<input @keyup.tab="fun" />

系统修饰键

请注意系统修饰键与常规按键不同，在和事件一起用时，事件触发时修饰键必须处于按下keydown状态。

  • .ctrl
  • .alt
  • .shift
  • .meta 按下出现搜索 => window健

注意：在 Mac 系统键盘上，meta 对应 command 键 (⌘)。在 Windows 系统键盘 meta 对应 Windows 徽标键 (⊞)。在 Sun 操作系统键盘上，meta 对应实心宝石键 (◆)。在其他特定键盘上，尤其在 MIT 和 Lisp 机器的键盘、以及其后继产品，比如 Knight 键盘、space-cadet 键盘，meta 被标记为“META”。在 Symbolics 键盘上，meta 被标记为“META”或者“Meta”

<!-- 系统修饰符需按下状态才会触发 -->
<input @keydown.ctrl="clear" />

自定义组合健

系统组合健

<!--按下ctrl  enter 触发事件-->
<input @keydown.ctrl.enter="clear" /> 

<!--抬起ctrl  a触发  其他不触发-->
<input type="text" @keyup.ctrl.a="cler">

系统修饰符与其他事件配合

<!-- 点击 + 按下Ctrl触发 -->
<div @click.ctrl="doSomething">Do something</div>

<!-- 点击即可生效,不需要按下,应为不是系统修饰符 -->
<button @click.q="handler">点击</button>
.exact 修饰符

允许你精确的控制系统修饰符组合触发的事件。

只支持系统按键并不支持键盘事件

<!-- 即使 ctrl 或 Shift 被一同按下时也会触发 -->
<button @click.ctrl="onClick">A</button>

<!-- 有且只有 Ctrl 被按下的时候才触发 -->
<button @click.ctrl.exact="onCtrlClick">A</button>
 
<!-- 没有任何系统修饰符被按下的时候才触发  按下alt ctrl等系统修饰符都不会触发 -->
<button @click.exact="onClick">A</button>

列表渲染

使用 v-for 指令基于一个数组来渲染一个列表。v-for 指令的值需要使用 item in items 形式的特殊语法，其中 items 是源数据的数组，而 item 是迭代项的别名

<!-- 数组 与 对象 与数字演示 -->
<li v-for="item in items"> {{ item.message }} </li>
  1. 用于展示列表数据
  2. 语法: v-for="(item,index) in xxx" : key="yyy"
  3. 可遍历: Array | Object | number | string | Iterable

数组更新检测

Vue 将被侦听的数组的变更方法进行了包裹，所以它们也将会触发视图更新。这些被包裹过的方法包括：

  • push()
  • pop()
  • shift()
  • unshift()
  • splice()
  • sort()
  • reverse()
  • arr[下标] 也可修改

变更方法，顾名思义，就是会对调用它们的原数组进行变更。相对地，也有一些不可变 (immutable) 方法，例如 filter()，concat() 和 slice()，这些都不会更改原数组，而总是返回一个新数组。当遇到的是非变更方法时，我们需要将旧的数组替换为新数组

模板template

不在DOM中渲染该元素,可通过<template>标签可以作为占位符使用

<ul>
  <template v-for="item in items">
    <li>{{ item.msg }}</li>
    <li class="divider" role="presentation"></li>
  </template>
</ul>

v-for v-if

v-if 比 v-for 的优先级更高。这意味着 v-if 的条件将无法访问到 v-for 作用域内定义的变量别名

<!--  这会抛出一个错误，因为属性 todo 此时  没有在该实例上定义 -->
<li v-for="todo in todos" v-if="!todo.isComplete">
  {{ todo.name }}
</li>

解决

<template v-for="todo in todos">
  <li v-if="!todo.isComplete">
    {{ todo.name }}
  </li>
</template>

表单

  • 在前端处理表单时，我们常常需要将表单输入框的内容同步给 JavaScript 中相应的变量。手动连接值绑定和更改事件监听器可能会很麻烦
  • v-model 在表单输入元素或组件上创建双向绑定
    • 根据表单输入元素或组件输出的值而变化

基本用法

<p>Message is: {{ message }}</p>
<input v-model="message" placeholder="edit me" />

注意在 <textarea> 中是不支持插值表达式的。请使用 v-model 来替代

<p style="white-space: pre-line;">{{ message }}</p>
<textarea v-model="message"></textarea>

修饰符

.lazy 失去焦点或更改内容后回车收集数据

<!-- v-model 会在每次 input 事件后更新数据, -->
<!-- lazy在change时更新而非“input”时更新 --> 
<input v-model.lazy="msg">

.number 输入字符串转为有效的数字

<input v-model.number="age" type="number">

这通常很有用，因为即使在 type="number" 时，HTML 输入元素的值也总会返回字符串。如果这个值无法被 parseFloat() 解析，则会返回原始的值。

.trim 输入首尾空格过滤

<input v-model.trim="msg">

总结

  • 文本v-model收集: value
  • 单选v-model收集: value; 且要给标签配置value值
  • 多选框v-model收集:
    • 未配置value属性，收集的是checked（勾选 or 未勾选: 布尔值)
    • 配置value属性
      • v-model初始值非数组，收集checked（勾选or未勾选，是布尔值)
      • v-model初始值数组，收集value组成的数组
  • 修饰符
    • lazy : 失去焦点再收集数据
    • number : 输入字符串转为有效的数字
    • trim : 输入首尾空格过滤

其他指令

v-cloak

用于隐藏尚未完成编译的 DOM 模板。

<style>
    <!--配置使用-->
    [v-cloak] {
       display: none;
    }
</style>

<!--{{msg}}没有渲染成功, -->
<div id="app" v-cloak>
   <div >
    {{msg}}
   </div>
</div>

v-pre

跳过该元素及其所有子元素的编译。可以用来显示原始 Mustache 标签。跳过大量没有指令的节点会加快编译。

<span v-pre>{{ this will not be compiled }}</span>

v-once

仅渲染元素和组件一次，并跳过之后的更新, 可以用于优化性能。

<span v-once>This will never change: {{msg}}</span>

<!-- 带有子元素的元素 -->
<div v-once>
  <h1>comment</h1>
  <p>{{msg}}</p>
</div>

v-memo

缓存一个模板的子树。在元素和组件上都可以使用。为了实现缓存，该指令需要传入一个固定长度的依赖值数组进行比较。如果数组里的每个值都与最后一次的渲染相同，那么整个子树的更新将被跳过

<!-- msg数据更改  valueA与valueb数据未更改不会进行重新渲染-->
<p>{{msg}}</p> 
<div v-memo="[valueA, valueB]">
  {{ msg }}
</div>

当组件重新渲染，如果 valueA 和 valueB 都保持不变，这个 <div> 及其子项的所有更新都将被跳过。实际上，甚至虚拟 DOM 的 vnode 创建也将被跳过，因为缓存的子树副本可以被重新使用
