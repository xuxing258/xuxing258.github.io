使用pinia

Pinia 是 Vue.js 的一个现代化状态管理库，由 Vue 核心团队成员开发，旨在提供更简单、更直观的状态管理解决方案

  1. 相比 Vuex，Pinia 的 API 更简单，减少了模板代码（如 mutations 的移除），直接通过 actions 修改状态。
  2. 模块化设计
    • 不需要嵌套模块，每个 Store 是独立的，通过文件自动组织逻辑（类似 composables 的用法）。
  3. Composition API 友好
    • 可以直接在组件中使用 setup() 与 Pinia 交互，更符合 Vue 3 的开发模式。

建一个 pinia（根存储）并将其传递给应用程序

// main.js
import { createPinia } from 'pinia'
app.use(createPinia())

创建仓库 stores/counter.js

  • Store 的实例变量通常以 use 开头
// defineStore用于定义一个数据管理容器，并且它需要一个唯一的名称，作为第一个参数传递：
// 这个名称，也称为id，是必需的，Pania 使用它来商店连接到 devtools。
import { defineStore } from "pinia";

export const useCounterStore = defineStore("main", {
  state: () => {
    return { count: 0 };
  },
  getters: {
    double: (state) => state.count * 2,
  },
  actions: {
    increment() {
      this.count++;
    },
  },
});

组件

// App组件
import { useCounterStore } from '@/stores/counter'
// 执行仓库获取数据
const store = useCounterStore();

getter

Getter 完全等同于 Store 状态的计算值。它们可以用 中的getters属性定义defineStore()。

接收state作为第一个参数来鼓励使用箭头函数：

Getter 只是幕后的计算属性，所以不可以向它们传递任何参数。不过，你可以从 getter 返回一个函数，该函数可以接受任意参数

export const useStore = defineStore('main', {
  state: () => ({
    counter: 0,
  }),
  getters: {
    doubleCount: (state) => state.counter * 2,
  },
})

store.$subscribe((mutation, state) => {
  console.log(mutation, state);

Actions

可以使用actionsin 属性进行定义方法,非常适合定义业务逻辑

它可以是异步，您可以在它们内部进行任何 API 调用甚至其他操作！

export const useStore = defineStore('main', {
  state: () => ({
    counter: 0,
  }),
  actions: {
    increment() {
      this.counter++
    },
    randomizeCounter() {
      this.counter = Math.round(100 * Math.random())
    },
  },
})

方法执行

// 执行
store.increment()  // 可传递参数

仓库可以创建多个, 各自仓库可以进行通信使用

import { useAuthStore } from './auth-store'

export const useSettingsStore = defineStore('settings', {
  state: () => ({
    preferences: null,
  }),
  actions: {
    // 进行异步使用
    async fetchUserPreferences() {
      const auth = useAuthStore()  // 使用其他仓库数据
      
      if (auth.isAuthenticated) {
        this.preferences = await fetchPreferences()
      } else {
        throw new Error('User must be authenticated')
      }
    },
  },
})

// getter获取其他仓库数据
import { useOtherStore } from './other-store'

export const useStore = defineStore('main', {
  state: () => ({
    // ...
  }),
  getters: {
    otherGetter(state) {
      // 获取仓库数据
      const otherStore = useOtherStore()
      return state.localData + otherStore.data
    },
  },
})

组合式方式

  • ref() 成为state属性
  • computed()变成getters
  • function()变成actions
import { defineStore } from "pinia";
import { ref, computed } from "vue"

export default defineStore("index", () => {
  const count = ref(0)
  const name = ref("路遥")
  
  function increment() {
    this.count++;
  };

  let double = computed(() => {
    return count.value * 2
  })
  return { count, increment, name, double }
})
