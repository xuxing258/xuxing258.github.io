装饰器

1.闭包

构成闭包函数的基本条件

  • 必须有一个内嵌函数：闭包函数需要在其内部定义一个或多个函数，这些内部函数被称为内嵌函数或嵌套函数。
  • 内嵌函数必须引用外部函数中的变量：闭包函数的内嵌函数必须引用外部函数中的变量。这意味着内嵌函数可以访问和操作外部函数的变量，并且这些变量的状态在内嵌函数被调用时保留。
  • 外部函数返回值必须是内嵌函数：外部函数需要返回其内部定义的内嵌函数。这样，外部函数调用后返回的是内嵌函数本身，而不是执行内嵌函数的结果。

满足这三个要素的函数就是一个闭包函数。闭包可以被视为一种封装机制。封装是面向对象编程中的一个重要概念，它允许将数据和相关的操作封装在一个单元中。

def func1():
    number = 10
    def func2():
        print(number)
    return func2


res = func1() # res等价于func2
res()
---------------------------------
def func1(number):
    def func2():
        print(number)
    return func2


res = func1(10) # res等价于func2
res()
-----------------------------------
def counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        print("count:", count)
    return increment

counter1 = counter()  # counter1等效于increment
counter1()  # count: 1
counter1()  # count: 2
  • 在python中闭包的关键作用就是保存外部函数的局部变量，并且这些变量不会随着外部函数的执行结束而销毁。

2.装饰器

装饰器允许在不修改原始函数代码的情况下，通过包装函数来扩展或修改其行为。

装饰器是通过闭包的方式实现的
– 外部函数接收一个函数作为函数的参数变量，内部函数中执行这个函数；
– 内部函数接收需要的参数给执行的函数，返回执行结果；
– 内部函数中可以添加额外的功能代码；
– 外函数将内函数返回

2.1 装饰器范例:

  • 装饰器遵循开放封闭原则，及原函数是封闭的，但是可以通过装饰器来修改原函数。
  • 装饰器其本质就是闭包，是特殊的闭包函数。
import time
start = time.time() # 1743168548.3764594 就是从1970年到现在的总秒数
print(start)
time.sleep(1) # 休眠程序,括号里面是休眠秒数

import time
def run_time(func): # func:send原函数
    def inner():
        start = time.time()
        time.sleep(1)
        func() # send原函数的位置
        end = time.time()
        print(f"运行了: {end - start}秒")
    return inner
def send(): # 原函数
    print("发送信息的功能函数")
send = run_time(send) # send等价于inner
send() # inner()

语法糖:@
import time
def run_time(func): # func:send原函数
    def inner():
        start = time.time()
        time.sleep(1)
        func() # send原函数的位置
        end = time.time()
        print(f"运行了: {end - start}秒")
    return inner
@run_time # send = run_time(send) # send等价于inner
def send(): # 原函数
    print("发送信息的功能函数")
@run_time # recv = run_time(recv)
def recv(): # 原函数
    print("接收信息的功能函数")

send() # inner()

2.2 带参数的装饰器

import time
def run_time(func): # func:num_sum原函数
    def inner(*args,**kwargs): # args(1,2,3) kwargs{"a":10,"b":20}
        start = time.time()
        time.sleep(1)
        func(*args,**kwargs) # 调用num_sum原函数的位置
        end = time.time()
        print(f"运行了: {end - start}秒")
    return inner
@run_time # num_sum = run_time(num_sum)
def num_sum(*args,**kwargs):
    result = 0
    for num in args:
        result += num
    print(result)
    for k,v in kwargs.items():
        print(k,v)


num_sum(1,2,3,a=10,b=20) # inner

    return inner

ret = num_sum(1,2,3) # inner

通过在函数定义上方使用 @run_time，我们将 num_sum 函数应用了装饰器。这意味着在调用 num_sum() 时，实际上是在调用装饰器返回的inner函数。这样，我们可以在不修改 num_sum 的情况下，为其添加额外的行为（记录执行时间）。

# 装饰器模板
def outer(func):
    def inner(*args,**kwargs):
        # 执行前增加功能
        res=func(*args,**kwargs) # 调用原函数
        # 执行后增加功能
        return res
    return inner

3.迭代器(了解)

迭代是指重复执行某个操作的过程，并且每次迭代得到的结果会作为下一次迭代的初始值。

可迭代对象：从语法形式上讲，内置有__iter__方法的对象都是可迭代对象，字符串、列表、元组、字典、集合都是可迭代对象。

# 判断是否为可迭代对象
# 1.iter方法
# str1 = "123"
# str1.__iter__()
#
# list1 = [1, 2, 3]
# list1.__iter__()
#
# tuple1 = (1, 2, 3)
# tuple1.__iter__()
#
# dict1 = {"a": 1, "b": 2, "c": 3}
# dict1.__iter__()
#
# set1 = {1, 2, 3}
# set1.__iter__()

# 2.使用isinstance方法
# a = "1234"
# print(isinstance(a, str))  # True
# from collections.abc import Iterable  # 导入可迭代对象
# print(isinstance([1, 2, 3], Iterable))  # True
# print(isinstance(100, Iterable))  # False

# 3.能被for循环

调用__iter__方法返回的结果就是一个迭代器对象(Iterator)。迭代器对象是内置有__iter__和__next__方法的对象

  • 执行迭代器对象__iter__方法得到的仍然是迭代器本身
  • 而执行迭代器__next__方法就会计算出迭代器中的下一个值。
list1 = [1, 2, 3]
data = iter(list1)  # list1.__iter__()
print(data)  # <list_iterator object at 0x000001A03F8C3C10>
val1 = next(data)
print(val1)  # 1
val1 = next(data)
print(val1)  # 2
val1 = next(data)
print(val1)  # 3

for循环原理

  • for 循环是一种迭代循环，用于遍历可迭代对象中的元素。
  • for循环语法中的in后可以跟任何可迭代对象，如列表、元组、字符串、字典等。
  • 当for循环开始执行时，它会首先调用可迭代对象的内置__iter__方法，以获取一个迭代器对象。
  • 然后，for循环会重复调用迭代器对象的__next__方法，以获取可迭代对象中的下一个值，并将这个值赋给循环变量。
  • 这个过程会一直重复，直到迭代器对象耗尽，即可迭代对象中的所有元素都被访问完毕，或者遇到break语句提前终止循环。
  • 当迭代器对象耗尽时，它会引发StopIteration异常，这时for循环会捕捉到该异常，结束迭代过程。
list1 = [1, 2, 3]
data = iter(list1)  # list1.__iter__()
for val1 in data:
    print(val1)
    

# 处理异常
list1 = ["1", "2", "3", "4"]
data = iter(list1)  # 1拿到迭代器
while True:
    try:  # try是尝试
        i = next(data)  # # 2不断的调用迭代器的next方法
        print(i)
    except StopIteration:  # except 捕获 # 3停止迭代
        break

迭代器提供了一种统一的遍历方式，无论是对列表、字典、集合还是自定义的可迭代对象，都可以使用相同的for循环语法进行遍历。

4.生成器(了解)

生成器是一个特殊的迭代器

# 列表推导式
numbers = [i for i in range(1, 10000001)]
print(numbers) # 影响程序的快慢,最主要的元素是我们的cpu和内存

生成器表达式

data = (i for i in range(1, 6))
print(data)  # 生成器对象generator
print(next(data))
print(next(data))
print(next(data))
print(next(data))
print(next(data))

data = (i for i in range(1, 6))
for i in data:
    print(i)

若函数体包含yield关键字，再调用函数，并不会执行函数体代码，得到的返回值即生成器对象，生成器内置有__iter__和__next__方法，所以生成器本身就是一个迭代器。我们可以用__next__触发生成器所对应函数的执行

def func():
    print("func1")
    yield 1 # 停止了
    print("func2")
    yield 2
    print("func3")
    yield 3
# 指定生成器函数,函数体不会被执行,返回了一个生成器对象
data = func()
print(data) # <generator object func at 0x00000160D35095F0>
# next方法里面放生成器对象,就会进入生成器函数并执行里面的代码
g1 = next(data)
print(g1) # 1
# 从上次yield的返回位置继续往下执行
g2 = next(data)
print(g2) # 2

g3 = next(data)
print(g3) # 3

def func():
    print("func1")
    yield 1 # 停止了
    print("func2")
    yield 2
    print("func3")
    yield 3

for i in func():
    print(i)

def my_range(start, stop, step=1):
    while start < stop:
        yield start
        start += step
        
g = my_range(0,3)
print(next(g))

有了yield关键字，我们就有了一种自定义迭代器的实现方式。yield可以用于返回值，但不同于return，函数一旦遇到return就结束了，而yield可以保存函数的运行状态挂起函数，用来返回多次值。
