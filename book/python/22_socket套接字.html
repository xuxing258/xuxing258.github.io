socket套接字

​ Socket套接字是计算机网络编程中用于实现网络通信的一种机制。它提供了一种编程接口，使得应用程序能够通过网络进行数据的发送和接收。套接字可用于不同的网络通信协议，如TCP/IP和UDP。

套接字的主要特点包括：

  1. 客户端和服务器端：套接字可以用于客户端和服务器端之间的通信。客户端套接字发起连接请求，而服务器端套接字等待接收连接请求。
  2. IP地址和端口：套接字通过IP地址和端口来标识网络上的特定应用程序或进程。IP地址用于标识主机，而端口用于标识特定应用程序。
  3. 通信模式：套接字可以使用不同的通信模式，如面向连接的TCP套接字和无连接的UDP套接字。TCP套接字提供可靠的、面向连接的通信，而UDP套接字提供不可靠的、无连接的通信。
  4. 数据传输：通过套接字可以发送和接收数据。数据可以是字节流（如TCP套接字）或数据报（如UDP套接字）。

1.UDP套接字

应用层的一种编程方法。

  • socket()：创建套接字

  • bind()：绑定IP端口

  • sendto()：发送消息

  • recvfrom()：接受消息

  • close()：关闭套接字

udp服务端代码

"""
服务端
    1.创建套接字
    2.绑定ip端口
    3.发送/接收消息
    4.关闭套接字
"""
# 导入socket模块
import socket

# 1.创建套接字
# AF_INET 表示ipv4
# SOCK_DGRAM 表示选用的UDP模式
server = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# 2.绑定ip端口
# 声明服务端的ip和端口号
# 套接字绑定接收的数据格式是元组
# 这个元组内的元素是(ip,port)
ADDR = ("127.0.0.1", 9999)
# 将声明的地址绑定到套接字上
# 相当于这个服务给他一个ip,一个端口
server.bind(ADDR)

# 3.接收消息
# 1024 一次接收最大的消息字节
# 第一个返回值:客户端发送过来的消息数据
# 第二个返回值:客户端的地址(ip以及端口)
msg, addr = server.recvfrom(1024) # 阻塞状态
print(addr)

# 打印客户端发送的消息
# decode() 消息解码 解码二进制成unicode
print("接收的客户端消息:", msg.decode('utf-8'))

# 4.发送消息
# encode() 消息编码 编码unicode成二进制
server.sendto("你好,收到你的消息了".encode('utf-8'), addr)

# 5.关闭套接字对象
server.close()

udp客户端代码

"""
客户端:
    1.创建套接字
    2.发送/接收消息
    3.关闭套接字
"""
# 导入socket模块
import socket

# 创建套接字
client = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
# 发送消息
msg = input(">>>:")
client.sendto(msg.encode("utf-8"), ("127.0.0.1", 9999))

# 接收服务端回应的消息
data, addr = client.recvfrom(1024)
print(addr)
print("服务端回应:", data.decode("utf-8"))

# 关闭套接字对象
client.close()

udp循环服务端代码

"""
服务端
    1.创建套接字
    2.绑定ip端口
    3.发送/接收消息
    4.关闭套接字
"""
# 导入socket模块
import socket

# 1.创建套接字
server = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)

# 2.绑定ip端口
ADDR = ("127.0.0.1", 9999)
server.bind(ADDR)

# 循环接收发消息
while True:
    # 3.接收消息
    msg, addr = server.recvfrom(1024)  # 阻塞状态
    if msg.decode("utf-8").upper() == "Q":
        print(f"客户端{addr}不点了")
        break
    # 打印客户端发送的消息
    print(f"接收客户端{addr}点的菜:", msg.decode('utf-8'))

    # 4.发送消息
    server.sendto(f"你好,收到你点的菜品了{msg.decode('utf-8')},马上准备".encode("utf-8"), addr)

# 5.关闭套接字对象
server.close()

UDP套接字特点:

  • ​ 可能会出现数据丢失
  • ​ 传输过程简单,实现容易
  • ​ 数据以数据包的形式传输
  • ​ 数据传输效率高

2.TCP套接字

2.1 面向连接的传输服务

  • 提供了可靠的数据传递,传输过程中无丢失，无失序，无差错，无重复
  • 可靠性保障机制(自动完成)
    • 在通信之前需要建立数据连接
    • 确认应答机制
    • 通信结束后正常断开连接

2.2 三次握手

  • 客户端向服务器发送报文请求连接
  • 服务收到请求回复报文可以连接
  • 客户端收到回复再次发送报文建立连接

2.3 四次挥手

  • 发起方发送报文请求断开
  • 接收方收到请求回复信息收到,并准备断开
  • 接收方准备完成，再次发送表示可以断开
  • 发送方收到确定，发送最终消息完成断开

tcp套接字细节

  • tcp连接中一端退出，另一端依然阻塞在recv，此时recv会马上返回一个空字节
  • 如果一端已经不存在,另一端任然通过send向其发送数据则会产生BrokenPipeError
  • 一个服务端可以同时连接多个客户端,也能够被重复连接

TCP编程流程

  • socket：创建套接字
  • bind：绑定地址
  • listen：设置监听
  • accept：等待处理连接
  • send/recv：发送/接受消息
  • close：关闭连接
  • connect: 客户端像服务端发起连接请求

2.4 单次通讯

tcp服务端代码

"""
    TCP服务端
    1.创建套接字对象 socket
    2.绑定ip端口 bind
    3.设置监听 listen
    4.等待客户端发起连接 accept
    5.发送/接收 send recv
    6.关闭连接对象 close
    7.关闭套接字 close
"""
# 导入socket
import socket

#  1.创建套接字对象 socket
# AF_INET ipv4
# SOCK_STREAM 表示选择tcp套接字
server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
# 2.绑定ip端口 bind
server.bind(('127.0.0.1', 8888))

# 3.设置监听 listen
# 因为我们在这个中间是需要建立连接的
# 这个服务端它要一直监听,随时等待客户端连接他
# 5表示最大的连接数
server.listen(5)
print("服务端建立成功端口号为8888")

# 4.等待客户端发起连接 accept
# 第一个返回值:与客户端建立连接的对象
# 第二个返回值:客户端的地址
conn, addr = server.accept()  # 阻塞状态
print(f"与{addr}客户端连接成功")

# 5.发送/接收 send recv
# 接收客户端的消息
# udp:sendto recvfrom 没有建立连接
# tcp:send recv       建立起来连接
data = conn.recv(1024).decode("utf-8")  # 阻塞状态

# 打印客户端的消息
print(f"{addr}客户端点菜:{data}")
# 回应客户端
conn.send(f"{data}菜品收到了,我们的厨师已经在准备了".encode("utf-8"))

# 6.关闭连接对象 close
conn.close()
# 7.关闭套接字 close
server.close()

tcp客户端代码

"""
tcp客户端:
    1.创建套接字 socket
    2.发送连接请求 connect
    3.发送/接收 send/recv
    4.关闭套接字 close
"""

import socket
client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# 向服务端发送连接请求
client.connect(('127.0.0.1', 8888))
print("与服务端连接成功")

# 发送消息给服务端
msg = input("点菜:").encode("utf-8")
client.send(msg)

# 接收服务端的回应
data = client.recv(1024).decode("utf-8")
print(f"服务端回应的是:{data}")

# 关闭套接字对象
client.close()

2.5 循环通讯

tcp服务端代码

# 导入socket
import socket

#  1.创建套接字对象 socket
server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
# 2.绑定ip端口 bind
server.bind(('127.0.0.1', 8888))

# 3.设置监听 listen
server.listen(5)
print("服务端建立成功端口号为8888")

# 4.等待客户端发起连接 accept
conn, addr = server.accept()  # 阻塞状态
print(f"与{addr}客户端连接成功")

# 循环收发消息
while True:
    # 5.发送/接收 send recv
    data = conn.recv(1024).decode("utf-8")  # 阻塞状态
    if not data:
        print(f"与{addr}客户端断开连接")
        break

    # 打印客户端的消息
    print(f"{addr}客户端点菜:{data}")
    # 回应客户端
    conn.send(f"{data}菜品收到了,我们的厨师已经在准备了".encode("utf-8"))

# 6.关闭连接对象 close
conn.close()
# 7.关闭套接字 close
server.close()

tcp客户端代码

import socket

client = socket.socket(socket.AF_INET, socket.SOCK_STREAM)

# 向服务端发送连接请求
client.connect(('127.0.0.1', 8888))
print("与服务端连接成功")
# 循环接收发消息
while True:
    # 发送消息给服务端
    msg = input("点菜:")
    if not msg:
        print("请重新点菜")
        continue
    if msg.upper() == "Q":
        print("不点了")
        break
    # tcp协议不允许发送空数据
    client.send(msg.encode("utf-8"))

    # 接收服务端的回应
    data = client.recv(1024).decode("utf-8")
    print(f"服务端回应的是:{data}")

# 关闭套接字对象
client.close()  # 四次挥手 在python中相当于发送了空数据

2.6服务端再服务

tcp服务端代码

# 导入socket
import socket

#  1.创建套接字对象 socket
server = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
# 2.绑定ip端口 bind
server.bind(('127.0.0.1', 9999))

# 3.设置监听 listen
server.listen(5)
print("服务端建立成功端口号为8888")

# 循环处理下一个客户端
while True:
    # 4.等待客户端发起连接 accept
    conn, addr = server.accept()  # 阻塞状态
    print(f"与{addr}客户端连接成功")

    # 循环收发消息
    while True:
        # 5.发送/接收 send recv
        data = conn.recv(1024).decode("utf-8")  # 阻塞状态
        if not data:
            print(f"与{addr}客户端断开连接")
            break

        # 打印客户端的消息
        print(f"{addr}客户端点菜:{data}")
        # 回应客户端
        conn.send(f"{data}菜品收到了,我们的厨师已经在准备了".encode("utf-8"))

    # 6.关闭连接对象 close
    conn.close()
# 7.关闭套接字 close
server.close()

3.使用场景

  • TCP适合对准确要求高，传输数据大的场景

    • 文件传输:数据下载(电影,音乐)，上传照片，访问网站
    • 邮件收发
    • 点对点数据传输:登录，远程，红包，一对一聊天

  • UDP适合可靠性要求相对不高，传输自由的场景

    • 视频流：直播，视频聊天
    • 广播：网络广播，群发消息
    • 实时性要求高：如游戏等
